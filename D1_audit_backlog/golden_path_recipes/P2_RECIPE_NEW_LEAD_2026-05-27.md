# П2. Рецепт «новый лид» (первичный входящий) — главный сценарий бота. 2026-05-27

Это сценарий, ради которого бот и делается: **холодный новый клиент, вечерний трафик с сайта.** Здесь НЕ работают ретеншн-приёмы для «своих» (отзыв преподавателя, «продолжаете?») — у нового нет истории и доверия к центру.

## Выборка (честно)
- Отбор «нового лида» структурный: телефон встретился в корпусе **1 раз** + нет маркеров «свой» + клиент реально говорил. Получилось **1 240 звонков** (июн 122 · июл 121 · авг 176 · сен 356 · окт 465 — поток растёт к осени).
- Дошли до оплаты — только **48** (≈4%). Это сам по себе вывод: **холодный новый почти никогда не платит в первом контакте** — цель первого касания не «продать», а квалифицировать + назначить следующий шаг.
- Приёмы достал глубоким чтением (субагенты): конвертировавшие читались первыми. Оговорка: выборка по интентам ~50-60 звонков на тему; «новый» — по структурному прокси (1 звонок), не по флагу.

## Как новый клиент открывает разговор (реальные триггеры)
- «Подскажите, что у вас вообще есть по [предмет] для [класс]?»
- «Какой формат — онлайн или очно? Где находитесь?»
- «Сколько стоит? А можно частями?»
- «Нам нужно подтянуть [физику/математику], не олимпиады, просто уровень.»
- «А расписание какое? У нас ещё секции/другие занятия.»
- (Олимпиады/ЕГЭ) «А реально подготовить, если раньше не занимались? Потянет?»

## Что движет решением нового клиента
Формат (онлайн/очно, есть ли запись урока) · расписание (главный фактор!) · цена и возможность платить частями · уровень/«потянет ли ребёнок» · доверие к незнакомому центру.

## Каркас ответа боту (для нового лида)
1. **Представиться брендом канала**, без анкеты из 5 вопросов.
2. **Один-два уточняющих**: класс + предмет + цель (+ формат).
3. **Дать заземлённый ответ сразу**: формат (онлайн/очно, запись), как подбирается группа (**тестирование = подбор уровня, не экзамен** — гасит «не потянет»), цена с вариантом оплаты по частям одним блоком.
4. **Снять один барьер** (расписание / цена / незнакомый центр) — фактом, не давлением.
5. **Конкретный следующий шаг с дедлайном**: «пришлю на email расписание/материалы и квитанцию, оплата в течение N дней, занятия с [дата]» **и** назначить, кто и когда свяжется. Без назначенного шага новый уходит «подумать» навсегда.

## Что реально сработало у дошедших до оплаты (winning moves)
1. **Быстрая запись без разгона:** класс+предмет+формат → сразу конкретное время группы → «записываю?».
2. **Цена + структура оплаты в одном блоке** (год И семестр И «можно частями») — не заставлять клиента переспрашивать.
3. **Конкретный шаг с дедлайном цены** («оплатить до [дата] — цена сохраняется») — даёт импульс без манипуляции.
4. **Объяснить «живой онлайн + запись урока»** — снимает барьер у тех, кто такого формата не видел.
5. (Олимпиады/ЕГЭ) **тест + возможность перейти между группами** — гасит «вдруг не потянет».

## Где новые срываются (точки отвала — лечить в первую очередь)
1. **Расписание не совпадает** с жизнью ребёнка (другие секции, день, удалённость) — блокер №1. Менеджерское «перезвоним, когда появится расписание» = клиент потерян. Боту нужен мягкий мост: лист ожидания + **назначенный** перезвон с датой, а не открытое «появится — сообщим».
2. **«Пришлите материалы» без назначенного перезвона** — второй по частоте отвал.

## Факты — по active_brand (как в RECIPES_v2)
Цены/форматы/скидки/оплата — только из `client_safe_facts_<brand>` (см. RECIPES_v2, числа сверены). Для нового клиента это те же таблицы; меняется только хореография разговора (этот файл), не факты.

## P0 и бренд-предохранители (в звонках новых лидов реально встретились — боту ЗАПРЕЩЕНО)
- «Доведём до финала олимпиады», «у нас дети хорошо сдают», «готовим до заключительного этапа» — **гарантия результата → запрет**, говорить только статистику без обещания конкретному ребёнку.
- «Ранее сотрудничали с Фотоном» (от менеджера УНПК) — **нарушение бренд-разделения → запрет**.
- Возврат/спор оплаты/жалоба → менеджер.

## Что доделать
- Рецепт построен на структурном прокси «новый» (1 звонок) — для точности нужен флаг входящий/первичный (`call_type`/разметка) и больше конвертировавших новых (их всего 48).
- Главный источник нового трафика для бота — **входящие в Telegram / чат сайта** (а не звонки-обзвоны). Этот рецепт надо обкатать и дополнить на них.
- Прогон каркаса на симуляторе — силами Codex (зелёные тесты ≠ готовность).
