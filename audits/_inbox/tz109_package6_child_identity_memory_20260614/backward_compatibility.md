# Backward compatibility

- `TELEGRAM_CHILD_IDENTITY_MODEL` default OFF, поэтому существующее поведение без флага сохранено.
- Существующие вызовы `build_dialogue_memory()` совместимы: новый параметр `resolved_children` имеет значение по умолчанию.
- Существующие вызовы `build_telegram_pilot_context()` совместимы: новый параметр `resolved_children` имеет значение по умолчанию.
- Старый сценарий `дочка/младший -> child_2` покрыт NEG-тестом при выключенном флаге.
- Сценарий двух детей в одном сообщении сохраняет разделение `child_1`/`child_2` даже при включенном флаге.
