# Backward Compatibility

## Что осталось совместимым

- Публичная структура `SubscriptionDraftResult` не менялась.
- API `detect_high_risk_input_markers` не менялся.
- Маршруты `draft_for_manager`, `manager_only`, `blocked` не менялись.
- Базовые safety-флаги `manager_approval_required` и `no_auto_send` не менялись.

## Что изменилось намеренно

- `theme:005_discounts` больше не является автоматическим `manager_only`.
- Нейтральные скидочные вопросы могут оставаться `draft_for_manager`.
- Высокорисковые формулировки по возврату, налогам, маткапиталу, жалобам, юридическим угрозам и статусу оплаты ловятся шире.

## Почему это безопасно

Система по-прежнему не отправляет сообщения клиентам сама. Даже `draft_for_manager` означает черновик для менеджера, а не автоответ. Для действительно опасных тем route остаётся `manager_only`.
