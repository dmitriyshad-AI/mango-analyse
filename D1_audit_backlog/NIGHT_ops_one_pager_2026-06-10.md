# Ночной контур и отказоустойчивость пилота — одностраничник (решение №4 Дмитрию)

Закрывает три пропущенных класса аудита: надёжность пилот-контура, мониторинг, деньги.

## 1. Сейчас (черновиковый пилот, день)

- Ночные входящие: бот черновики НЕ делает (поллер можно не гонять ночью) ЛИБО делает — менеджер утром разбирает. РЕШЕНИЕ: рекомендую гонять круглосуточно (черновики утром экономят менеджеру час), но ночные P0 ждут утра — менеджер начинает день с примечаний «бот передал менеджеру», P0 первыми. Это уже дописано в памятку.
- Heartbeat поллера: правило «0 черновиков за 3 рабочих часа при входящих ≠ 0 = тревога». Дёшево: поллер пишет heartbeat-файл с временем последнего цикла; мой ежечасный монитор проверяет и сигналит в чат. (Мини-пункт в ТЗ-9-стиле, 20 строк кода — Кодексу вместе с live-пробой.)
- Перезапуск: runbook из трёх строк в README draft loop (как проверить журнал, как перезапустить --loop, когда снять/поставить STOP). launchd-автостарт — после недели стабильного ручного режима.
- Протухание токенов (Wappi/AMO/codex-auth): класс сегодняшнего инцидента. Симптом в журнале = серия ошибок 401/403 → поллер пишет статус `auth_error` и ОСТАНАВЛИВАЕТСЯ (не молотит), менеджер работает как раньше. Обновление токена — runbook-строка.

## 2. К ночной автономии (НЕ сейчас, проектное решение заранее)

- Ночной P0: автоответ-удержание («приняли, ответственный сотрудник свяжется утром, с 10:00») + утренний дайджест P0 первым делом. Дежурный человек НЕ требуется до заметного ночного трафика; алерт в Telegram Дмитрию — опция при первых неделях.
- Ночной STOP-протокол: стоп-файл работает и ночью; кто может нажать — Дмитрий удалённо (файл в ~/.mango_secrets на маке — доступ через экранное подключение) — проверить процедуру один раз днём.

## 3. Деньги (честная оценка)

Токены сейчас НЕ пишутся (только счётчики вызовов) — точная стоимость черновика невозможна. По вызовам: черновик = 1 draft + ~0.65 верификатор + ~0.12 реген + 1 memory ≈ 2.8 вызовов модели. Грубая вилка при типичных ценах подписочного Codex — ДЕШЕВО на пилотных объёмах (десятки черновиков/день), реальный лимит — квоты подписки, не доллары. РЕШЕНИЕ не требуется сейчас; к автоному: (а) добавить токен-счётчик в раннер/поллер (гипотеза G6, дешёвая), (б) после недели пилота посчитать «черновиков на 1 запись» и решить про дневной потолок вызовов в поллере (страховка от зацикливания — само по себе полезно: лимит N черновиков/чат/день уже есть смыслово через дебаунс+объединение, добавить общий дневной cap = 3 строки).

## Решение №4 Дмитрия (одним «ок»)

Гонять поллер круглосуточно с ночными P0-удержаниями к этапу автономии; heartbeat+auth_error-стоп — Кодексу мини-пунктами; дневной cap вызовов в поллер; токен-счётчик в раннер после пилотного старта. Если «ок» — внесу пункты в ТЗ live-пробы/ТЗ-9 без отдельного цикла согласований.
