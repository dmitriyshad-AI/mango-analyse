## 2.A — Тон (item 5), флаг TELEGRAM_PH2_TONE
Метрика тона уже есть (`tone_score.py`). Здесь — ПОНИЗИТЬ канцелярит в выходе: на ходах с высоким `tone_canc` направить X2-переписать ответ по рубрике (прямой ответ первой фразой, признание запроса, шаг ОТ бота, убрать «в рамках текущего учебного центра»/«осуществляется»/дежурный хвост). НЕ менять факты/маршрут — только формулировку.
- NEG (КРИТИЧНО, урок R1 — gold-перепись поднимала выдумку 0→1-3): тон-перепись может ВЫДУМАТЬ. Поэтому faithfulness + число-гард ОБЯЗАТЕЛЬНО прогоняются ПОСЛЕ переписи, а не только до. Если перепись добавила/исказила число/факт/бренд — откат к исходному ответу. Перепись меняет ТОЛЬКО формулировку.
- Контроль: канцелярит-ответ → tone_canc падает после переписи; факты/числа неизменны; искусственно «улучшенный» ответ с лишним числом → откат.

## 2.B — Возражения (item 4), флаг TELEGRAM_PH2_OBJECTION
`detect_objection` → выбрать безопасный приём-шаблон (наполнение — грунтованными фактами через существующие гарды):
- **price** → признать + ценность + (если факт есть и бренд верный) напомнить про рассрочку/Долями — **рассрочка только Фотон, у УНПК её НЕТ** (active_brand!); предложить подобрать формат под бюджет. НЕ выдумывать скидку.
- **think** → честная рамка + лёгкий шаг («могу прислать расписание/пробное»). Без давления.
- **compare** → сильные стороны своей программы общими словами, без принижения конкурента и без сравнения брендов.
- **format** → ОДИН уточняющий вопрос (переиспользовать B1 clarify) → ответ из факта нужного scope.
- **distance** → оценка дороги (Level A, с маркером) + онлайн-альтернатива, если есть.
- NEG: P0/возврат → менеджер (возражение не перебивает P0); cross-brand → каноничная фраза; число-/scope-гард держат; шаблон не дублирует уже сказанное.
- Контроль: «дорого» → приём price (не дженерик-хендофф); «дорого, хочу вернуть деньги» → P0 менеджер (приём не маскирует претензию).

## 2.C — Тревога (item 6) + детский диагноз, флаг TELEGRAM_PH2_ANXIETY
`detect_anxiety` → префикс-признание эмоции, ПОТОМ содержательный ответ по фактам:
- **capability** («не потянет») → «нормально переживать; программа от базового уровня, преподаватель в мини-группе» (по факту формата) + пробное.
- **late_start** («поздно начинать») → спокойно «начать можно и сейчас», старт-даты по фактам, без вины.
- **level_fit** («подойдёт ли / справится ли по уровню») → **НЕ диагностировать самому** (страж general_advice): «уровень лучше подобрать на пробном/с менеджером», тёпло. **Это и есть фикс `foton_advice_diagnosis`** — индивидуальную оценку ребёнка бот не подтверждает уверенным «Да».
- NEG (КЛЮЧЕВОЙ): тревога ≠ P0, НО реальная претензия/жалоба/возврат → менеджер P0; бот не ставит индивидуальный диагноз ребёнку; не выдумывает «справится» как факт.
- Контроль: «справится ли дочка» → хедж + передать (НЕ уверенное «Да»); «боюсь не потянет» → признание + факт формата (не хендофф); реальная жалоба → P0.

## Wiring (спека, Кодекс верифицирует интеграцию)
Детекторы — на слое понимания (рядом с планнером/intent, на существующем понимании, без новых LLM-вызовов). Порядок: P0/scope/бренд-гарды ПЕРВЫМИ (они перебивают), затем objection/anxiety-приём выбирает шаблон, затем наполнение грунтованными фактами через число-/scope-гард, затем тон-перепись (2.A), **затем ПОВТОРНО faithfulness + число-гард** (ловят выдумку переписи, урок R1), затем output-санитайзер. Приём НИКОГДА не идёт впереди P0/cross-brand/число-гарда.
