## Спецификация экстрактора v1 (по требованию аудита — таблицей, не «на усмотрение»)

| Поле | Паттерны записи (только из реплик КЛИЕНТА, регистронезависимо) | Примечания |
|---|---|---|
| grade | `(\d{1,2})[ -]?(й|ый|ого)? класс`, `за (\d{1,2}) класс`, `класс[: ]+(\d{1,2})`, «перешли в N», «заканчивает N» (→N+1 НЕ вычислять — писать как сказано + quote) | 1-11; вне диапазона — не писать |
| subject | словарь: математика/матем, физика, информатика/программирование, русский, английский… (+склонения) | только из словаря, никаких догадок |
| format | очно/очный/в центр/приезжать; онлайн/дистанц/из дома | оба упомянуты → оба слота с разными quote |
| branch/площадка | Красносельская, Сретенка, Долгопрудный, Институтский, Пацаева, Менделеево + «рядом с метро X» (писать сырьём) | не нормализовать к бренду |
| child_name | имя из именного контекста: «сына/дочь зовут X», «для X», «X в N классе» (словарь имён + морфология как в П1) | лемма + quote |
| child_key | новый при: «втор(ой|ая)», «младш», «старш», «ещё один реб», другое имя в именном контексте | по умолчанию child_1 |
| payment_pref | «частями», «помесячно», «за семестр», «за год», «маткапитал» | — |
Конфликт значений поля → новое побеждает, старое superseded (хранится). Реплики бота/менеджера слоты НЕ создают. Слот без quote в промпт не попадает.

Боль: память наполняется эвристиками и может «вспомнить» небывшее (исторический инцидент Волны 2: память додумывает; эхо имени закрыто presale, но домысленные слоты живы). Бонус: memory-LLM-вызов идёт на КАЖДЫЙ ход (321/321 в wave4) = ~36% боевых LLM-затрат — после этого ТЗ кандидат на замену детерминированной сборкой.

## Дизайн v1 (намеренно детерминированный — без нового LLM-вызова)

Слот пишется ТОЛЬКО при точном совпадении в РЕПЛИКЕ КЛИЕНТА (не менеджера, не бота) и хранит происхождение:

```python
@dataclass(frozen=True)
class MemorySlot:
    field: str            # "grade" | "subject" | "format" | "branch" | "child_name" | ...
    value: str
    turn_index: int       # номер реплики клиента
    quote: str            # дословная цитата-носитель (≤120 симв.)
    child_key: str = ""   # "child_1"/"child_2" — слоты ПО РЕБЁНКУ, не по диалогу (семьи с 2+ детьми)
```

Правила:
- В промпт попадают только слоты с непустым quote; рендер: «класс: 7 (клиент, реплика 3: „ребёнок в 7 классе")».
- Конфликт значений (клиент поправился) → побеждает поздний, старый помечается superseded (не удаляется — для регрейда).
- Смена ребёнка в диалоге («а для младшей…») → новый child_key, слоты не смешиваются.
- Свободное LLM-резюме: при флаге ON НЕ генерится (вместо него — детерминированная сборка слотов + последние 10 реплик). Это и есть экономия memory-вызова.
- Эвристики-источники текущей памяти при ON отключаются (NEG: OFF — байт-в-байт).

## NEG/тесты
- OFF ⇒ поведение байт-в-байт.
- Слот без цитаты в промпт не попадает (юнит).
- Реплика менеджера/бота слот НЕ создаёт.
- Два ребёнка: «сыну в 7, дочке в 4» → два child_key, вопрос о цене относится к последнему упомянутому, бот при неоднозначности переспрашивает (golden-сценарий).
- Латентность/вызовы: при ON memory-роль = 0 вызовов (summary-метрика llm_calls).
