# ТЗ — F8: Оси подбора в базе знаний (каталог продукт+цена). Диалог D3 (v2, после аудита)

Дата: 2026-06-21. v2 — числа сверены по реальному снимку r4.1, добавлена зависимость «селектор↔бот». Зона: база знаний/каталог. Канон `main`. Реализация (B/C) — только после моей сверки Этапа A.

## 0. Проблема

Снимок KB не имеет осей подбора → бот не может сам подобрать «продукт+цену под запрос» и уходит к менеджеру. Это структурный рычаг против over-handoff.

| Ось в `structured_value` | Фактов | Примечание |
|---|---|---|
| amount (точная цена) | 54 | у регулярных Фотон-курсов есть; у УНПК онлайн — НЕТ |
| period | 70 | semester/year/week |
| format | 98 | online/offline |

Задача Этапа A: переписать это в findings-документ + явно отметить **слепые зоны**: (а) **subject** не хранится нигде у регулярных курсов; (б) **цены УНПК онлайн** (37 000 / 41 800 / 59 000 / 69 900) есть только в тексте, `amount=None`; (в) тарифы М9/М11 (18 900/47 250/59 900/94 500) — отдельными строками в тексте, не фактами с amount; (г) 1 факт-диапазон (Фотон онлайн 3-4 кл./год 29 900–33 300) — `amount_min/max`, не точка. Стоп на мою сверку.

## 2. Этап B — модель осей (после сверки). Разделить два источника заполнения:

- **Механически выводимо из `fact_key` (БЕЗ внешнего источника):** `format`, `period`, `grade_min/grade_max` (правило `3_4→min3,max4`; `5_11→min5,max11`), `brand`. Заполнить.
- **Требует ИСТОЧНИКА (КЦ-база Google Drive), без него не выдумывать:** `subject` у регулярных курсов; `amount` у УНПК онлайн; `tariff_id`+`amount` для тарифов М9/М11. Пометить «ось отсутствует», вынести расхождения мне.
- **Риск пересборки KB:** перед пересборкой зафиксировать список изменённых `fact_id`; ПОСЛЕ — грепнуть их `client_safe_text` (мис-экстракция недопустима — в снимке уже есть следы: 2 факта ранней брони с пустым текстом).

## 3. Этап C — селектор подбора + ПОДКЛЮЧЕНИЕ к боту (критично)

- Селектор по осям: (класс, предмет, формат, период, бренд) → ОДНА позиция с точной ценой или явный «нужен уточняющий слот».
- **Зависимость (из аудита):** `answer_registry` — СИРОТА (к боту не подключена), её НЕ использовать. Живой ретривер — `fact_retrieval._select_confirmed_facts` в `telegram_pilot_context_builder`, но его `KEY_ALIASES` не покрывают grade/subject. **Селектор встроить рядом с `_select_confirmed_facts` (или вызвать до него) в живом пути; иначе он работает только в тестах и боту не помогает.** За флагом, текущий путь не ломать.

## 4. Приёмка + NEG

- **Приёмка:** «Фотон / 9 класс / онлайн / год» → одна точная цена из осей. Семантик-ревью базы PASS.
- **Открытый кейс (честно):** «УНПК / онлайн / 9 класс / год» сейчас НЕ закрывается (amount=None в SV) — либо внести цены в источник, либо пометить как не-в-scope до пополнения.
- **NEG:** нет оси (нет предмета) → запрос уточняющего слота, не догадка; факт-диапазон 29 900–33 300 → НЕ финальный ответ, а уточнение «какой тариф»; бренд не смешивается.
