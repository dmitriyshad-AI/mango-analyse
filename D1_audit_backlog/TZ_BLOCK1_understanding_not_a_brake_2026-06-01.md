## СТАРТ Блока 1 — ПОСЛЕ регрейда A+B+A2 (условие входа)

## Проблема (по аудиту + регрейду A)
Верхний слой душит вход в ответ ДВУМЯ способами:
1. **Рамка «низкая уверенность → менеджер».** `build_understanding_prompt`, строка **334**:
   «Если факта нет или уверенность низкая: answerability=manager_only…». Для автономного бота это вредно:
   низкая уверенность должна означать «найди факт / уточни одним вопросом / ответь частично», а не сразу уход.
2. **P0-форс regex поверх модели.** `parse_contract` **383-388**: `is_p0 = модель OR p0_reason_pregate`;
   при is_p0 → `answerability=manager_only`. `p0_pre_gate` (**443**) форсит P0 по `codes_from_text`
   (гигантские regex p0_recall_spec) ИЛИ классификатору. Regex перетриггеривает (слова «обман/оплатил/
   верн…» в безобидном контексте) → лишний manager_only → over-handoff.

## Решение

### Часть 1 — understanding-промпт: низкая уверенность ≠ менеджер
Строка 334 заменить по смыслу на:
```
"- Если прямого факта нет, но в каталоге есть ключ, который ПО СМЫСЛУ покрывает вопрос — поставь его в "
"needed_fact_keys и answerable='self' (ответит из факта/частично). Если вопрос неоднозначен — задай ОДИН "
"уточняющий подвопрос, не уходи к менеджеру. answerability=manager_only ТОЛЬКО при P0 или когда в каталоге "
"реально нет покрывающего ключа. current_question заполняй всегда.\n"
```
Цель: перестать штамповать manager_only при малейшей неуверенности. Сетка A/A2 + verify_output ловят, если
модель переоценит покрытие (выдумка не уйдёт клиенту).

### Часть 2 — P0-точность: модель первична, regex — сеть только для ЯВНЫХ P0
В `p0_recall_spec`/`parse_contract` разделить P0-коды на ДВА класса:
- **HARD (всегда форсят manager_only, не ослаблять):** реальная претензия возврата (`verните деньги`,
  спор оплаты PAYMENT_DISPUTE), юридическая угроза LEGAL, явная жалоба COMPLAINT-explicit, обещание
  результата/поступления. Эти regex остаются жёсткими.
- **SOFT/неоднозначные (совещательные):** одиночные слова-маркеры в нейтральном контексте («обман» в
  вопросе «вас не обманывают конкуренты?», «оплатил» в «я уже оплатил, когда старт?»). Для SOFT-кода:
  если модель уверенно `is_p0=false` (confidence высокая) И нет HARD-кода → НЕ форсить manager_only;
  отдать решение модели.
Правка в `parse_contract` 383-388: `is_p0 = model_is_p0 OR has_HARD_pregate_code`; SOFT-код идёт в
`p0_reason` как сигнал, но не форсит. Подтвердить чтением структуру `codes_from_text`/`classify_answer_safety`
и пометить, какие коды HARD, какие SOFT.

### Часть 3 — мягкие жалобы распознаёт МОДЕЛЬ (вливаем сюда TZ_complaint_handling)
В understanding-промпт добавить: жалоба/недовольство по СМЫСЛУ (не только слово «жалоба») —
«ребёнок ничего не понял», «зря заплатили», «толку нет», «не нравится как ведут» → is_p0=true,
p0_reason=complaint, manager_only, эмпатичный хендофф без сбора данных (инфраструктура
`_complaint_handoff_text` есть). COMPLAINT_RE остаётся страховкой для явных. Жалоба-с-вопросом → приоритет
у жалобы. (Это закрывает прежнее ТЗ жалоб — отдельно не делать.)
NEG: гипотетический возврат до оплаты ≠ жалоба (refund_policy, бот отвечает про остаток) — не сломать.

### Часть 4 — подтвердить, что latch-release P0 не ослаблен
В git уже есть «Wave 7 autonomous P0 latch release on N neutral turns». Убедиться чтением, что после
HARD-P0 латч отпускается по нейтральным ходам и не глушит весь диалог навсегда (из регрейда: P0 на одном
ходе не должен глушить безобидные следующие). Не дублировать, только проверить.

## Тесты + НЕГАТИВНЫЙ контроль (без него блок не закрыт)
ПОЗИТИВ:
- «не уверен, но есть ключ по смыслу» → answerable=self, бот отвечает/уточняет, НЕ уходит.
- мягкая жалоба «ребёнок ничего не понял» → complaint manager_only, эмпатично без данных.
- SOFT-маркер в нейтрале («вас не обманывают?») → НЕ P0, бот отвечает.
НЕГАТИВ (критично — не ослабить):
- реальная претензия «верните деньги, занятия не идут» → HARD P0 manager_only.
- спор оплаты / юр.угроза / обещание результата → manager_only (HARD-коды форсят).
- гипотетический возврат до оплаты → refund_policy, отвечает про остаток (не сломать).
- cross-brand → бренд-гард держит; выдумки 0 (A/A2 сетка).
- автономия выключена / бренд unknown → как сейчас.
