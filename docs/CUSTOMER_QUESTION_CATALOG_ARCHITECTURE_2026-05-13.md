# Архитектура единого каталога вопросов клиентов

Дата: 2026-05-13

## Зачем это нужно

Единый каталог вопросов клиентов нужен, чтобы будущий бот и рабочее место менеджера опирались не на случайные примеры, а на накопленную историю общения: звонки, Telegram и почту. Каталог не является готовой базой ответов для автоматической отправки. Это безопасный слой подготовки: он собирает вопросы, объединяет похожие, показывает частотность, предлагает черновые шаблоны ответов и явно отмечает, где нужен актуальный факт или проверка РОПа.

Главный принцип: бот не должен сам придумывать цены, расписание, скидки, документы и адреса. Для таких тем каталог хранит шаблон ответа и требование к актуальному источнику факта.

## Источники

Каталог строится только из локально доступных артефактов и не делает живых записей во внешние системы.

| Источник | Что читаем | Режим |
|---|---:|---|
| Звонки | `stable_runtime/sales_insight_knowledge_base_after_quality_backfill_20260510_v11_frozen_gate/enriched_reviews.csv` | Только чтение |
| Telegram | `telegram_exports (2)/local_vm_2024-04-01/messages.jsonl` | Только чтение |
| Почта | `_external_handoffs/mail_archive_2026-05-12/**/mail_archive.sqlite` | Только чтение, `mode=ro` |
| Справочные материалы | `docs/`, `.codex_local/kc_source_extract_20260513/texts/` | Только поиск потенциальных источников фактов |

## Поток обработки

1. Извлечение кандидатов: из звонков, Telegram и почты берутся входящие вопросы клиентов и, где возможно, ближайший ответ менеджера.
2. Очистка текста: телефоны, почты и другие рискованные персональные данные убираются из публичных выходных файлов.
3. Нормализация: похожие формулировки приводятся к единому ключу.
4. Классификация: вопрос получает тему, например стоимость, расписание, адрес, документы, программа, скидка, пробное занятие.
5. Объединение: одинаковые или близкие вопросы собираются в класс вопроса.
6. Шаблон ответа: для каждого класса создается черновой шаблон, но без подстановки неподтвержденных цен или расписания.
7. Проверка фактов: для тем с меняющейся информацией создается требование к актуальному источнику факта.
8. Выгрузка для РОПа: формируется таблица, где можно утвердить, исправить или запретить ответы.

## Основные сущности

`QuestionItem` - отдельный найденный вопрос или клиентский сигнал. В нем хранится очищенный текст, источник, дата, тема, требуемые факты и безопасная ссылка на источник без прямых контактов.

`QuestionClass` - группа похожих вопросов. Это рабочая единица для РОПа: один класс должен получить один утвержденный подход к ответу.

`AnswerTemplate` - шаблон ответа. Для цен, расписания, скидок, программ и документов он не является финальным ответом, пока не подключен актуальный файл фактов.

`CurrentFactSource` - найденный источник, где потенциально могут лежать цены, расписание, адреса, скидки, документы или программы. По умолчанию такие источники требуют ручного подтверждения свежести.

## Статусы ответов

| Статус | Значение |
|---|---|
| `template_ready_needs_current_fact` | Есть шаблон, но нужен актуальный факт: цена, расписание, адрес, скидка, документ или программа |
| `draft_answer_exists_needs_review` | Есть черновой ответ, но его должен проверить РОП |
| `manager_only` | Тема должна уходить менеджеру, бот не отвечает сам |
| `needs_rop_answer` | Данных мало, нужен эталонный ответ от РОПа |

## Безопасность

Слой каталога не скачивает аудио, не запускает распознавание, не запускает Resolve/Analyze, не пишет в AMO, Tallanto, CRM, Telegram или почту, не меняет runtime-базы и не удаляет исходные файлы.

Выходные файлы лежат только в `product_data/question_catalog/`. Защита пути не позволяет записывать результаты в `stable_runtime`.

## Что это дает продукту

Каталог становится мостом между историей общения и будущим ботом. Вместо того чтобы бот отвечал на основе старой разрозненной базы знаний, мы получаем список реальных вопросов клиентов, частотность этих вопросов, безопасные черновики ответов и явные места, где нужна актуальная цена, расписание или решение менеджера.

Следующий продуктовый шаг - дать РОПу таблицу проверки, утвердить классы ответов, подключить актуальные файлы цен и расписания, а затем использовать только утвержденные классы в предпросмотре ответа бота.
