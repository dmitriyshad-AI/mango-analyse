# Риски

- Клиентский риск: клиентских ответов нет; результат предназначен для менеджерского preview. Для будущего бота поля CRM-проекции нельзя считать bot-safe.
- Данные/записи: live-записи нет. AMO/Tallanto/CRM write не выполнялся. Источник timeline открыт read-only.
- Identity: семейные/общие/ambiguous телефоны fail-closed, по-сделочные поля блокируются.
- AMO join: текущий `read_api` маскирует AMO ids; это блокирует все строки workbook и требует решения контракта до Этапа 2.
- Платежи: суммы/статусы Tallanto не выводятся в bot-safe; перед менеджерским действием платежные сигналы требуют сверки.
- Откат: все изменения за default-OFF флагами, кроме добавления новых read-only модулей и тестов. При выключенных флагах старые writer/builder paths сохраняют прежнее поведение.
