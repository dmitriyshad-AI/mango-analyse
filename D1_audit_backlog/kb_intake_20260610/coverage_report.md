# Сверка покрытия базой знаний (Этап Б). Клод #2, 2026-06-10

Главный результат работы. Для каждого кластера вопросов (Этап А, кроме P0) — отвечает ли база ПО СМЫСЛУ
(бот может ответить ТЕКСТОМ клиентского факта, а не «слова похожи»): **полное / частичное / НЕТ**, с fact_key.
Эталон — снимок r3 (`kb_release_20260610_v6_7_staging_r3`, 668 клиентских фактов). Бренды раздельно.
Сверку вели 2 субагента (факт; процесс+совет), выводы выборочно перепроверены по сырью.

## Итог покрытия (коротко)

База в ХОРОШЕМ состоянии — лучше, чем показывал ранний скан процессов: команда уже завела **60 фактов-
процессов** (`processes_2026_06_10.*`, по 30 на бренд), и они закрыли большинство процессных вопросов.

- **Полное покрытие** (оба бренда) — ~30 из 43 кластеров, включая почти весь высокочастотный спрос:
  лагерь, расписание, преподаватели, адрес, договор, маткапитал, справка-вычет, пробное, контакты, тест,
  способы оплаты, личный кабинет, записи, перенос, пропуски, сертификат, лист ожидания.
- **Частичное** — ~11 кластеров (ответ собирается, но без одного прицельного факта или с дисклеймером).
- **НЕТ (жёсткая дыра)** — всего 2: `F_deadline` (срок записи) у Фотона; `P_foreign` (оплата/обучение
  из-за рубежа) у обоих. Обе — низкочастотные.

Вывод: настоящих «пустот» мало. Бóльшая ценность — в **расхождениях сайт↔база** (Этап В) и в **бренд-
асимметриях** (у УНПК тоньше ряд процессных фактов; у Фотона нет некоторых фактов лагеря/предметов/прувов).
