## Формат файла (как в существующих наборах)
Одна jsonl: строка `simulator_spec` (как играть клиента — переиспользовать из tripwire_round5_8) +
строка `judge_spec` (как оценивать; включает `confirmed_facts_snapshot` по брендам) + N строк `persona`.

Схема persona (точная, по существующему набору):
`type=persona`, `dialog_id`, `brand` (foton|unpk), `category`, `gold_topic_id`, `persona` (кратко),
`goal`, `held_facts`{grade,subject,format}, `behaviors`[реплики клиента], `expected_route`
(bot_answer_self|draft_for_manager|manager_only), `success_criteria`, `fail_criteria`, `brand_forbidden`[],
`max_turns`, `mood`, `style`, `adversarial`, `crm_test`, `injected_p0`.

## Параметризация → 100-300 диалогов
Сетка: brand(2) × archetype(22) × вариации. Вариации на персону: класс {7,9,11 базово, + выборочно 5/6/8/10},
предмет {математика, физика, информатика}, формат {онлайн, очно}, открытие {одиночное | составное ~40%},
настроение {нейтральное | торопливое | сомневающееся}. Балансировка, чтобы покрыть:
- ~55-60% enrollment-архетипы, ~20-25% summer (лагерь/выездные), ~20% evergreen — как веса rev3;
- ~40% диалогов открываются составным вопросом (C-логика, реальные 40,6%);
- классы 9/11/10 чаще (пики), но 5-8 представлены;
- по каждому бренду — полный список архетипов.
Объём: целимся **~150-200** (внутри 100-300). Точное число — балансировкой сетки, Кодекс зафиксирует
факт. размеры по (бренд × фаза × архетип) в run_meta.

## Маппинг полей (правила генерации для Кодекса)
- `held_facts` = выбранные класс/предмет/формат вариации.
- `behaviors` = дословные цитаты архетипа (rev3), адаптированные под held_facts (подставить класс/предмет,
  не меняя смысл). Для составных (C) — первый ход объединяет 2+ темы.
- `expected_route`: E*/C*/V2-6/S* безопасные → bot_answer_self или draft_for_manager (по наличию факта);
  V1 гипотетика → bot_answer_self (успокоить из факта); P0a/P0b/реальная претензия → manager_only,
  injected_p0=true.
- `success_criteria` / `fail_criteria` = «золотой путь» архетипа (rev3) + бренд-специфика (напр. Фотон E5
  МОЖЕТ предложить рассрочку; УНПК E5 — нет, помесячно/семестр/год).
- `brand_forbidden` = маркеры ДРУГОГО бренда (списки выше).
- `gold_topic_id` = тема архетипа; `category` = enrollment|summer|evergreen|safety.

## Бренд-разделение (инвариант)
Каждый диалог — РОВНО один бренд (канал). brand_forbidden обязателен у каждой персоны. Утечка другого
бренда = hard-gate fail у судьи. Никогда не смешивать в одном диалоге.
