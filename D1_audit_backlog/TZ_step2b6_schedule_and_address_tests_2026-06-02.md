## Тест выхода
1. **Офлайн-регрессия** на 58 диалогах Прогона 1 + проверка, что переключение на v6.4 не сломало другие темы
   (дифф по немигрированным = 0; помним, фолбэк после 2b.6 пуст — значит все правила теперь через rules_engine).
2. **НЕГАТИВ-тесты (обязательны):**
   - «в какие дни 7 класс физика?» при наличии только контакт-часов → НЕ «Пн-Вс 10-18», менеджер сверит;
   - точные дни конкретной группы не подтверждены → «менеджер сверит класс/предмет/площадку», не выдумать;
   - «бывают по выходным?» → мягкий ориентир, не подтверждённый слот;
   - «когда старт?» + факт → 12-27 сентября по площадке (из v6.4-факта);
   - бренд: группы Фотон ≠ УНПК (раздельно);
   - ПОЗИТИВ: «сколько раз в неделю?» + факт → 1 раз/неделю, answer_self;
   - адрес в расписании = Верхняя Красносельская 30 (не Скорняжный) — согласован с contact_address.
3. **Полный pytest** зелёный (вкл. починенные 2 адресных теста; кроме ~9 инфра). LLM здесь не зовётся — 0 вызовов.
4. **Один точечный прогон — ровно 8 диалогов** (сокращён ради скорости), `--parallel 4`. Набор покрывает
   ключевое по schedule (по 1 диалогу, бренды чередовать Фотон/УНПК):
   1) контакт-часы ≠ дни: «в какие дни 7 класс физика?» при наличии только контакт-часов → НЕ «Пн-Вс 10-18»;
   2) неопубликованная группа: точные дни группы не подтверждены → «менеджер сверит класс/предмет/площадку»;
   3) выходные: «бывают по выходным?» → мягкий ориентир;
   4) старт: «когда начинаются занятия?» → 12-27 сентября по площадке (из факта);
   5) кадэнс (позитив): «сколько раз в неделю?» → 1 раз/неделю, answer_self;
   6) дни группы из факта (позитив): «когда математика 11 класс очно?» → суббота/вс из v6.4-факта;
   7) бренд-сплит: УНПК-вопрос про расписание не подмешивает группы Фотона;
   8) адрес в расписании = Верхняя Красносельская 30 (не Скорняжный).
   Этого хватает на безопасность+ключевое; полный набор не нужен. Ориентир нагрузки: ~8 диал. × ≤3 хода ≈
   80-110 LLM-вызовов.
