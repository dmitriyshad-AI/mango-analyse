# TZ-116 v2: Группа 4, офлайн-режимы понимания

Дата: 2026-06-15

## Рамка

Делаем только офлайн-инструменты и замеры. Каждый новый контур работает за явным режимом `off|shadow|primary`, где `off` является значением по умолчанию и сохраняет старое поведение.

`primary` не включается в рабочих прогонах без отдельного решения. В AMO, Tallanto, CRM и файлы живого бота ничего не пишем. Реальные персональные данные в облачные модели не отправляем.

Живой путь бота не трогать:

- `src/mango_mvp/channels/subscription_llm_parts/*`
- `src/mango_mvp/insights/sanitizers.py`
- `src/mango_mvp/insights/tone_score.py`
- `src/mango_mvp/insights/phase2_detectors.py`

## Порядок

1. A: CRM `llm_shadow` офлайн-замер на фиксированном наборе, без кэша и без writeback. Отчёт расхождений LLM ↔ эвристика.
2. C: офлайн-замер каталога вопросов. Основной каталог не пересобирать.
3. B: `outcome_linker` режим `off/shadow` и учёт отрицания в shadow-контуре. Default `off`.
4. E: `infer_brand` только за режимом `legacy|cyrillic_v2`, default `legacy`.
5. D: STOP для реальных звонков. Только оценочный скрипт и synthetic/gold. Реальные звонки в облако не отправлять до отдельного разрешения.

## Приёмка

- Фича-ветка от `main`.
- Default `off` не меняет старое поведение.
- Нет live write.
- Нет ASR / Resolve+Analyze.
- Нет отправки ПДн в облако.
- Полный `pytest` зелёный.
- Отчёт в `tasks/_done/`.
