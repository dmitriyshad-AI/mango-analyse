# Risk review

## Основной риск

Главный риск старого поведения: `dialogue_memory.py` мог назначить `child_2` только по словам "дочка", "девочка", "младший", хотя профиль клиента уже мог знать правильный `child_key`.

## Как снижено

- Флаг `TELEGRAM_CHILD_IDENTITY_MODEL` по умолчанию выключен.
- При включенном флаге используется только готовый `current_child_key` / `resolved_children`, уже подготовленный вне живого пути.
- В живой путь не добавлен вызов LLM, stoplist, cache или trace резолвера.
- Если готового ключа нет, поведение остаётся прежним.
- Если в одном сообщении два ребёнка, старое разделение сохраняется, чтобы не схлопнуть реальные разные детские слоты.

## Остаточный риск

- Качество зависит от того, насколько корректно внешний слой профиля передаст `current_child_key`.
- При нескольких `resolved_children` без явного `current=true` память не выбирает ребёнка сама и возвращается к старому правилу.
