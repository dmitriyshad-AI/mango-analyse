# Golden-path рецепты — метод, сезонность, инварианты (C2)

Дата: 2026-05-26. Автор: второй Cowork-диалог (анализ звонков → рецепты). Зона вывода — только эта папка.
Статус: **v0, первый срез.** Полный корпус звонков ещё не подключён — летний/лагерный профиль добираю доменно и из v6.3-фактов, частоту лета уточню по звонкам.

## Что это
Рецепт «golden-path» = для частого намерения клиента заранее заданы: какие fact_keys из базы нужны, какие точные значения (раздельно по брендам), каркас черновика для менеджера, защита бренд-разделения и правила хендоффа. Цель (идея C2): крепкий консистентный ответ на ~80% частых вопросов, меньше over-handoff — **без потери безопасности**.

## Жёсткие инварианты (рецепт без них недействителен)
- **Бренды раздельно.** Фотон и УНПК — разные организации. В одном ответе никогда не смешивать и не сравнивать. `active_brand` задаётся каналом; рецепт даёт отдельный блок на каждый бренд.
- **Только подтверждённые факты.** Все числа/даты в рецептах взяты из `product_data/knowledge_base/kb_release_20260520_v6_3_team_answers/facts_registry.jsonl` (поле `client_safe_text`, `allowed_for_client_answer=true`). Каждое значение помечено своим `fact_key`. Ничего «по памяти».
- **Без выдумок и склейки.** Нельзя соединять два факта в одно утверждение и нельзя называть сущность (платформа/продукт/цифра), которой нет в фактах.
- **Режим черновика.** Бот не отправляет клиенту сам — готовит черновик + риски, менеджер одобряет.
- **P0 → менеджеру всегда:** возврат, жалоба, спор по оплате, гарантия результата. Рецепт P0 не «отвечает», а корректно передаёт.
- **Числа сверять грепом.** Судья симулятора врёт по «выдумке» (неполный снимок). Любую метрику регрейдить grep'ом по `client_safe_text` нужного бренда.

## Профиль спроса
**Осень УНПК (сен-окт 2025, 965 сообщений; данные Claude #1):**
подбор класса/предмета 19% · формат онлайн/очно 15% · цена 10% · ЕГЭ/ОГЭ/олимпиады 7% · запись/места 5% · доступ/платформа/логин 4% · расписание 3% · скидки/лагерь/договор/преподаватели ~2%.
Это профиль старта учебного года. Лето (лагеря/ЛВШ) в этом экспорте почти отсутствует.

**Доменная сезонность (для прогноза июнь-дек 2026):**
- **Лето (июнь-авг):** лагеря / ЛВШ / городские летние школы + ранняя запись на новый год + цены/формат 2026/27.
- **Авг-сен:** запись, расписание, цены, подбор класса/предмета.
- **Окт:** старт занятий, логистика, доступ/платформа.
- **Ноя-дек:** налоговый вычет за обучение.

## Приоритет рецептов (этот v0)

## Как читать рецепт
Каждый рецепт: **намерение и триггеры → нужные fact_keys → проверенные значения (отдельно Фотон / отдельно УНПК) → каркас черновика → защита бренд-разделения → P0/хендофф → калибровка over-handoff.** Значения — в `RECIPES_v0_2026-05-26.md`.
