# Приёмочный референс Волны 2 (Claude-сторона). 2026-05-26

Подготовлено заранее, пока Codex аудитит/внедряет Волну 2. Эталоны/ожидаемые вердикты по классу
«выдумка-склейка», по которым я сверю верификатор+семантику Codex. Инструмент: мой независимый
`wave2_gluing_regression_golden.py` (рабочая папка Claude).

## Ключевой вывод (важен для приёмки)
Склейка — РАЗНЫЕ подклассы, ловятся РАЗНЫМИ барьерами. Один барьер недостаточен:
- **A — именованная сущность ВНЕ фактов** (Zoom, выдуманное ФИО) → ловит детерминированный **V7**.
- **B — сущность ИЗ фактов, приписана НЕ ТОМУ объекту** («кабинет на МТС Линк») → V7 НЕ ловит (сущность
  есть в фактах!), ловит только **семантика V1/V2**.
- **C — число из факта A в контексте B** (цена в роли срока рассрочки; частота интенсива на регулярный
  курс) → ловит **семантика V1/V2**.
Значит V7 и семантика ОБА обязательны — это проверяю при приёмке.

## Результат моей независимой проверки V7
Прототип V7 (детерминированный): **5/5** на решающих кейсах — флагает A1/A2, чист на NEG1/2/3, и
корректно НЕ флагает B/C (там сущность в фактах / это число-в-контексте → дело семантики).

## Критерий приёмки Волны 2 (как сверю)
1. V7 Codex: на A1/A2 → finding; на NEG1/2/3 и B/C → НЕ флагает (whitelist работает, over-handoff не растёт).
2. Семантика Codex (на живом прогоне): на B1/B2/C1/C2 → `unsupported`/склейка; на NEG → чисто.
3. Каждый подкласс (A/B/C) закрыт хотя бы одним барьером, 0 пропусков известных склеек.
4. Тёплый слой не порождает склейку на этом наборе; доля отклонений тёплого ниже прежней.
5. Безопасность/fail-closed/P0/бренды держатся; тон не просел от T1.
