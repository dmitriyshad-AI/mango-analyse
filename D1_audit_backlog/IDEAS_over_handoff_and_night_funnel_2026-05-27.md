# Идеи: починка over_handoff + вечерняя рекламная воронка. 2026-05-27

Автор: Claude + 6 субагентов (60 идей) + мои 10. Здесь — дедуплённый список уникальных полезных идей
(70 сырых → ~45 уникальных). Приоритет: сократить over_handoff и хорошо обслуживать новых вечерних
лидов с Яндекс Директ/РСЯ (cdpofoton.ru → бот Фотона, kmipt.ru → бот УНПК), пока менеджера нет онлайн.

Ниже формат каждой идеи: **Что делать → Что изменится → Что даёт.** Ранжирование — в конце файла.

## R. Ядро over_handoff: развилка «ответить сам vs менеджер»

**R1. Перевернуть логику по умолчанию (ключевая).** Эскалировать ТОЛЬКО при (а) явном настоящем P0
из закрытого списка или (б) отсутствии факта в базе. Во всех остальных случаях — отвечать самому. →
Бот перестаёт уходить «на всякий случай». → Снимает основной класс ложной эскалации. [A1,A9,F4,G1]

**R2. P0-проверка не должна короткозамыкать до поиска ответа (код).** Сейчас ветка P0 срабатывает
ДО извлечения фактов и augment'а возврата (pipeline стр. ~906 раньше 930). Сделать так, чтобы сначала
проверялось «есть ли безопасный ответ», и лишь потом — реальная ли это претензия. → На безобидных
уточнениях бот не сваливается в сухой хендофф. → Чинит корень бага возврата фазы 2. [G1]

**R3. Метка «гипотетика vs реальная претензия» в памяти нити.** При первом ответе на возврат/конфликт
ставить флаг hypothetical; снимать только при явном сигнале претензии/недовольства. → Нить остаётся
в режиме «объясняю», не откатывается. → Прямо закрывает наблюдаемый дефект диалога 02. [A2]

**R4. Состояния нити вместо переклассификации каждый ход.** Нить = конечный автомат (exploring /
explaining_return / p0_escalated); в P0 уходим только по явному триггеру. → Стабильное поведение в
длинном диалоге. → Убирает «залипание на хендофф». [A6]

**R5. Разбор составного вопроса по частям.** Каждую смысловую часть классифицировать отдельно;
безопасные — ответить, P0-часть — пометить «по этому подключу менеджера». → «Сколько стоит + можно ли
вернуть?» даёт цену сразу. → Лид не теряется в ожидании. [A3,E3]

**R6. Правило «ответь на безопасную часть + один шаг», градуированный хендофф.** Если есть хоть одна
безопасная часть — ответить и дать один следующий шаг; менеджер только если ВСЯ нить P0 или факта нет
даже после уточнения. → Бот не бросает лида с пустыми руками. → Удержание в нерабочее время. [A4,C7]

**R7. Мягкий хендофф вместо сухого «передам менеджеру».** Даже в реальном P0 — сначала содержательно
(что понял / что можно), потом про передачу; запретить «передам менеджеру» без предшествующего
полезного. → Хендофф звучит как забота. → Лид не чувствует, что его выбросили. [A7,C8,F1]

**R8. Уровни уверенности ответа.** Факт «подтверждён» → отвечать полностью; «частично» → с оговоркой;
«нет факта» → мягкий хендофф. → Бот не отказывает там, где есть частичный факт. → Меньше over_handoff
на «серых» вопросах без выдумки. [F2]

**R9. Эвристика «ответил бы дежурный менеджер за 30 секунд?».** Добавить в понимание ориентир,
близкий к реальной практике. → Понятный критерий «отвечать/нет». → Меньше эскалаций на очевидном
(цена, формат, расписание-ориентир). [A8]

## N. Вечерняя/ночная рекламная воронка

**N1. Зеркало рекламного объявления в приветствии (UTM-интент).** Передавать с лендинга предмет/класс/
формат в запуск бота; первое сообщение повторяет интент. → Вместо «чем помочь?» — «вы смотрели ЕГЭ по
физике, 11 класс — расскажу». → Воронка стартует там, где кончилась реклама. [B1,C5]

**N2. Захват лида по одному вопросу за раз.** Класс → предмет → формат → имя → контакт, каждый
отдельным сообщением, без «анкеты из 5 пунктов». → Не отпугивает ночного пользователя. → Выше шанс
дойти до контакта. [B2]

**N3. Карточка/досье лида менеджеру на утро.** Структурно: имя, класс, предмет, формат, вопросы,
настроение/возражения, метка «ночной холодный» → в очередь менеджеру (read-only). → Менеджер начинает
с тёплого контекста. → Ни один ночной лид не теряется. [B3,B9]

**N4. Конкретное время обратной связи.** Не «свяжемся в рабочее время», а «позвоним до 11:00 [дата]»,
зафиксировать в карточке. → Лид знает, когда ждать. → Меньше тревоги, не уходит к конкуренту до утра. [B4]

**N5. Мягкая бронь места без оплаты.** «Зафиксируем интерес, менеджер сохранит место до [дата+2]». →
У менеджера утром есть повод для звонка. → Холодный лид → тёплый с конкретным поводом. [B7]

**N6. Честная срочность из факта базы.** Если до старта/смены мало дней или мало мест (подтверждённый
факт) — упомянуть один раз, без нагнетания. → Срочность только настоящая. → Быстрее конверсия, доверие
не страдает. [B10,C6]

**N7. Режим, знающий часы работы (time-aware).** Рабочие часы бренда — факт; днём «менеджер на связи»,
вечером «отвечу сам, менеджер утром». → Поведение явно зависит от времени. → Уместный тон и ожидания. [G4]

**N8. Мягкая квалификация лида.** Проверить, наш ли профиль (есть ли класс/предмет/формат); если нет —
честно сориентировать, не тянуть. → Отсев нецелевых с рекламы. → Экономия бюджета и времени менеджера. [G6]

**N9. Дожим незавершённого лида (re-engagement).** Если ночью лид не закончил захват — мягкое утреннее
напоминание (с согласия). → Бот возвращает «подвисший» интерес. → Меньше слитых рекламных кликов. [G8]

## T. Тон и продающий разговор

**T1. Примеры лучших менеджеров в промпте (few-shot тёплости).** 5-7 живых образцов речи из плейбука
звонков (снятие риска, гашение тревоги, чёткий шаг). → Бот говорит образцами, не «инструкцией». →
Тон ближе к человеческому без переделки логики. [C1]

**T2. Один мягкий следующий шаг + каталог шага по стадии воронки.** Финал ответа — конкретное действие,
релевантное стадии (узнал цену → бронь; формат → пробное), не случайное и не «передам менеджеру». →
Каждый ответ ведёт вперёд. → Бот ведёт к записи, а не отчитывается. [C2,G7]

**T3. Запрет повтора фразы (буквального и смыслового).** Хранить последние 2-3 финальные фразы; если
повтор по форме или смыслу — заменить/переформулировать. → Уходит эффект «заезженной пластинки»
(диалог 6!). → Диалог живой, лид не видит копипасту. [C4,E4]

**T4. Микроподтверждение сути в начале ответа.** Первая фраза отражает запрос: «Понял — онлайн для 9
класса». → Убирает ощущение, что бот не слушал. → Теплее, выше продолжение диалога. [C9,E8]

**T5. Детектор тревожного вопроса → сначала спокойствие, потом факт.** Распознавать «а вдруг не
подойдёт / можно вернуть / а если не сдаст» и отвечать по шаблону снятия риска. → Вместо сухого «не
знаю» — короткое успокоение + факт. → Тревожный лид не уходит. [C3]

**T6. Отвечать на ТОМ уровне конкретики, что дал клиент (НЕ диапазоном).** Делать: если клиент назвал
класс (напр. 7), а цена в базе задана на вилку (5–11) — подавать ответ ДЛЯ ЕГО класса: «для 7 класса:
…», не вываливать вилку. Без выдумки: класс должен попадать в диапазон-факт; не попадает — не
утверждать. Чинить в черновике/понимании (тёплый слой факты не меняет). → Бот не заставляет клиента
проверять «а 7-й сюда входит?». → Ответ ощущается адресным, а не «справкой по прайсу». [найдено
Дмитрием на диалоге 01]

**T7. Запрет дублирующего пересказа (одна мысль — один раз).** Делать: в черновике/понимании запретить
повторять уже сказанное другими словами в том же ответе («…29 750 ₽… / Это цена для онлайн 5–11…»). →
Уходит робо-наполнитель. → Ответ короче и живее. [усиление C4/T3 на уровень одного ответа]

**Судья (предусловие, иначе мы слепы):** заточить рубрику тона, чтобы она ШТРАФОВАЛА пересказ-дубль и
ответ-диапазоном-вместо-конкретного. Сейчас судья даёт таким ответам tone 2/2 — высокий балл тона у
судьи НЕ равен живой речи.

## S. Безопасность, включающая автономность + измерение + выкатка

**S1. Реестр «авто-безопасных тем» — единый источник для бота, судьи и тестов.** Список тем, где факт
есть и риска P0 нет; хендофф по ним запрещён, если факт найден. → Системная защита от перестраховки и
от «whack-a-mole». [F4,A1]
**S2. Метрика over_handoff ПО ХОДАМ, не по диалогам.** → Операционно честный замер, видно конкретный
шаг перестраховки. [F3]
**S3. Мерить over_handoff и захват лида на РЕАЛЬНОМ наборе вопросов рекламного лида, не на злом
tripwire.** Собрать топ-вопросы новых вечерних лидов (Telegram-экспорт/звонки). → Метрика
репрезентативна целевому трафику. → Не оптимизируем под нерелевантный злой набор. [G2]
**S4. Бизнес-KPI воронки: доля лидов, дошедших до захвата контакта/брони — рядом с тоном.** → Меряем
то, что важно для рекламы, а не только «человечность». [G5]
**S5. Авторегрейд ложных хендоффов: после прогона помечать ходы, где ушёл к менеджеру, а факт в базе
есть.** → Конкретный список случаев на правку. [A5,A10,F5]
**S6. Поэтапная выкатка ночного режима волнами (один источник → N диалогов/сутки → полный трафик).** [F6]
**S7. Ночной стоп-кран: при всплеске P0/аномалии — режим «только менеджер» до утра + алерт.** [B6,F7]
**S8. Анти-провокация/джейлбрейк фильтром ДО модели («ты бот», «притворись», «другой бренд»).** [F8]
**S9. Двойной предохранитель автономной отправки ночью + аудит-лог + петля обучения.** Перед прямой
отправкой клиенту — финальная детерминированная проверка (бренд/числа/P0/выдумка), иначе мягкий
черновик; все ночные ответы логируются, размечаются и кормятся в few-shot/тесты. [F9,G9,G10]
