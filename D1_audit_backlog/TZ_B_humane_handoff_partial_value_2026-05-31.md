## Что в коде (прочитано)

## Правки

### Правка 1 — тёплые вариативные пулы вместо 3 роботов (НЕ-P0)
Заменить одиночные строки на пул из 4-6 тёплых формулировок + выбор с анти-повтором (см. Правка 3).
Тон: не «точно ответить не могу» (холодно, отрицание), а «чтобы не ошибиться — уточнит менеджер».
Примеры пула (Дмитрий утвердит формулировки):
- «Чтобы ответить точно, эту деталь сверит менеджер — передам ему именно ваш вопрос.»
- «Тут не хочу гадать: менеджер подтвердит точно и свяжется с вами.»
- «Чтобы не ввести в заблуждение, этот пункт уточнит менеджер — он вернётся с ответом.»
Заменяемые точки: `question_detail` (2191), `generic` (2194), `SAFE_FALLBACK_DRAFT_TEXT` (subscription:77).
Где есть `{detail}` — сохранить вставку темы («…уточнит менеджер именно про {detail}»).

### Правка 2 — ВСЕГДА частичная ценность перед уходом (НЕ-P0)
Сейчас частичное даёт только `secondary_fact`. Расширить: перед роботами `question_detail`/`generic`
попытаться добавить ЛЮБОЙ известный client_safe ориентир по теме, если он есть в `facts`. То есть, если
после Решения A полный ответ не собрался, но в фактах есть релевантный client_safe фрагмент — вынести
его как «Из подтверждённого: …» + честный уход по недостающей детали. Если ориентира нет — тёплый уход
из Правки 1. Использовать существующий `_secondary_fact_text`/`_coverage_findings` как источник, не
плодить новый. Бьёт по «уход без всякой пользы».

### Правка 3 — анти-повтор: НИКОГДА одна фраза ухода дважды в диалоге
В `_avoid_repeating_text` (2237): при детекте повтора — НЕ подставлять одну робот-фразу, а:
1) выбрать из тёплого пула вариант, ещё НЕ встречавшийся в этом диалоге;
2) если все варианты исчерпаны — эскалировать иначе: «Вижу, это важно — отдельно отмечу менеджеру, чтобы
   он ответил именно по этому пункту» (один раз), не повторяя дословно.
Расширить охват анти-повтора: проверять near-repeat по ВСЕМ выходам ухода (включая `SAFE_FALLBACK_DRAFT_TEXT`
из subscription_llm), а не только по тем, что проходят через `_avoid_repeating_text`. Подтвердить чтением,
все ли пути ухода прогоняются через `_avoid_repeating_text` (в run_pipeline возвраты обёрнуты в него —
проверить, что и боевой путь subscription_llm тоже).

### Правка 4 — P0/жалоба: форму НЕ менять, добавить анти-повтор
Ротацию эмпатичных фраз (pipeline 80-83, subscription 80-95) оставить по смыслу (без темы/данных), но
прогнать через тот же анти-повтор, чтобы «Приняли обращение…» не повторялось дословно 3 раза. НЕ
добавлять частичную ценность по теме в P0/жалобу/возврат (политика 5.2 — держать).

## Тесты + НЕГАТИВНЫЙ контроль

ПОЗИТИВ:
- «где вы?» (после A не закрылось бы — но если факт-адреса нет) → тёплый уход, НЕ «Спасибо за сообщение»;
  на повтор вопроса — ДРУГАЯ формулировка, не дословный дубль.
- уход при наличии частичного client_safe ориентира → «Из подтверждённого: … ; деталь уточнит менеджер».
- 3 хода подряд с уходом по одной теме → 3 РАЗНЫЕ формулировки (анти-повтор).

НЕГАТИВ (критично):
- жалоба → эмпатичный хендофф БЕЗ темы и данных (политика 5.2 не ослаблена); меняется только вариант
  фразы, не суть.
- возврат-претензия → менеджер, без частичной «ценности» по сумме/решению (не обещать).
- тёплые фразы НЕ содержат мета/обещаний/самораскрытия (прогнать через `verify_output`).
- route не меняется (B — только текст ухода, не маршрут); бренд/мета остаются 0.
