# ТЗ — Уборка инфраструктуры (один заход, после калибровки-2)

**Статус:** готово к исполнению по команде Дмитрия (после коммита калибровки-2)
**Исполнитель:** главный Codex. **Принцип:** ничего не удалять до архивации; каждый разрушительный шаг — после инвентаря; стоп при любой неожиданности.
**Цель:** одна каноничная линия кода в ГЛАВНОЙ папке, ноль worktree-зоопарка, M1 на чистом развёртывании, файловая шина задач. Ничего не потерять.

## Фаза 0 — Инвентарь и безопасность (read-only, отчёт Дмитрию ДО фазы 1)

1. **Живые процессы — первая проверка** (прошлый стоп был именно тут): `ps aux | grep -E "run_telegram|polling|mango"` — любой живой процесс из любого дерева → СТОП, спросить Дмитрия (известный прецедент: `run_telegram_public_pilot_bots.py --mode poll`).
2. Инвентарь деревьев: `git worktree list`; для каждой ветки — `git log --oneline codex/consolidation-clean-type-20260605..<branch>` (уникальные коммиты, не влитые в канон). Список «есть ли что терять».
3. Инвентарь ценного untracked ПО КАЖДОМУ дереву: `runs/`, `*.jsonl` наборы, `audits/_inbox/*`, временные доки. Список с размерами.
4. Страховка истории: `git bundle create ~/mango_all_refs_$(date +%Y%m%d).bundle --all` (вся история в один файл, положить в Яндекс).

## Фаза 1 — Канон в главную папку

5. Убедиться: каноничная ветка `codex/consolidation-clean-type-20260605` содержит ВСЁ (последний коммит = калибровка-2); tracked-дерево worktree чистое.
6. Архивировать ценный untracked из главной папки и worktrees: `runs/`, наборы, audit packs → `~/Yandex.Disk.localized/OpenClaw/Actual Mango Tests/_cleanup_archive_20260607/<имя_дерева>/`.
7. Главная папка: если tracked-дерево грязное — закоммитить как есть в её ветку (`codex/d1-humanity-layer-20260525`, коммит «pre-cleanup snapshot»). Ветку НЕ удалять (история).
8. Переключить главную папку на канон: `git checkout codex/consolidation-clean-type-20260605`. Untracked-доки (D1_audit_backlog и пр.) при checkout сохраняются — проверить после.
9. Сразу после переключения: заменить CLAUDE.md новой редакцией (`D1_audit_backlog/CLAUDE_md_proposed_2026-06-06.md`, без секции-шапки) — если Дмитрий дал ок; закоммитить.
10. Проверки: `pytest tests/` полный зелёный в главной папке; `ls D1_audit_backlog | wc -l` (доки на месте); семантический гейт прогнать на актуальном релизе.

## Фаза 2 — Снос worktree-зоопарка

11. Для каждого из ~11 worktrees: (а) уникальные коммиты из п.2 — если есть невлитое ценное, доложить Дмитрию ДО удаления; (б) ценный untracked уже в архиве (п.6); (в) `git worktree remove <путь>` (с `--force` только если п.(а)/(б) подтверждены).
12. `git worktree prune`; `git worktree repair`; итог: `git worktree list` = только главная папка.
13. Ветки НЕ удалять в этот заход (история бесплатна; отдельным решением позже).
14. Новый порядок работы: Кодекс работает в главной папке на feature-ветках; worktree — только при реальной параллельной работе, с удалением после мержа.

## Фаза 3 — M1 на чистое развёртывание

15. Собрать ПОЛНЫЙ свежий бандл канона (не overlay): `src/ scripts/ tests/ product_data/knowledge_base/<v6.5> + наборы` → Яндекс `mango_clean_<HEAD>/` (overlay-дрейф M1 закончился: там сейчас a0448f33 + 3 слоя заплаток).
16. M1: переразвернуть из чистого бандла (старое — в `_m1_old_$(date)/`, не удалять), smoke-проверка: `--bot-mode fake` прогон одного сценария за секунды.

## Фаза 4 — Файловая шина задач

17. Создать: `tasks/_inbox_codex/` (+ `_running/`, `_done/`) в главной папке; `OpenClaw/Actual Mango Tests/tasks/_inbox_m1/` (+ `_running/`, `_done/`, `status.md`) в Яндексе.
18. В AGENTS.md Кодекса добавить: «В начале сессии и по слову "inbox" — читай tasks/_inbox_codex, бери верхнюю задачу, перенеси её файл в _running, по завершении — в _done с отчётом».
19. Watcher-демон M1 — НЕ в этот заход (отдельное ТЗ после уборки).

## Фаза 5 — Выходной отчёт

- Что заархивировано и куда (список), что удалено, итоговая топология (`git worktree list`, ветки, HEAD главной).
- pytest зелёный; доки на месте; M1 smoke прошёл; шина создана.
- Любая неожиданность по пути (невлитые коммиты, живые процессы, конфликты) = СТОП + вопрос, не «решить молча».

## Чего НЕ делать

`git reset --hard` / `git clean -fd` где есть неархивированное; удаление веток; удаление чего-либо в Яндексе; трогать stable_runtime; запускать live-ботов. Прогоны во время уборки не гонять (замороженная среда).
