## КОММИТ 1 — leak-гейт класса лесов/инструкций (ловит навсегда) + источник readiness

### 1a. Расширить `meta_leak`/`sanitize` паттернами класса
В детектор внутренних маркеров (там же, где `source_id`/`fact_id` regex и `strip_internal_service_markers`) добавить:
- **Префикс-леса (СТРИП префикса, оставить текст после):** `черновик для ситуации «...»:` → отдать клиенту только
  текст ПОСЛЕ двоеточия (он client-safe, напр. «Это отдельные организации…»). Аналогично любые `... для ситуации «...»:`.
- **Эхо директив промпта (СТРИП префикса):** строки вида `без обещаний ...:`, `без давления ...:` как ПРЕФИКС перед
  фактом → срезать префикс, оставить факт.
- **Императив-инструкции в клиентском тексте (БЛОК → фолбэк на безопасный шаблон):** фразы, адресованные боту, а не
  клиенту: «...повторять (их) не нужно», «не упоминай ...», «если ... уже есть в диалоге ...». Это компрометирует
  весь ответ → фолбэк (короткий честный шаг), не показывать клиенту.
Подход консервативный: префиксы-леса — стрип (текст после чистый); инструкции-предложения — блок/фолбэк.

### 1b. Источник readiness (убрать инструкцию из шаблона)
В rules_engine readiness-фолбэк (из eaecefca) содержит «Если класс, предмет и формат уже есть в диалоге, повторять
их не нужно» — это инструкция, попавшая в client-текст. Убрать из шаблона (оставить только клиентскую часть
«Менеджер подтвердит порядок записи…»).

NEG коммита 1: «черновик для ситуации «X»: ТЕКСТ» → клиенту только «ТЕКСТ»; readiness НЕ выводит «Если
класс/предмет/формат…/повторять не нужно»; «без обещаний оценки: ФАКТ» → клиенту «ФАКТ»; императив-инструкция в
ответе → фолбэк, не клиенту. И контроль: нормальный клиентский текст со словом «если/повторите» НЕ режется ложно
(точные якоря, не широкий regex).

## КОММИТ 2 — тёплый информативный хендофф (триал/запись), не холодный
По регрейду cfg3: «есть пробное?» и «как записаться?» уходят сухим «передам менеджеру». По политике триал/детали
записи организует менеджер — но ХЕНДОФФ должен быть тёплым и информативным:
- **триал** («есть пробное?»): «Да, пробное есть — менеджер подберёт доступный вариант и запишет.» (route может
  быть manager, но текст информативный, не сухой; пробное только при подтверждённом факте бренда).
- **enrollment** («как записаться/что нужно»): дать ШАГИ из enrollment-факта, что есть (запись дистанционно,
  договор), + тёплый менеджер по специфике; не голое «передам менеджеру».
NEG: триал-хендофф содержит «пробное есть … менеджер подберёт/запишет» (не сухое); enrollment-ответ содержит шаги
из факта; без давления; бренд один; P0 не затронут.
