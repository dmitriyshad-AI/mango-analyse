# Phase 12 · TZ-08 (НОВЫЙ) — `offline_grade_scope` для олимпиады (закрывает E7_olymp_04)

Снимок: HEAD `36e23cb8`. Приоритет диспетчера: 51. **Зависит от Phase 2 KB (Блок В, п.1 `applies_to.grades`) — без него бесполезен.**

## 1. Цель
Не подтверждать олимпиадную подготовку (Физтех и др.) для класса, которого нет в проверенном факте — в ЛЮБОМ формате (очно/онлайн). Существующий `_olympiad_online_safe_template` (TZ-05) покрывает только ОНЛАЙН; E7_olymp_04 — про ОЧНОЕ (6 класс), не покрыт. Закрывает hard_gate **E7_olymp_04** (бот подтвердил Физтех для 6 класса очно).

## 2. Текущее состояние в legacy
- Аналога для очной олимпиады нет; `_olympiad_online_safe_template` (`:2413`) — online-only.
- Retrieved-факт `olympiad_targets.client_safe_text` НЕ несёт ограничения по классам; ограничение «9 и 11» есть только в тексте `prices_regular_2026_27.online_olympiad_phystech_classes.client_safe_text` (онлайн-продукт) и в manager_only-note. То есть для очного контекста машинно-читаемого grade-scope НЕТ.
- `_wrong_scope_fact_findings` (`fact_claim_audit.py:180`) не проверяет «олимпиада vs класс персоны».

## 3. Точка вставки в v2
Диспетчер, priority 51 (scope класса раньше цены, рядом с olympiad_online). НОВЫЙ селектор `_offline_grade_scope_olympiad_template`. Срабатывает на формат-агностично (очно/онлайн), когда драфт/контракт касается олимпиады и `persona.grade` известен.

## 4. Зависимости (KB) — ОБЯЗАТЕЛЬНО Phase 2
- Нужно поле `applies_to.grades: [9, 11]` на олимпиадных Физтех-фактах (Блок В, п.1.4 + п.4). Без него селектор не знает допустимые классы.
- **Требует подтверждения Дмитрия:** ограничена ли ОЧНАЯ олимпиадная подготовка теми же 9/11, что и онлайн (в KB «9 и 11» зафиксировано только для онлайн-продукта). Если очная имеет иной набор — указать его. До подтверждения селектор работает в «безопасном» режиме: не утверждать наличие для класса вне подтверждённого набора, отдавать менеджеру.

## 5. Точная правка (псевдокод нового селектора)
```python
def _offline_grade_scope_olympiad_template(result, client_message, context):
    draft = result.draft_text or ""
    # 1) драфт касается олимпиады?
    if not OLYMPIAD_MENTION_RE.search(draft):   # «олимпиад|физтех|рсош|перечнев|росатом|курчатов|ломоносов|оммо»
        return ""
    grade = _persona_grade(context)             # из contract/known_slots/dialogue_memory
    if not grade:
        return ""                               # класс неизвестен — пусть обычный путь
    allowed = _olympiad_allowed_grades(context) # из applies_to.grades факта (Phase 2); по умолчанию {} → безопасно
    if allowed and int(grade) in allowed:
        return ""                               # класс допустим — не вмешиваемся
    # класс вне допустимого набора (или набор неизвестен) → безопасный шаблон
    return OFFLINE_GRADE_SCOPE_SAFE_TEXT        # см. ниже
OFFLINE_GRADE_SCOPE_SAFE_TEXT = (
    "Олимпиадная подготовка к Физтеху подтверждена для 9 и 11 классов. "
    "По вашему классу наличие отдельной олимпиадной программы уточнит менеджер — "
    "подтверждённой информации в боте нет."
)
_OFFLINE_GRADE_SCOPE_SPEC = TemplateSpec(
    name="offline_grade_scope", priority=51, produce=_offline_grade_scope_olympiad_template,
    route_on_apply="draft_for_manager", flag="offline_grade_scope_safe_template_applied",
    checklist="Олимпиада: не подтверждать класс вне applies_to.grades факта; по другим классам — менеджер.",
)
```
ПРИМЕЧАНИЕ по формулировке: текст содержит «9 и 11» как ПОДТВЕРЖДЁННЫЙ факт; если Дмитрий подтвердит иной набор для очного — заменить. Не утверждать «для 6 класса нет» (нет отрицательного факта) — формулировать «уточнит менеджер».

## 6. Регрессы
Positive:
1. `eval_unpk_E7_olymp_04 t2` (6 класс очно, «готовите к олимпиадам по физике?») → НЕ «Да… Физтех» без оговорки; OFFLINE_GRADE_SCOPE_SAFE_TEXT (9/11 подтверждено, по 6 классу — менеджер). Флаг поднят.
2. E7 t3 («это физтех, да?») → не подтверждать Физтех для 6 класса как доступный.
3. E7 t4 (прямой вопрос «физтех только для старших?») → честный ответ «9 и 11 подтверждены», НЕ подмена ценой (это смежный verifier-фикс).
Контрольные negative:
4. 9 или 11 класс + олимпиада → НЕ блокировать (класс в applies_to.grades).
5. Не-олимпиадный вопрос (обычная физика очно) → селектор НЕ срабатывает.
6. Класс неизвестен → не вмешиваться (обычный путь), не выдумывать.
7. Онлайн-олимпиада 9/11 → работает TZ-05, не конфликтует (priority 50 vs 51; один применяется).
