## Условие срабатывания (gate-in, всё детерминированно, на существующих сигналах)
Спрашиваем ТОЛЬКО когда ВСЁ:
1. маршрут был бы `draft_for_manager` (НЕ `manager_only`, не bot_answer_self).
2. НЕ P0 — источник P0 ЕДИНЫЙ: `p0_recall_spec`/`is_p0`/route `manager_only`/p0-латч. **НИКАКИХ новых регулярок P0** (запрет из ТЗ-26). Вопрос НИКОГДА не перебивает P0.
3. НЕ менеджерское действие: запрос данных счёта/карточки/договора/баланса, `send_crm_data`/`send_payment_link`/`send_document`/`capture_lead`/`advance_stage` → уход, не вопрос. Фича работает только на ветке `answer_only`-намерения.
4. не хватает БЫСТРОГО клиентского параметра из whitelist строго: `{класс/grade, предмет, формат, время}` — и точка (телефон/фамилия/договор НЕ спрашиваем).
5. **параметр реально разблокирует/меняет ответ** (по факт-паку: значения/числа расходятся по этому слоту — `_a2_qualifier_unlocks_fact`/логика `_scope_clarification_question` при >1 значении). Если факт одинаков для всех значений → НЕ спрашивать, отвечать.
6. у параметра НЕТ клиентского провенанса ЗА ВЕСЬ диалог (`do_not_reask_slots` + `client_confirmed_slots` из ТЗ-119) — не переспрашивать названное/эллипсис.
7. подвопрос+scope в client-safe зоне матрицы автономности (autonomy_requires_exact_scope).

## Один вопрос + анти-допрос/анти-цикл
- ОДИН вопрос, приоритет слота: класс > предмет > формат > время. Детерминированные шаблоны (не давать модели свободно сочинять — тон/мета-риск); бренд из канала, без смешения.
- Замыкание петли — автоматически: клиент назвал → провенанс confirmed → на следующем ходу бот отвечает из фактов сам (gate-in уже не найдёт недостающего слота). Отдельный «приём ответа» не городить.
- ЕДИНЫЙ счётчик вопросов (переиспользовать `_a2_already_asked`, не плодить второй): ≤1 уточнение за ход, бюджет ≤2 за диалог (начать с 1). Если клиент УКЛОНИЛСЯ (слот так и не появился) → мягкий уход к менеджеру, повторно не спрашивать.
- Развести с ТЗ-119: страж = «бот ВЫДУМАЛ параметр → переспроси»; эта фича = «параметра НЕ хватает, ушёл бы → спроси». Если оба триггерят — приоритет одному, второй no-op, текст не дублировать.
