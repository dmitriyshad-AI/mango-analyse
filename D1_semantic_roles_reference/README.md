## Состав пакета
Код (`reference/`):
- `semantic_roles.py` — распознаватель типизированных ролей (оси формат/оплата/источник/
  возврат/мультитема + омонимы запись/перевод), параметр `context` для follow-up.
- `decision_policy.py` — план ответа: мультитема, refund presale/dispute, запрет
  маткапитал×рассрочка, шаблон только как fallback.
- `held_state.py` — held-состояние диалога: append-only, класс C (поправка перезаписывает),
  P0-latch, контекст для распознавателя.

Тесты (`tests/`) — все зелёные, гоняются без модели:
- `test_semantic_roles.py` — 35 юнитов на омонимы/возврат/мультитему/запрет/шаблон.
- `test_multiturn.py` — 11 многоходовых (наследование перевода, класс C, latch, append-only).
- `triage_round5.py` — структурный триаж 24 персон round-5 (0 провалов).
- `sim_full_v8.py` — многоходовая симуляция всех 322 персон v8 (0 аномалий по инвариантам).
- `audit_static.py` — статические аудиты: бренд-утечка + числа-вне-снимка (0/0).
- `fixtures/word_trap_cases.jsonl` — банк слов-ловушек.

`logs/` — сохранённые прогоны.

## Что решает (5 корней round-4)
- омонимы по смыслу слова (очно/точной, запись/урок, перевод) — типизированные оси;
- предпродажный возврат vs спор — `refund_frame` вместо узких regex (чинит ложный P0
  на «передумаю до начала»);
- запрет «рассрочка при маткапитале» — способ и источник оплаты разведены на разные оси;
- мультитема — план возвращает все темы к ответу;
- шаблон только как fallback — флаг `template_allowed`.

## Граница честности
Распознаватель — по границам слов (сигналы), не семантическая модель. Парафразы без
ключевого слова добирает генерация (видит историю + факты). Слой ролей отвечает за
СТРУКТУРНУЮ развязку и правильную маршрутизацию, не за recall каждой формулировки.
