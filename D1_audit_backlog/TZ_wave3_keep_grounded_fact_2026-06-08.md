# ТЗ — Волна 3 «Перестать выбрасывать найденный факт» (пакет за флагами)

**Статус:** ФИНАЛ (аудит второго архитектора — 3 раунда внесены).
**Дата:** 2026-06-08. **Основание:** брейншторм 6 субагентов + находка по сырью: в 167/167 ложных уходов факт БЫЛ найден (`retrieved_match`), но срублен (~50% redact, ~40% promise-guard, ~59% verifier/fallback). Корень over-handoff 60% = «знание есть, но мы его выбрасываем», не «нет знаний».
**Дерево:** ветка `wave3-keep-grounded` от текущего main (после приёмки/Волн 1-2). В main НЕ мержить до замера+приёмки.
**Принцип:** один проход Кодекса, все правки за флагами **default OFF** (+ общий флаг-пакет `TELEGRAM_WAVE3` для пакетного включения). Все правки детерминированные, работают **над уже-grounded фактом**, жёсткие классы (цена/процент/даты/бренд/P0/ПДн/юр) НЕ трогают. Сначала read-only карта ВСЕХ точек, показать ДО кода (как Волны 1-2).

## Уже в коде — НЕ дублировать

## Правки (8), каждая — свой флаг default OFF

**В1. Partial-answer floor.** Флаг `TELEGRAM_PARTIAL_ANSWER_FLOOR`. Якорь: точки хендоффа `_force_draft_for_manager` / `empty_factual_answer_self` (dialogue_contract_pipeline). Инвариант: если на ходу есть ≥1 `retrieved_match`-факт — он ОБЯЗАН прозвучать до ухода; «передам менеджеру» без произнесённого факта запрещён. Произносится только grounded-факт, прошедший фильтры; спорное по-прежнему уходит. NEG: факт есть → произнесён + уход по остатку; P0-ход → факт не мешает увести; нет факта → как сейчас.

**В2. Calc-over-grounded.** Флаг `TELEGRAM_CALC_OVER_GROUNDED`. Якорь: free-number gate (`_free_number_gate_findings`), number_audit. Детерминир. калькулятор: разрешён БЕЛЫЙ СПИСОК операций (сумма базовых цен + применение одной известной скидки) ТОЛЬКО над `retrieved_match`-числами ОДНОГО бренда; результат помечается производным от фактов. Любой не-grounded вход или вне белого списка → запрет (как сейчас). NEG: «2 предмета со скидкой» из двух фактов → посчитано; чужой бренд / несуществующая скидка / свободное число → не считать.

**В3. Promise-vs-fact split.** Флаг `TELEGRAM_PROMISE_VS_FACT`. Якорь: `apply_unsupported_promise_guard`. Гард срабатывает только на ЛИЧНЫЕ обязательства бота («гарантирую/точно успеете/обязательно поступит»), но НЕ на пересказ grounded-факта в будущем времени («занятия проходят…», «места сохраняются до…» — если это факт базы). NEG: гарантия результата → блок (как сейчас); пересказ факта в будущем → проходит.

**В4. Soft-redact + градация по relation_to_base.** Флаг `TELEGRAM_SOFT_REDACT_GRADED`. Якорь: output sanitizer redact + действия verify_output (downgrade/block) + relation_to_base. Третий исход вместо бинарного вырезания: на ЗАКРЫТОМ списке низкорисковых (размер группы, типовой день недели, длительность занятия) → отдать с маркером неуверенности; на adjacent+не-жёсткий класс → с маркером; цена/процент/даты/юр/бренд — НИКОГДА не маркер, жёстко как сейчас. NEG: размер группы → маркер; цена → жёстко (не маркер); далёкий/жёсткий → блок.

**В5. Скоуп-адресная подача.** Флаг `TELEGRAM_SCOPE_ADDRESSED`. Якорь: fact_scope_spec + scope-aware гейт (Волна 1). Если класс клиента ВХОДИТ в диапазон grounded-факта (5-11 ⊇ 7) → ответить адресно «для 7 класса: …»; класс вне диапазона → не утверждать. Проверка вхождения детерминированная, цена из факта без изменения. NEG: класс в диапазоне → адресно; вне → не утверждать.

**В6. Per-clause gating.** Флаг `TELEGRAM_PER_CLAUSE_GATE`. Якорь: composite contract / planner_intent. Резать ход на смысловые части, прогонять гейт по КАЖДОЙ отдельно: безопасные собираются в ответ, P0/спорная часть уходит одним маркером, не топит весь ответ. NEG: «цена + можно вернуть?» → цена отвечена, возврат → менеджер; P0-часть не «протекает» в безопасную; бренд на каждой части.

**В7. Closed-world negative.** Флаг `TELEGRAM_CLOSED_WORLD_NEGATIVE`. Якорь: knowledge snapshot + предикат «уверенно нет». Когда факт-отсутствие подтверждено закрытым миром базы (явный negative-факт или scope полностью покрыт, а продукт вне него) → ответ «такого формата нет, есть вот это», а не уход. Бренд-привязка обязательна (у УНПК рассрочки нет — факт УНПК, не сравнение с Фотоном). NEG: подтверждённое отсутствие → честное «нет+альтернатива»; неуверенность → уход как сейчас; без cross-brand.

**В8. Эллипсис-резолвер перед retrieve.** Флаг `TELEGRAM_ELLIPSIS_RESOLVE`. Якорь: dialogue_memory → retrieve_facts. Короткие реплики («а онлайн?», «сколько?», «для 9 класса?») разрешать по памяти диалога (тема/бренд/класс/формат) ДО поиска факта. NEG: эллипсис → подставлен контекст → факт найден; конфликт брендов в нити → каноничная фраза разделения, не догадка; active_brand из канала, не из памяти.

## Порядок применения флагов (правка аудита)
Понимание входа: В8 (эллипсис) дополняет слоты → В6 (per-clause) расщепляет ход → по каждой части: В7 (нет факта → честное нет) / В5 (класс в диапазоне → адресно) / В1 (есть факт → произнеси) / В2 (числа → calc). Гейт выхода: В3 (promise-split) + В4 (soft-redact-градация) на финальном тексте. Порядок зафиксировать, чтобы флаги не конфликтовали.

## Граница безопасности (сквозная, НЕ трогать)
Цена/процент/даты/юр-данные, бренд-токены, P0-латч/маршрут, ПДн-сканы (вкл. эхо из памяти), authoritative gate — остаются детерминированно-жёсткими. Все послабления — ТОЛЬКО над уже-grounded фактом и в закрытых списках. OFF-паритет обязателен: со всеми флагами OFF поведение = main байт-в-байт (полный pytest зелёный).

## Тесты и замер
- pytest: полный + новые NEG на каждую правку (включая адверсариальные: бренд/P0/выдумки не ослаблены). `pytest -n 8` допустим (если есть xdist).
- Замер на М1: базлайн **all-OFF** (= приёмочный) vs пакет **`TELEGRAM_WAVE3=1` (all-ON)** на каноничном smoke_v2_acceptance, `--parallel 4`, судья v9. При регрессе — изоляция: отключать подозрительные флаги по одному (НЕ перебор 2^8). Регрейд по сырью обязателен.
- Метрики успеха: over_handoff_rate существенно вниз (с 60%), `unsupported_product_number`+redact-срубы вниз, partial-floor поднимает долю ходов с произнесённым фактом; БЕЗ роста выдумок/бренд-утечек/P0-нарушений (жёсткий гейт 0).

## Вне scope (НЕ в Волну 3)
Автоном-каркас (матрица/ворота/теневой/стоп-кран — после пилота, Б7); персонализация из CRM и обучение на правках (зависят AMO/пилота); синонимы-алиасы из данных (это наполнение KB через semantic-ревью, не код-флаг — отдельно); deferred-SEND с таймером (дорого, поздний автоном); standalone-плейбуки целиком (дорого).

## Аудит второго архитектора (внесён)
- **Р1 «8 флагов — атрибуция/объём»:** продающие убраны (уже в коде) — осталось 8 связных безопасных; общий флаг-пакет + индивидуальные; Кодекс делает read-only карту ВСЕХ точек ДО кода; при объёме — реализация группами в одной ветке, один замер. Принято.
- **Р1 «calc риск нового числа»:** белый список операций + только grounded-входы одного бренда + помётка производного. Принято.
- **Р1 «маркер на цене опасен»:** soft-redact только на закрытом низкорисковом списке; цена/дата/процент — жёстко. Принято.
- **Р2 «взаимодействие флагов»:** задан порядок применения (вход→per-clause→по частям→гейт выхода). Принято.
- **Р2 «дубль с существующими TONE_*/Q_CLARIFY»:** не дублировать, сверить картой, усилить существующее. Принято.
- **Р3 «OFF-паритет + замер»:** OFF-паритет байт-в-байт обязателен; замер базлайн vs пакет + точечная изоляция, не 2^8. Сошлось.
