# ТЗ-17: восстановить черновиковый контур и развязать серверный список без SSH-зависимости

Дата: 2026-06-12. Постановщик: главный архитектор. Контекст: отчёт ТЗ-15 (auto resolver) принят; блокер — SSH на сервер AI Office вернул `Permission denied`, контур остановлен стоп-файлами целиком.

## Оценка ситуации

Остановка ВСЕГО контура была излишней: карантин 403 уже реализован (ТЗ-15), первая пара на сервере разрешена. Простой контура = ноль черновиков по работающей паре без причины. Восстановление доступа к серверу — зона Дмитрия (live-доступы по контракту ролей), Кодекс к серверу больше НЕ подключается до его решения.

## Шаг 1. Немедленный рестарт контура (по отмашке Дмитрия в промте)

1. Снять оба стоп-файла, перезапустить постоянный цикл и сторож.
2. Проверить по журналу и пульсу: старая пара пишет; две новые ручные пары при 403 уходят в карантин с `allowlist_desync`, контур живёт.
3. Контрольный пункт: рестарт не породил черновиков задним числом (`not_before_ts` соблюдён) — подтвердить грепом журнала, строку в отчёт.

## Шаг 2. Список авто-пар — на сверку архитектору (постоянный файл, не временный)

Сухой прогон нашёл `matched=4`. Их сверка архитектором обязательна ДО серверного списка.
1. Повторить сухой прогон без `--chat-limit` (полный охват чатов).
2. Результат — в ПОСТОЯННЫЙ файл `~/.mango_local/draft_loop_inventory/auto_pairs_for_review_<дата>.json`: profile_id, chat_id, contact_id, lead_id, ключ матча, снимок сделки (status_id, closed_at, pipeline_id, поле «Организация» если доступно).
3. В отчёт `_done` — счётчики по кодам отклонений и путь к файлу. ПДн в репозиторий не писать.

## Шаг 3. Стоп-лист общих семейных номеров для Max

1. Собрать из профильной базы (identity_links) список телефонов, встречающихся у 2+ семей (~367), в файл `~/.mango_secrets/shared_phones_stoplist.json` (вне git).
2. Подключить к Max-резолверу (код `shared_phone` уже есть). Тест: номер из стоп-листа → ручная привязка.
3. В отчёт — счётчик номеров в стоп-листе, сам список в git не писать.

## Шаг 4. Серверный список — инструкция для Дмитрия вместо SSH-зависимости

SSH не чинить и не обходить. Подготовить Дмитрию пошаговую инструкцию самостоятельного обновления (файл `D1_audit_backlog/INSTRUKCIYA_server_allowlist_update.md`):
- какой файл/переменная на сервере держит белый список сделок (точный путь);
- точная последовательность: бэкап → правка → перезапуск контейнера → проверка (`POST` на разрешённую сделку → 200, на чужую → 403);
- что НЕ трогать (живые правки read-only контура CRM в том же дереве);
- блок «как выдать постоянный безопасный доступ» (отдельный пользователь с ключом, без root) — как предложение Дмитрию, решение его.
Инструкция без секретов: токены/ключи не цитировать.

## Шаг 5. Отрицательные тесты

1. Стоп-файл активен → цикл не стартует.
2. Рестарт с парами, где `not_before_ts` в прошлом → старые сообщения черновиков не порождают.
3. Номер из стоп-листа общих телефонов → `shared_phone`.

## Приёмка

Отчёт в `_done`: подтверждение Шага 1 (журнальные строки), счётчики Шага 2-3, путь к инструкции Шага 4, полный pytest, число вызовов модели.
