# Risk Review

## Основные риски

1. Question catalog файлы уже имели параллельные изменения.
   - Изменение в extractor ограничено добавлением явных source-ключей в metadata.

2. Deal-aware gate мог начать зависеть от каталога вопросов.
   - Защита: индекс полностью optional; без него старое поведение не меняется.

3. Каталог мог начать блокировать слишком широко.
   - Защита: на первом этапе включены только самые рискованные конфликты.

4. Нет проверки на реальных runtime-таблицах.
   - Осознанно: по условиям блока тесты unit/fake, без `stable_runtime`.

## Следующий контроль

Перед rebuild нужно построить source index на реальном каталоге вопросов в dry-run и посмотреть частоты срабатываний.
