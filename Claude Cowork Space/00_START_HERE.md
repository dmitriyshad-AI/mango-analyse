## Текущие договорённости (зафиксировано в переписке)

- Качество — в генерации; правила — только safety. Снести зелёные safe-template-подмены → генерация + few-shot + один rewrite.
- Финальный жёсткий слой: P0-override (нельзя перетереть) + фильтр служебных маркеров (был leak `kb_release…`).
- Гибридный P0: жёсткий список локирует, семантика только повышает; fail-closed (сомнение → менеджер).
- Честный замер: судье НЕ подсовывать собственный шаблон бота как факт; чистый holdout отдельно от dev; метрика «отправил бы без правок %»; число хардкод-веток должно ПАДАТЬ.
- Перед прогоном holdout — перезапустить ботов, чтобы взяли новый код.

## Приоритет (единый roadmap, кратко)

1. Качество бота по ТЗ v2. 2. Гигиена KB (smoke + один источник фактов + P0-recall). 3. Включить уже
построенную историю клиента (customer_timeline, ослабить пере-строгий гейт + вернуть CRM-контекст в промпт).
4. Честная петля замера. 5. Заморозить SaaS/agent/writeback/mail. 6. Ширина — только после ежедневного
использования + доказательства на holdout.
