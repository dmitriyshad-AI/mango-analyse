## Контроль при переносе в sources

- Сохранить бренд-разделение: факт Фотона не должен отвечать за УНПК МФТИ и наоборот.
- Все факты в первой таблице client-safe; внутренних пояснений, source-id и служебных пометок нет.
- Для расписания не обещать сохранение времени как гарантию: формулировка только «постараемся сохранить».
- Факт `trial.other_group_free_trial` добавлять только для Фотона; для УНПК аналогичный факт не добавлять.
- При переносе `attendance.online_instead_offline_exception_2026_27` поставить срок действия/контекст 2026/27.
- Для УНПК держать два разных онлайн-тарифа: выходные 5–11 классы (1 занятие в неделю по 3 ак.ч., 37 000/59 000) и будни 9/11 классы (2 занятия в неделю по 2 ак.ч., математика/физика/информатика, 41 800/69 900).
- Для Фотона не добавлять старую формулировку «очная подготовка к Физтеху отдельных уровней не имеет»: подтверждён новый смысл — отдельных очных групп «Физтех» нет, олимпиадные группы внутри общего расписания.
- Для gating-таблиц выше не оставлять клиенту цены индивидуальных занятий-услуги и не предлагать очное пробное УНПК/Пацаева.
- Лагерные тарифы с индивидуальным обучением/Премиум+ из v6.5 не переводить в manager_only этим staging-решением: они подтверждены отдельно.
- Скидки «второй предмет» не удалять и не переводить в manager_only.
- Рассрочку не добавлять УНПК; Фотон-рассрочку оставить по существующим подтверждённым фактам v6.5.
- Старые факты про отдельный продукт «олимпиадная подготовка Физтех онлайн» переписать: отдельного продукта нет, есть группы олимпиадного уровня внутри обычных онлайн-курсов.
