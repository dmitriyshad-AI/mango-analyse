# ТЗ MAIN — калибровка draft-промпта против over-handoff (полуфабрикат). 2026-05-31.

Автор: Клод 1. Точки проверены чтением 9e874ccf (на v2 промпт реально исполняется). Цель — поднять
автономию до потолка пользы ДО пилота: бот отвечает везде, где факт есть, и уходит к менеджеру только
когда факта нет/он чужой/P0. Сейчас over-handoff критичен (4/10 нити к менеджеру при доступных фактах).

## Корень (разбор 2 FAIL нити v2, по сырью)

НЕ розыск (факты найдены, retrieved непусто, missing=[]) и НЕ критик (findings=[], unsupported=[]).
Это сам draft-шаг: модель-черновик выдаёт шаблон «Спасибо за сообщение. Передам менеджеру» вместо
ответа из факта. Драйверы в `build_draft_prompt` (dialogue_contract_pipeline.py:757):
1. «Если сомневаешься, уточни или узко передай менеджеру; это важнее правила "ответить живо".»
2. «Если это вопрос "есть ли X", отвечай именно про X: не пиши "да/можно/доступно", если подтверждён
   только соседний факт Y.»

Модель трактует НЕДОСЛОВНОЕ совпадение факта с вопросом как «сомнение» / «соседний факт» и уходит:
- unpk_02: вопрос «олимпиада по ФИЗИКЕ есть?» + факт «олимпиадная подготовка ФИЗТЕХ для 9/11» → ушёл.
- foton_05: вопрос «в АВГУСТЕ в Москве лагерь?» + факт «лагерь Москва 3-14 АВГУСТА» → ушёл.
То есть защита от выдумок мешает отвечать из РЕАЛЬНОГО факта при иной формулировке.

## Правка — калибровка инструкций в `build_draft_prompt`

Заменить/дополнить блок «Правила ответа» так (сохранив всё про P0/бренд/числа):

Старое (убрать или смягчить):
```
"Если это вопрос «есть ли X», отвечай именно про X: не пиши «да/можно/доступно», если подтверждён только соседний факт Y. "
...
"Если сомневаешься, уточни или узко передай менеджеру; это важнее правила «ответить живо». "
```

Новое (вставить):
```
"Если в фактах есть ответ на вопрос ПО СМЫСЛУ — отвечай из него, даже если формулировка факта не "
"совпадает с вопросом дословно. Считай совпадением по смыслу: синонимы и иные названия того же продукта "
"(вопрос «олимпиада по физике» + факт «олимпиадная подготовка Физтех» — это ОДНО И ТО ЖЕ, отвечай да); "
"конкретное внутри общего (вопрос «в августе» + факт «3-14 августа» — да; вопрос «для 10 класса» + факт "
"«5-11 класс» — да). НЕ уходи к менеджеру только из-за разной формулировки.\n"
"«Соседний факт», который подставлять НЕЛЬЗЯ, — это факт про ДРУГОЙ продукт/предмет/способ оплаты/формат "
"(физика vs математика; рассрочка vs Долями; очно vs онлайн), а не тот же факт другими словами.\n"
"Передавай менеджеру при сомнении ТОЛЬКО если: по теме вопроса факта нет вовсе; найденный факт про "
"другой продукт/тему; это P0 (возврат/жалоба/спор оплаты). Если факт по теме есть и покрывает вопрос по "
"смыслу — отвечай сам, не уходи к менеджеру.\n"
```

Сохранить без изменений: «Никогда не утверждай расписание/класс/предмет/формат/цену/скидку/дату/тему,
которых нет в фактах или словах клиента»; всё про P0/бренд/числа/обещания; missing→узкий хендофф.

## Тесты (позитив + ОБЯЗАТЕЛЬНЫЙ негатив — баланс)

ПОЗИТИВ (раньше уходил зря — теперь отвечает):
- «олимпиада по физике есть?» + факт «олимпиадная подготовка Физтех 9/11» → ответ ДА из факта, route
  bot_answer_self, без «передам менеджеру».
- «в августе в Москве лагерь?» + факт «лагерь Москва 3-14 августа» → ответ да с датами.
- «для 10 класса цена?» + факт «онлайн 5-11 класс цена X» → отвечает X.

НЕГАТИВ (выдумки НЕ должны вернуться — критично):
- факт про математику, клиент спросил «физика» → НЕ подставляет математику (хендофф/нет факта).
- факт «рассрочка банк», клиент спросил «Долями» → НЕ подставляет (разные способы).
- факта по теме нет вовсе → узкий честный хендофф (как раньше).
- P0 (возврат/жалоба) → менеджер (как раньше).
- факт про очно, клиент про онлайн → НЕ выдаёт очно за онлайн.

## Замер (только перепрогон на v2 — обе стороны видны)

Это правка с ДВУСТОРОННИМ риском: слабо → over-handoff не упадёт; сильно → выдумки вернутся. Поэтому:
- Кодекс гонит pytest+smoke сам (включая NEG-тесты выше). Клод 1 проверяет в песочнике.
- Перепрогон на v2 (флаг ON), нить+батч на 2 мака: over-handoff (доля хендоффов, FAIL по «факт был, ушёл»)
  ДОЛЖЕН упасть; hard_gate (выдумки) ДОЛЖЕН остаться 0. Если выдумки выросли — откат/смягчение.

## Ограничения

- Калибровка ТОЛЬКО инструкций build_draft_prompt; не трогать гард пустых фактов, критик, reverify, P0.
- Правило #1: точное место блока «Правила ответа» подтвердить чтением (промпт длинный, сдвиги строк).
