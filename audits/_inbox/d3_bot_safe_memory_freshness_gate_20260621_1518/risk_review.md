# Risk review

## Safety boundaries

Не трогались:

- live-бот;
- AMO;
- Tallanto;
- боевая БД;
- stable_runtime.

Флаг `TELEGRAM_BOT_SAFE_CRM_CONTEXT` остался default OFF.

## Основные риски

1. Модель может проигнорировать prompt-инструкцию. Это проверяется только OFF/ON замером.
2. В смешанном контексте active + needs_review появляется хедж-инструкция. Это ожидаемо, потому что в prompt есть неподтверждённый фрагмент; active-only защищён отдельным тестом.
3. Старые bot-safe chunks без `next_step_status` продолжат работать как раньше, без нового маркера свежести.

## Почему не передаём весь next_step

Полный `metadata.next_step` может содержать текст действия, потенциально спорный или чувствительный. Для задачи свежести нужен только статус.
