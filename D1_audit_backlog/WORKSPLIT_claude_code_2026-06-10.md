# Распределение работ: Claude Code / Cowork-Claude / Кодекс (Fable, 2026-06-10)

Принцип: Кодекс — канон и всё, что мержится в main. Я (Cowork) — архитектура, ТЗ, регрейды, контент-аналитика, вердикты. Claude Code — ИЗОЛИРОВАННЫЕ большие работы, не трогающие канон до явного мержа Кодексом. Третий пишущий контур вводим только с правилами: отдельная ветка/worktree, никакого push в main, результат = дифф/артефакт на ревью, мерж — только Кодекс.

## Отдать Claude Code (большое, изолированное, после старта пилота)

1. **Майнинг корпуса звонков/переписки (F3 реестра)** — read-only по 64k расшифровок: кластеризация процессных и продающих ответов менеджеров → черновики фактов KB + кандидаты в голд-пак (B6) с маскировкой чисел. Выход: YAML-черновики на приёмку Дмитрия + мой semantic-ревью. Кода бота не касается вообще — идеальный первый заход.
2. **Расщепление subscription_llm.py (3.3 аудита)** — механический перенос по модулям на изолированной ветке, тесты зелёные, дифф на ревью Кодексу. После пилота.
3. **Ревизия 111 флагов (3.2)** — таблица «флаг → жив/мёртв/в профиле → решение», PR на удаление проигравших. После пилота.
4. **Эмбеддинг-индекс фактов (A2)** — прототип гибридного поиска на изолированной ветке + офлайн-сравнение точности с A1 по логам прогонов. Если A1 примем.
5. **Архивная гигиена (3.1)** — скрипт выноса старых прогонов/снапшотов в Яндекс-архив + политика retention (предложение, исполнение после «ок»).

## Остаётся у меня (Cowork)

Регрейды по сырью (ночные пары, pilot_smoke15, 100%-разбор первых дней пилота); ТЗ с аудитором; semantic-ревью базы и клиентских текстов; вердикты и приоритеты; ежечасный монитор шины; разбор журнала пилота → коды причин правок.

## Остаётся у Кодекса

Всё, что мержится: ТЗ-7 хвосты, live-проба, мини-фиксы KB, мини-ТЗ шины, включения по вердиктам (В6/рубрика), пилотный запуск, перенос наработок Claude Code в канон.

## Чего Claude Code НЕ давать

Прямые правки main; всё клиентское без semantic-ревью; live-доступы (AMO/Wappi секреты не выдаются третьему контуру); прогоны на М1 (станция замеров — только шина/Кодекс).
