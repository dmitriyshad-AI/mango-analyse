## Правка 1 — Б1: скоуп-осознанный числовой гейт

**Якоря** (≈ `6351de27`, искать по символу):
- `dialogue_contract_pipeline.py`: `fact_surfaces = _free_number_surfaces(" ".join(str(value) for value in facts.values()))` (~4207) — плоский union чисел ВСЕХ фактов; finding `unsupported_product_number` (~4234).
- Действия: `subscription_llm.py` `"unsupported_product_number": "block"` (~564), `"wrong_scope": "downgrade"` (~575).
- Существующая скоуп-машинерия (ПЕРЕИСПОЛЬЗОВАТЬ, не дублировать): `fact_scope_spec.detect_fact_scopes` / `fact_scopes_allowed`; `fact_claim_audit._wrong_scope_fact_findings`; `answer_quality_rewriter._wrong_scope_fact_selected_finding`.

**Проблема:** гейт сверяет число ответа с union чисел ВСЕХ фактов хода (exact+adjacent плоско). → (а) wrong-scope проходит (онлайн-цена на очный вопрос «подтверждена», если есть в любом смежном факте); (б) верный перефраз/арифметика совпавшего скоупа рубится в `unsupported_product_number`→block (12 хэндоффов в до-фикс приёмочном subset20).

**Цель:** число сверяется с фактом, чей СКОУП совпал с вопросом, через существующий `fact_scope_spec`.
**Флаг** `TELEGRAM_NUMBER_GATE_SCOPE_AWARE` (default OFF). ON:
- число, совпавшее по скоупу источника → поддержано — в т.ч. перефраз/арифметика, НО ТОЛЬКО если значение равно значению exact-факта совпавшего скоупа или явно выводимо арифметикой над значениями ТОГО ЖЕ факта; произвольное число → block (как сейчас);
- число из несовпавшего/смежного скоупа → `wrong_scope` → downgrade (действие уже есть), не «supported».
OFF → прежнее плоское поведение.

**NEG:**
1. OFF-паритет: на существующих кейсах поведение байт-в-байт прежнее.
2. wrong-scope: «сколько стоит очно физика 9 класс», в паке есть онлайн-цена → ON ловит `wrong_scope`→downgrade; OFF пропускает.
3. перефраз: верное число совпавшего скоупа перефразировано/посчитано из факта → OFF рубит, ON держит.
4. адверсариальный: НОВОЕ число под видом перефраза (не выводимо из факта) → ON всё равно block.

## Правка 2 — Б5: закрыть pure_handoff-дыру верификатора

**Якоря:**
- `subscription_llm.py` skip: `if dialogue_contract_is_pure_handoff_text(result.draft_text) and not handoff_claim_text:` → `skip_reason="pure_handoff"` (~6764).
- `dialogue_contract_pipeline.py`: `_is_pure_handoff_text` (~6123); `_FACTUAL_CLAIM_RE`/`_HANDOFF_FACTUAL_CLAIM_RE` (~202/208) — ловят только ₽/%/даты.
- тест-якорь: `tests/test_subscription_llm_draft_provider.py::test_semantic_output_verifier_skips_only_locked_or_pure_handoff_texts`.

**Проблема:** текст-хэндофф без чисел → pure_handoff → семантический верификатор СКИПается. Качественные нечисловые обобщения («сильные преподаватели», «гарантируем интерес») в обёртке хэндоффа уходят в черновик без смысловой проверки.

**Цель:** скип ТОЛЬКО для канонических «пустых» шаблонов; хэндофф с содержательным остатком → через семантический верификатор.
**Флаг** `TELEGRAM_VERIFIER_HANDOFF_CLAIMS` (default OFF). ON:
- скип pure_handoff разрешён ТОЛЬКО если текст ∈ замкнутое множество канонических шаблонов (`SAFE_FALLBACK_DRAFT_TEXT` + явные locked safe-templates);
- любой непустой содержательный остаток после удаления канонических хэндофф-фраз → НЕ скип → семантический верификатор (advisory: annotate/downgrade, штатно).
OFF → прежнее.

**Бело-список, НЕ чёрный (решение аудита):** не детектор «оценочных слов» (это тот же keyword-whack-a-mole, что выпиливаем), а замкнутый белый список безопасно-пустых шаблонов; при сомнении — не скипать (добавляет проверок = безопасно).
**Разграничение с Б1:** Б5 гонит через верификатор только КАЧЕСТВЕННЫЕ НЕЧИСЛОВЫЕ обобщения; числа/цены держит числовой гейт (Б1) — без двойной проверки. Согласовать с калибровкой верификатора из step5-D5 («не флагай склейку двух реальных фактов», «не проверяй цены») — Б5 их не нарушает, т.к. не трогает числа.

**NEG:**
1. OFF-паритет: тест `...skips_only_locked_or_pure_handoff_texts` зелёный при OFF.
2. чистый «передам менеджеру» → ON: skip_reason=pure_handoff (как сейчас).
3. «у нас сильные преподаватели, передам менеджеру» → ON: верификатор отрабатывает (skip_reason пуст, checked).
4. хэндофф с ЧИСЛОМ → остаётся в числовом гейте (Б1), Б5 не дублирует.
5. P0/бренд при ON → маршрут не меняется.
