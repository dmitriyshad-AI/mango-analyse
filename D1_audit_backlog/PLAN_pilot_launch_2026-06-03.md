## Логика лестницы (от низкого риска к высокому)
Черновик-с-одобрением (риск ~0, менеджер вычитывает) → доказали на людях → автоотправка с защитой → вечерняя
автономия. Каждая ступень открывается ТОЛЬКО после метрик предыдущей.

---

## ЭТАП 0 — предпилот (инфраструктура), ~1-2 недели
Цель: бот доступен менеджерам как черновик + видимость качества. Без клиентов.
- **AMO-интеграция (черновик в карточке):** входящее из Telegram/МАКС → бот пишет черновик → менеджер видит его в
  карточке сделки, правит/шлёт в один клик. active_brand берётся из воронки/источника AMO (не угадывать). Read-only
  поля клиента для контекста. [Кодекс: вебхук/Salesbot-прокси + виджет/примечание; см. идею A3 из штурма]
- **Конфиг автономии:** живая матрица «тема → черновик / всегда-менеджер» (Дмитрий крутит без деплоя) + стоп-кран по
  теме. На старте ВСЁ — черновик.
- **Мониторинг:** сэмплирование ответов + алерт на бренд-смешение/P0-пропуск/выдумку/утечку в проде (Claude
  регрейдит выборку еженедельно).
- **Метрика доверия:** `unedited_rate` — доля черновиков, отправленных менеджером БЕЗ правок.

## ЭТАП 1 — внутренний пилот на сотрудниках, ~1-2 недели
Цель: обкатать на безопасной аудитории, поймать реальные косяки.
- Сотрудники задают реальные/каверзные вопросы; менеджеры работают с черновиком.
- **Петля «ошибка → тест + факт»:** каждый реальный косяк → NEG-тест и/или KB-факт (качество компаундится).
- Выход: unedited_rate растёт, безопасность 0 на реальных формулировках, менеджеры говорят «помогает».

## ЭТАП 2 — пилот на ЛОЯЛЬНЫХ клиентах (черновик), ~2-4 недели
Цель: первое касание реальных клиентов, под полным контролем менеджера.
- Небольшая подготовленная группа лояльных клиентов; КАЖДОЕ сообщение — черновик с одобрением менеджера.
- P0 (возврат/жалоба/спор/гарантия) — всегда менеджеру, бот не трогает.
- **Здесь впервые честно меряем:** реальный over-handoff, тон на живых людях, конверсия лид→оплата, скорость ответа,
  unedited_rate. Это и есть настоящая приёмка (синтетика была прокси).

## ЭТАП 3 — слой автоотправки (СТРОИМ, до автономии), ~2-3 недели
Цель: защита без менеджера-пола (нужно ДО любой автономии).
- Стоп-кран (мгновенно выключить тему/бота), захват лида, эскалация P0, частотные лимиты, строгий гейт на исходящие
  (только подтверждённый факт, только история диалога, бренд заперт).
- Первая проактивность РУКАМИ менеджера: утренний дайджест «кого написать» + реактивация «ушёл подумать» (черновик).

## ЭТАП 4 — вечерняя/ночная АВТОНОМИЯ на узком сегменте, осторожно
Цель: автоотправка там, где доказан эффект и низок риск.
- Старт на ОДНОМ денежном узком сегменте (напр. ночной рекламный трафик на конкретный продукт), не широко.
- Только темы из матрицы автономии, прошедшие этапы 1-2 чисто; P0 → менеджер всегда; слой этапа 3 включён.
- Мониторинг + сэмплирование в реальном времени; стоп-кран наготове.

---

## Метрики (по этапам)
- Безопасность (всегда = 0): бренд-смешение, P0-пропуск, выдумка, утечка, identity-нарушение, обещание/давление.
- `unedited_rate` — северная звезда доверия к черновику.
- Реальный over-handoff (на живом трафике, не синтетика).
- Конверсия лид→оплата, время первого ответа, доля автономных ответов.
- Выручка на менеджера / на сегмент (ROI бота).

## Кто что делает
- **Кодекс:** AMO-интеграция (вебхук/Salesbot, черновик в карточке, active_brand по воронке, read-only поля,
  логирование); конфиг автономии + стоп-кран; детектор продающего шага и `unedited_rate` в метрики; слой
  автоотправки (этап 3).
- **Дмитрий:** доступы к AMO; выбор лояльной группы и денежного сегмента; контент недостающих фактов; решения по
  матрице автономии (какие темы бот ведёт сам); подтверждение к каждому этапу.
- **Claude #1:** регрейд реального трафика по сырью (еженедельно); петля «ошибка → тест/факт»; semantic-review новых
  фактов; пороги перехода между этапами; разбор, где бот теряет клиента.

## Принцип, который не нарушаем
Каждая ступень открывается ТОЛЬКО по метрикам предыдущей на РЕАЛЬНОМ трафике. «Зелёная синтетика ≠ готовность» —
это доказала вся наша работа. Автономия — последняя, не первая.
