# Semantic review

## Проверка бизнес-смысла

Изменение не генерирует клиентский текст, но влияет на смысловую память бота: к какому ребёнку относится класс/предмет из текущей реплики.

Проверенные сценарии:

- Флаг OFF: готовый модельный `child_key` игнорируется, старое поведение сохранено.
- Флаг ON: одиночная реплика "дочке в 7 класс" привязывается к готовому `child_key` из профиля, а не к регулярному `child_2`.
- Флаг ON и два ребёнка в одном сообщении: "сыну в 7, дочке в 4" остаются двумя разными детскими слотами.

Вывод: смысловой риск неверной привязки одиночного детского упоминания снижен без автоматического схлопывания реальных разных детей.
