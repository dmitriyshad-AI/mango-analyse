# Боевой пакет v2 (для интеграции Кодексом). 2026-05-26

Авторство архитектуры/логики — Claude; боевую стыковку + симуляторы делает Кодекс (у него рантайм, codex CLI, pytest, понимание грязного дерева).
Это апгрейд уже внедрённой Кодексом Фазы 1 (`dialogue_contract_pipeline.py`) до ПОЛНОЙ v2 по ТЗ §1-§13.

## Содержимое пакета
- `modules/` — канонические модули v2-логики (LLM инъектируется, офлайн): dialogue_understanding (контракт-ПЛАН + слоты-с-источником + client_state + P0-пре-гейт),
  quality_layer (семантическая верность FAIL-CLOSED + form-check + X2-тепло), output_verifier (жёсткие проверки), fact_provider (ретривал по ключам), pipeline (полный поток + под-тумблеры).
- `tests/test_pipeline.py` — офлайн-тесты на моке. **Результат: 13/13 passed** (включая fail-closed семантики, слот-без-источника, не-числовую выдумку, X2-ре-верификацию, бренд-утечку).
- `INTEGRATION.md` — точная дельта Фаза-1 → полная v2: что добавить/изменить, точки интеграции, какие реальные раннеры/склад/каталог подключить.
- `GUARDS.md` — guard-таблица: оставить / отключить в v2 / спорно (с правкой по autonomy_matrix).
- `INVARIANT_13_7_PROOF.md` — доказательство §13.7: кто меняет `draft_text` в v2 (ровно три класса).

## Как применять (Кодекс)
1. Прочитать ТЗ `D1_audit_backlog/TZ_redesign_dialogue_contract_2026-05-26.md` (§1-§13) — это спецификация.
2. По `INTEGRATION.md` поднять Фазу 1 до полной v2 (канон логики — `modules/`).
3. Подключить реальные раннеры (LLM #1/#2/#3), склад, каталог, поролевую историю, стилевые примеры; прунинг гвардов по `GUARDS.md`; держать §13.7.
4. Офлайн pytest → СТОП → диф Claude на ревью → один прогон симулятора (флаг ON vs OFF, полный набор).

## Жёсткие инварианты (НЕ ронять)
Контракт называет ключи (не значения); слот без источника не утверждать; P0 детерминированный пре-гейт; жёсткий verifier + семантическая верность
ОДНОВРЕМЕННО, семантика FAIL-CLOSED; бренд-скоуп жёсткий; нет факта → узкий хендофф; X2 — только форма, ре-верифицируется; §13.7 (только 3 мутатора текста);
маршрут считает КОД после ретривала, не LLM-`answerability`.
