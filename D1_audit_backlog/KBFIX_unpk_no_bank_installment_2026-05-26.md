# KB-фикс: явные отрицательные факты (известное отсутствие). 2026-05-26

Автор: Claude. Исполнитель: Codex (сборка релиза базы). Короткая правка, отдельно от Волны 3.
Read-only стороной Claude проверю результат.

## Зачем (обоснование)
Бот по дизайну НЕ выводит «нет» из отсутствия факта (closed-world опасен: пустота может значить
«не внесли», а не «реально нет»). Поэтому, чтобы бот честно отвечал клиенту «нет», нужен ЯВНЫЙ
подтверждённый отрицательный факт. Сейчас по «есть ли банковская рассрочка у УНПК?» такого факта нет →
бот хватает соседний (общая оплата) и говорит «Да» (диалог 23 живого прогона).

## Что добавить
**Отрицательный client-safe факт для УНПК (оба компонента в одном бренде, БЕЗ упоминания Фотона):**
- `payment_options_unpk.bank_installment.absent` (или по вашей схеме):
  client_safe_text (ревизия 2, строже — без продажных обещаний скидки в неверном контексте):
  «В УНПК отдельной банковской рассрочки нет. Оплату можно обсудить с менеджером: доступны помесячная
  оплата, оплата за семестр или за год; условия и скидки менеджер подтвердит по выбранной программе.»
  Поля: `allowed_for_client_answer=true`, `requires_manager_confirmation=false`, `forbidden_for_client=false`,
  brand=unpk, `forbidden_client_mentions` включает Фотон/Т-Банк/Долями (бренд-разделение).
  Почему так: НЕ писать «семестр/год СО СКИДКОЙ» утвердительно (бот может пообещать скидку не в том
  контексте) — скидку оставить за менеджером «по выбранной программе».

**Жёсткий инвариант бренд-разделения:** отрицательный факт УНПК НЕ должен ссылаться на Фотон («а у
Фотона есть») — только «у нас нет, но есть помесячно». Аналогично для любого known-absent факта.

## Заодно (обобщение класса «известное отсутствие»)
Завести явные отрицательные факты для частых «а есть ли у вас X?», где ответ «нет», по каждому бренду
раздельно. Кандидаты (проверить, чего ещё нет, но спрашивают): банковская рассрочка УНПК (этот фикс);
модульные курсы М9/М11 (статус discontinued — есть ли уже client-safe «нет»?). Не плодить впрок —
только реально частые вопросы (см. сезонный анализ звонков второго диалога, когда будет).

## Проверки и сборка
- Сверить: у Фотона факт про рассрочку (положительный) на месте — контраст реальный, но в РАЗНЫХ
  брендах (не в одном ответе).
- Пересобрать снимок v6.3 ПРАВИЛЬНЫМ билдером `build_kb_release_v6_1_team_answers.py` (применяет манифест),
  НЕ старым v3. Гейт целостности Волны 1 должен пройти (`quality_passed=true`).

## Критерии приёмки (проверю независимо)
1. В снимке есть отрицательный факт УНПК про банковскую рассрочку, `allowed_for_client_answer=true`,
   без упоминания Фотона/Т-Банк/Долями.
2. Бот на «есть рассрочка через банк?» (УНПК) → честно «нет, но есть помесячно/семестр/год», НЕ «Да».
3. Бренд-разделение не нарушено; целостность Волны 1 зелёная; число фактов выросло на ожидаемое.
4. Это работает в связке с Волной 3 §2 (C8): бот использует отрицательный факт как ответ «нет».
