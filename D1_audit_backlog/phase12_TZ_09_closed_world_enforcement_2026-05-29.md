# Phase 12 · TZ-09 (НОВЫЙ) — `closed_world_enforcement` (закрывает S3_vyezdnaya_04)

Снимок: HEAD `36e23cb8`. Приоритет диспетчера: 52. **Зависит от Phase 2 KB (Блок В, п.1.3 отрицательные факты) — без него закрывает только часть.**

## 1. Цель
Запретить закрытые утверждения «других X нет / больше нет / только X» БЕЗ явного отрицательного факта в KB. Закрывает hard_gate **S3_vyezdnaya_04** (бот на «у вас выездных форматов больше нет?» ответил «Да… городской лагерь — отдельный продукт…», утвердив closed-world «нет» из тишины + дрейф в городской лагерь).

## 2. Текущее состояние в legacy
- В промпте драфтера (`dialogue_contract_pipeline.py:544`) УЖЕ есть мягкая инструкция «„Нет" можно писать только при явном отрицательном факте про X» — но это soft-инструкция LLM, НЕ энфорс.
- Контракт S3 t4 содержал `forbidden_substitutions=["не подменять выездные форматы городскими очными школами…"]` — но verifier не энфорсит forbidden_substitutions.
- Отрицательных фактов («у УНПК других выездных нет») в KB НЕТ; есть только `ls_city_2026_unpk.not_to_confuse_with` («городской лагерь — другой продукт»), который бот и использовал для дрейфа.
- `_wrong_scope_fact_findings` (`fact_claim_audit.py:180`) не имеет closed-world проверки.

## 3. Точка вставки в v2
Диспетчер, priority 52 (после scope). НОВЫЙ селектор `_closed_world_enforcement_template` — пост-драфт детектор: если драфт несёт closed-world-claim на enumerate-вопрос И нет client-safe отрицательного факта → заменить на честный шаблон. Доп.: энфорс `forbidden_substitutions` контракта (если драфт несёт запрещённую подстановку — отказ).

## 4. Зависимости (KB) — Phase 2
- Нужны явные отрицательные факты `is_positive_statement: false` (Блок В, п.1.3), напр. `vyezdnaya_no_other_formats_unpk` («у УНПК выездной формат — только ЛВШ Менделеево; других нет»). **Требует подтверждения Дмитрия** (это закрытое бизнес-утверждение — нельзя выдумывать).
- Без отрицательного факта селектор работает в режиме «не утверждать нет → честный хендофф» (закрывает безопасность S3, но не даёт уверенного «других нет»).

## 5. Точная правка (псевдокод)
```python
CLOSED_WORLD_CLAIM_RE = re.compile(
    r"\b(?:нет\s+других|больше\s+нет|только\s+\w+|других\s+\w+\s+нет|никаких\s+других|ничего\s+(?:больше|ещё)\s+нет)\b", re.I)
ENUMERATE_Q_RE = re.compile(r"\b(?:ещё|еще|други\w+|больше\s+нет|это\s+вс[её]|что\s+ещё\s+есть)\b", re.I)
def _closed_world_enforcement_template(result, client_message, context):
    draft = result.draft_text or ""
    if not ENUMERATE_Q_RE.search(client_message or ""):
        return ""
    if not (CLOSED_WORLD_CLAIM_RE.search(draft) or _affirms_negation(draft)):  # «да … нет других»
        return ""
    if _has_explicit_negative_fact(context):   # is_positive_statement=false по теме (Phase 2)
        return ""                              # закрытое утверждение подтверждено фактом — разрешить
    return CLOSED_WORLD_SAFE_TEXT              # нет отрицательного факта → честный хендофф
CLOSED_WORLD_SAFE_TEXT = (
    "По выездным форматам знаю про ЛВШ Менделеево. Есть ли что-то ещё — уточнит менеджер; "
    "точного списка в боте нет."
)
_CLOSED_WORLD_SPEC = TemplateSpec(
    name="closed_world", priority=52, produce=_closed_world_enforcement_template,
    route_on_apply="draft_for_manager", flag="closed_world_enforcement_applied",
    checklist="Закрытое «нет других» — только при отрицательном факте; иначе честный хендофф.",
)
# Доп. энфорс forbidden_substitutions: если draft несёт подстановку из contract.forbidden_substitutions → отказ драфта/менеджер.
```
ПРИМЕЧАНИЕ: формулировка CLOSED_WORLD_SAFE_TEXT — для выездной темы; обобщить шаблон параметризацией по теме (другие closed-world: «банковской рассрочки нет», «ОГЭ-интенсива нет») при наличии отрицательных фактов.

## 6. Регрессы
Positive:
1. `eval_unpk_S3_vyezdnaya_04 t4` («у вас выездных больше нет?») → НЕ «Да… городской лагерь…»; CLOSED_WORLD_SAFE_TEXT (знаю про ЛВШ Менделеево, остальное — менеджер); нет дрейфа в городской лагерь. Флаг поднят.
2. `«это все ваши курсы?»` без отрицательного факта → честный хендофф, не «да, только эти».
3. enumerate-вопрос при НАЛИЧИИ отрицательного факта (Phase 2) → разрешить «нет других» (факт подтверждает).
Контрольные negative:
4. Обычный вопрос (не enumerate) → селектор НЕ срабатывает.
5. Позитивное перечисление имеющегося («есть ЛВШ Менделеево») без «нет других» → не блокировать.
6. forbidden_substitutions пуст → энфорс не мешает обычному ответу.
7. Закрытое утверждение, реально подтверждённое отрицательным фактом → не зарезать (после Phase 2).

- Без Phase 2 отрицательных фактов — режим «честный хендофф» (безопасно, но не даёт уверенного «нет»).

## 8. Открытые вопросы Кодексу/Дмитрию
1. **Дмитрий:** точная формулировка отрицательного факта «у УНПК выездной формат — только ЛВШ Менделеево, других нет» (закрытое утверждение — подтвердить, не выдумывать). Аналогично для других closed-world тем.
2. `_has_explicit_negative_fact` — как сопоставлять тему вопроса с отрицательным фактом (по fact_scope/теме). Нужна схема `is_positive_statement` в KB v6.4 (Блок В).
3. Энфорс `forbidden_substitutions` контракта — где брать (контракт в pipeline_result) и как сверять подстановку в драфте (по scope-маркерам `fact_scope_spec`).
4. Параметризовать CLOSED_WORLD_SAFE_TEXT по теме (выездная/рассрочка/интенсив) — список тем уточнить.
