## §2. Принципы и правила

1. **Calibrate before enforce.** Сначала калибровка защит (Phase 11), потом принуждение использовать
   факты (Phase 1).
2. **Маленькие волны, не один батч.** Между волнами — микро-прогон на затронутых архетипах.
3. **Полный прогон 212 — ОДИН раз после всех волн.**
4. **Безопасный слой не трогаем.** Брэнд (0/212), P0 (0/212 пропусков), мета-утечки (1/212).
5. **Регресс-suite отдельно от eval.** Eval — do-not-tune-against.
6. **Тест входа волны = перепрогон затронутых кейсов на текущем коде.** Часть может уже быть
   зелёной — экономия работы.
