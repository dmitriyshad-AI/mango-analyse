# Контракт правила (Шаг 2a): «расписание». 2026-06-02.

Автор: Клод 1. По эталону. Источник — `_class_schedule_publication_answer`, `_asks_class_schedule_days`,
`_draft_uses_contact_hours_as_schedule`, `_soft_weekend_guidance_text` (pipeline 2468-2581). Прочитано по
сырью + память. Перекликается с задачей расписания у Кодекса (данные из групп Tallanto).

## Как сейчас (лазанья)
Расписание размазано: распознавание «дни занятий» + проверка «черновик подставил контактные часы как дни» +
soft-weekend + публикация. Плюс данные расписания НЕ в коде — они деривируются из групп Tallanto (день/время
в названии группы).

## Целевой контракт (слой 2)

```yaml
rule_id: schedule
title: Расписание (дни/время занятий, публикация, старт)
intent: schedule_inquiry
intent_subvariants: [class_days_time, publication_status, weekend_slots, start_date]
# Данные расписания — ДЕРИВИРУЮТСЯ из Tallanto-групп (не хардкод). Это пересечение с задачей расписания.
required_fact_keys:
  - schedule.groups.*                  # дериватив из Tallanto: бренд×площадка×день×слот×предмет×класс
  - academic_year_2026_27.start*       # старт занятий (12-27 сентября, по площадке)
  - academic_year_2026_27.weekly_lessons   # 1 раз/неделю (regular)
  - schedule.publication_status        # расписание конкретной группы опубликовано/появится
data_rules:
  source_of_truth: tallanto_groups     # день/время в НАЗВАНИИ группы (парсинг), бренд×площадка×слот×предмет×класс
  group_schedule_is_dynamic: true      # точное расписание группы часто ещё НЕ опубликовано → менеджер сверит
  contact_hours_not_class_days: true   # КОНТАКТНЫЕ часы (Пн-Вс 10-18) ≠ дни занятий — НЕ подставлять
  weekend_slots: soft_guidance         # «бывают слоты по выходным» — мягкий ориентир, не точное подтверждение
  grid: {slots_per_day: 3, slot_hours: 2}   # сетка МФТИ (если в факте)
brand_split: true                      # площадки/группы раздельно по бренду
scope: {dimensions: [brand, площадка, день, слот, предмет, класс]}
blocking_conditions:
  - exact_group_schedule_unpublished   # точные дни/время конкретной группы не подтверждены → менеджер сверит класс/предмет/площадку
route_effect: bot_answer_self          # старт/кадэнс/публикация-статус из факта; точные дни группы → manager/уточнение
text_effect: composer_generates_from_data
preserve_exceptions:
  - КОНТАКТНЫЕ часы (10:00-18:00, Пн-Вс) НИКОГДА не выдавать как дни/время занятий (наш wrong_intent-гард)
  - точное расписание группы без проверки НЕ подтверждать → «менеджер сверит класс, предмет, площадку»
  - «бывают по выходным» — как мягкий ориентир, не как подтверждённый слот
  - старт занятий по площадке (12-27 сент) — из факта, не путать с расписанием занятий
```

## Распределение по слоям
- **Planner:** `intent=schedule_inquiry` + subvariant (дни/публикация/выходные/старт) + класс/предмет/
  формат/площадка + uncertainty. Заменяет `_asks_class_schedule_days`.
- **KB/Tallanto-факты:** дериватив групп (день/время), старт, кадэнс. Значения из Tallanto, не код.
- **Слой 2:** правила (contact_hours≠class_days, group_schedule_dynamic→manager, weekend=soft, grid) + scope.
- **Composer:** старт/кадэнс/статус из факта; точные дни группы → честный дефер «менеджер сверит».
- **Гейт:** контакт-часы как расписание → `wrong_intent_fact` downgrade (это наш существующий чек, теперь
  декларативное правило); точные дни без факта → downgrade.

## Связь с задачей расписания у Кодекса
Данные `schedule.groups.*` — это РОВНО то, что Кодекс деривирует из Tallanto (сверка 25.05 МФТИ + новый
Фотон-лист). То есть слой 2 расписания питается из той задачи: Tallanto-группы → деривированные факты →
правило отвечает по ним. Бренд-сплит групп (красный=Фотон/чёрный=УНПК) ложится в `brand_split`.

## Обязательные НЕГАТИВНЫЕ тесты
1. «в какие дни занятия 7 класс физика?» при наличии только контактных часов → НЕ выдать «Пн-Вс 10-18» как
   дни (контакт-часы ≠ дни занятий) → менеджер сверит.
2. Точное расписание конкретной группы не опубликовано → «менеджер сверит класс/предмет/площадку», не выдумать.
3. «бывают по выходным?» → мягкий ориентир, не подтверждённый слот.
4. Старт: «когда начнутся занятия?» + факт → 12-27 сентября по площадке (из факта).
5. Бренд: группы Фотон ≠ УНПК (раздельно).
6. ПОЗИТИВ: «сколько раз в неделю?» + факт → 1 раз/неделю, route=answer_self.

## Вывод
Расписание = Tallanto-факты (значения) + правила (контакт-часы≠дни, динамика группы→менеджер, weekend=soft) +
composer. Это смесь паттернов: KB/Tallanto-факт (как цены) + live-неопубликованность (как места кампа) +
скрытое исключение контакт-часов (наш wrong_intent-гард → правило). Пять семейств разобраны: рассрочка,
скидки, цены, лагерь, расписание — весь спектр моделей переноса покрыт. Осталось мелкое (пробное, платформа,
маткапитал/налог) — оно проще и уже частично в гейте/A2.1.
