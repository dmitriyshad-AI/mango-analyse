# Semantic Review

Проверенный смысловой риск: бот не должен утверждать клиенту конкретный шаг из памяти, если этот шаг помечен как спорный или пустой.

Результат:

- При `needs_manager_review` и утверждении вида `место забронировано`, `заявка подтверждена`, `место закреплено` черновик заменяется на безопасное уточнение через менеджера.
- При `active` такой же текст не меняется: подтверждённая нить общения сохраняется.
- Нейтральный перевод к менеджеру без конкретного спорного шага не меняется.
- Слой не зависит от LLM-верификатора и не может молча fail-open при его таймауте.

Смысловой статус: `semantic_pass` для механизма. Реальные кейсы 04/13/18 требуют следующего поведенческого регрейда на сырье.
