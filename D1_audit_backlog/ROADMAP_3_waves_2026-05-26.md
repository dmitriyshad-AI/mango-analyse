# Маршрут внедрения: 3 волны (Тир-1+2 из бэклога идей). 2026-05-26

Решение Дмитрия: внедрять не одним мега-ТЗ, а тремя волнами по подсистемам, с прогоном+регрейдом
между волнами. Цельность видения — в этом документе; детальное ТЗ пишется по одной волне за раз.
Источник идей: `IDEAS_backlog_2026-05-26.md` (§9 ранжирование).

## Принцип порядка
Сначала — надёжное ИЗМЕРЕНИЕ и ЦЕЛОСТНОСТЬ базы. Пока измерение врёт, а база может корраптиться,
любое улучшение тона/маршрута мы катим вслепую и не отличим «улучшили» от «сломали». Только потом —
безопасность автономии (анти-склейка), и в конце — полезность (тон/over-handoff), где риск перегиба
выше всего и нужен честный замер под рукой.

## Волна 1 — фундамент: целостность базы + измерение  ← ТЗ пишется первым
Состав: **B2** (кросс-сверка text↔structured_value по ВСЕМ числам), **B3** (sanity-диапазоны полей),
**M4** (обязательный CI-гейт целостности при сборке), **M7** (доверительный интервал + порог второго
прогона), **M3** (детерминированный grep чисел claim↔retrieved до судьи), **M8** (разметка hard-gate:
объект/измерение/инфра).
Поведение бота НЕ трогаем. Уже сделано (не дублировать): узкий `invalid_weekly_frequency_fact`,
судья из `retrieved_facts`. Готовая основа для B2/B3: мой сканер `kb_integrity_scan.py`.
Критерий перехода к Волне 2: пересборка ловит порчу текста зелёным гейтом; в summary есть интервал и
needs_second_run; провалы размечены по причине; ручной регрейд чисел почти не нужен.

## Волна 2 — безопасность автономии: анти-склейка + тёплый слой
Состав: **V1** (утверждение против ОДНОГО факта), **V5** (анти-склейка в тёплый промпт), **V7** (класс
«именованная сущность вне фактов»), **V10** (регресс-набор на склейку), кластер **T1/C1** (тепло меняет
только форму / фактический скелет), **V4** (калибровка тёплого: добавил факт vs перефразировал).
Почему до полезности: режим all_eligible включён, склейка — измеренный пробой; усиливать автономию,
не закрыв склейку, опасно.
Критерий перехода к Волне 3: на регресс-наборе склейка ловится; доля отклонений тёплого снижена без
роста реальных пропусков; на холдауте склейка = 0.

## Волна 3 — полезность: понимание, маршрут, меньше сухих хендоффов
Состав: **U1** (детектор мягкого возврата), **U5** (трёхуровневый маршрут self/partial/manager),
**P9** (факт есть — отвечай), **C8** («известное отсутствие» как факт), **T5** (один хороший вопрос
вместо пустого хендоффа), кластер **U6/B5/B6** (тёплый частичный из мягких фактов + recall по теме).
Закрывает оба оставшихся средних класса (мягкий возврат = U1; частичный ответ = U6) и бьёт по тону/
over-handoff — но уже на честном измерении и закрытой склейке.
Критерий: тон вырос, over_handoff снизился, при этом выдумки 0, P0/бренды держатся (проверяемо, т.к.
измерение честное после Волны 1).

## После трёх волн
Честный полный round-5/6 на починенном измерении → СМОТРИМ цифры → решаем, нужна ли **C2** (golden-path
рецепты, M-L/сред). Не фиксируем C2 заранее: возможно, точечные победы волн 2-3 уже дадут достаточно.

## Сквозные правила (все волны)
Не ослаблять безопасность/P0/бренд-скоуп. Холдаут не тюнить. Codex сначала аудитит ТЗ волны, потом
внедряет за флагом, прогоняет, отдаёт мне на независимый регрейд. Между волнами — прогон, не копить.
