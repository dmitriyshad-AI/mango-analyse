# Промты для бизнес-аудита модулей проекта

Дата: 2026-05-19

Цель: запустить в разных диалогах согласованный read-only аудит проекта не только по коду, но и по бизнес-пользе: что реально помогает запуску Telegram-бота, CRM-обогащению, email-черновикам и будущему SaaS.

## Как использовать

1. Открой нужный диалог.
2. Вставь промт из соответствующего файла.
3. Попроси диалог работать строго read-only.
4. По итогу каждый диалог должен создать свой файл аудита в `docs/`.
5. Затем в главном диалоге нужно собрать единый реестр решений.

## Файлы

- `00_COMMON_AUDIT_RULES.md` — общие правила для всех диалогов.
- `01_GLOBAL_STRATEGY_AUDIT_PROMPT.md` — главный аудит проекта и стратегия.
- `02_CALLS_PIPELINE_AUDIT_PROMPT.md` — звонки, аудио, ASR, Resolve+Analyze, canonical master.
- `03_EMAIL_PIPELINE_AUDIT_PROMPT.md` — почта, email-архив, будущие черновики писем.
- `04_TELEGRAM_BOT_AUDIT_PROMPT.md` — Telegram-пилот, база знаний, черновики, MEGA-тесты.
- `05_SAAS_PRODUCT_AUDIT_PROMPT.md` — переход к SaaS и продуктовая упаковка.

## Важный принцип

Это не разработка и не уборка. Это инвентаризация пользы:

- что уже работает;
- что формально есть, но не используется;
- что выглядит лишним, но может быть ценным;
- где есть бизнес-риск;
- что мешает запуску Telegram-пилота;
- что нужно делать дальше по приоритетам.
