# Roadmap

Дата обновления: 2026-05-23

Назначение: короткая рабочая дорожная карта. Подробная историческая версия находится в `docs/PROJECT_HISTORY_AND_PRODUCT_ROADMAP_2026-05-13.md`.

## Цель проекта

Сначала создать внутреннего ИИ-сотрудника для отдела продаж и клиентских коммуникаций. Потом, после подтвержденной пользы внутри компании, двигаться к продукту для других организаций.

## Целевое состояние

Система должна:

- видеть историю клиента по звонкам, письмам, AMO, Tallanto и мессенджерам;
- понимать конкретную сделку AMO;
- показывать менеджеру краткую полезную картину;
- предлагать следующий шаг;
- помогать писать ответы клиентам;
- не писать опасные данные без проверки;
- давать РОПу понятные правила и контроль качества;
- в будущем работать через сайт, Telegram, Max, CRM и рабочее место менеджера.

## Этап 7. Первый внутренний запуск

Статус: в работе. Запущены два публичных Telegram-бота для внутреннего пилота сотрудников: Фотон и УНПК МФТИ. Используется база знаний v6.3, gold-ответы v3, read-only CRM/Tallanto/context builder, P0-регистр и локальный pilot storage.

Нужно закрыть:

- запустить два раздельных контура: Фотон и УНПК МФТИ;
- вести единый журнал пилота: входящее, ответ, маршрут, факты, флаги, задержка, известные данные, оценка сотрудника;
- строить ежедневный отчёт владельцу/РОПу;
- сохранять P0/register и очередь смыслового ревью;
- импортировать feedback сотрудников в store;
- для новых лидов давать полезные ответы, интересовать обучением и мягко вести к следующему шагу;
- для известных клиентов отвечать автономно только по безопасным темам и только при наличии проверенного факта;
- P0-темы всегда отправлять менеджеру;
- в нерабочее время давать максимум безопасной пользы, но не выдумывать факты и не обещать спорные действия;
- замерять ежедневное использование, полезность черновиков и случаи ручного вмешательства.

Главная метрика:

сколько менеджеров реально пользуются системой каждый день.

Первое правило автономности:

- если тема не попадает в матрицу автономности или есть сомнение, бот не отвечает сам;
- факт считается пригодным для автономного ответа только если в базе он помечен как клиентский и актуальный;
- если сообщение содержит несколько тем и одна из них P0/high-risk, весь вопрос уходит менеджеру.

## Этап 8. Расширение внутри компании

Нужно:

- подключить больше менеджеров;
- добавить форму сайта;
- расширить Telegram-пилот после прохождения `v8_targeted16`, статичных `v6/v5` и последующего полного v8;
- добавить Max позже, не в текущем блоке;
- подключить почтовые черновики;
- сделать рабочее место менеджера.
- добавить мягкие напоминания клиентам только после подтверждения правил частоты и сценариев;
- добавить AMO/Tallanto-запись сначала через подтверждение менеджера, затем точечно автоматизировать безопасные поля.

## Этап 9. Переход к продукту для других организаций

Условие перехода:

- система стабильно используется внутри компании;
- есть метрики пользы;
- есть понятные процессы внедрения;
- есть защита данных и права доступа;
- есть повторяемая установка для другой организации.
