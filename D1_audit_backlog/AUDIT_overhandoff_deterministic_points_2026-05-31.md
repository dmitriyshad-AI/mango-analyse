# Аудит детерминированных точек over-handoff (уход к менеджеру при доступном факте). 2026-05-31.

Автор: Клод 1. Read-only разбор 9e874ccf. Цель: найти места, где скрипт/инструкция уводит к менеджеру,
хотя факт по теме есть/может быть найден — вместо понимания намерения и ответа фактом.

## ГЛАВНОЕ — understanding ставит manager_only «при низкой уверенности» (корень до розыска)

`build_understanding_prompt` (pipeline.py ~300), инструкция:
> «Если факта нет или **уверенность низкая**: answerability=manager_only, но current_question заполни.»

Это перестраховка на уровне ПОНИМАНИЯ: модель при сомнении в формулировке/теме ставит manager_only
ДО розыска фактов. Тогда `run_pipeline` видит answerability=manager_only → уходит к менеджеру, даже
если факт по теме ЕСТЬ в store. Калибровка draft-промпта (9525f31c) этот путь НЕ лечит — draft не
вызывается, когда понимание уже решило manager_only.

Это ПАРНЫЙ корень к draft-калибровке. Лечение — калибровка understanding-инструкции:
- «answer_self по умолчанию для обычных вопросов про обучение/цены/форматы/расписание/продукты.»
- «manager_only ТОЛЬКО при P0 (возврат/жалоба/спор оплаты/юр-угроза) или вопрос явно вне сферы центра.»
- «Низкая уверенность в ФОРМУЛИРОВКЕ — НЕ повод для manager_only: заполни current_question и
  needed_fact_keys, пусть розыск и draft попробуют ответить из факта. Уверенность отражай в поле
  confidence, но не превращай в хендофф.»
NEG: P0/вне сферы → manager_only остаётся; не выдумывать (это держат гард пустых фактов + критик).

## Связь с текущим прогоном (9525f31c, идёт)

Калибровка draft (9525f31c) лечит over-handoff на уровне DRAFT (модель уходит при сомнении). Прогон
покажет, СКОЛЬКО она сняла. Остаток over-handoff — вероятно от understanding manager_only (пункт 1):
если бот уходит к менеджеру с route=manager_only (а не bot_answer_self+хендофф-текст), значит понимание
поставило manager — это пункт 1, нужна парная калибровка understanding.

Диагностика остатка по сырью: route на FAIL-ходах. route=draft_for_manager/manager_only + факт в
retrieved → understanding-manager (пункт 1). route=bot_answer_self + хендофф-текст → draft (уже
калибровано). Так разделим, нужна ли understanding-калибровка.
