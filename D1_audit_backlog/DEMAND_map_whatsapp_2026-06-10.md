# Частотная карта клиентского спроса — WhatsApp-экспорт (2026-06-10)

Источник: `all_whatsapp_chats.txt` (~17 МБ, 366 614 строк). Анализ read-only, скриптом в песочнице.
В отчёте только агрегаты и обезличенные шаблоны; сырых цитат с ПДн нет.

## Метод (эвристика)

- Формат файла: блоки `===== CHAT: <номер> =====`; внутри даты `YYYY-MM-DD`; исходящее сообщение = строка `ЧЧ:ММ` + строка `You`; входящее = строка с телефоном + `ЧЧ:ММ`; далее строки текста до следующего блока. Хвост `End of History` + лицензионный футер отбрасывается.
- Цитаты-ответы (`Replying to` + цитата в кавычках) вырезаются: классифицируется только собственный текст клиента.
- «Первое содержательное сообщение клиента» = первое входящее длиной >15 символов, не служебное (`The media is missing`, `Message deleted`, `Not supported WhatsApp internal message`, пропущенные звонки).
- Классификация: регекс-словарь тем (ё→е, нижний регистр), multi-label — одно сообщение может попасть в несколько тем; суммы >100%. Словарь расширен по данным: добавлены темы «общий запрос информации», «данные для записи (ФИО/почта)», «подтверждение/благодарность», «отказ/пока не актуально», «формат очно/онлайн».

## (б) Обезличенные шаблоны формулировок — топ-10 тем (без «другое» и «подтверждение»)

1. **класс/предмет/программа**
   - «Добрый день. Хотела записать дочь на курс математики. По воскресеньям?»
   - «Уточните пожалуйста, есть ли у вас летние программы для детей <N> лет?»
2. **формат очно/онлайн**
   - «Добрый день, есть ли у вас курсы для <N>-ти классников, <N> лет, онлайн?»
   - «Добрый день, а запись на курсы для <N> класса в Москве очно уже открыты?»
3. **лагерь/ЛВШ/смена**
   - «Здравствуйте! Подскажите пожалуйста, ещё есть места в лагерь на июль?»
   - «Доброе утро! Подскажите, как записаться в летнюю школу на третью смену?»
4. **общий запрос информации**
   - «Добрый день, интересуют курсы для <N> класса, какие есть программы?»
   - «Добрый день! Вы мне звонили по поводу курсов. Можете перезвонить?»
5. **запись/как записаться**
   - «Добрый вечер! Подскажите, можно записаться на учёбу <N> класс, онлайн?»
   - «Доброго дня! Ещё можно записаться на смену в августе в Долгопрудный?»
6. **расписание/дни/время**
   - «Добрый день, подскажите, как проходят занятия, по какому расписанию?»
   - «Здравствуйте! Как проходит обучение, по какому графику, какие предметы?»
7. **оплата/рассрочка/QR**
   - «Скажите пожалуйста, предполагается ли рассрочка и каковы условия?»
   - «Добрый день! Можно мне скинуть на почту ссылку для оплаты курсов?»
8. **документы/договор/справка**
   - «Добрый день. Квитанцию об оплате зимней школы по какому адресу направлять?»
   - «Можно оплатить маткапиталом? И как заключить договор?»
9. **данные для записи (ФИО/почта)**
   - «Добрый день ещё раз. Спасибо за информацию. Адрес электронной почты: <почта>»
   - «Добрый день! Письмо с результатами не получила. Просьба прислать на почту <почта>»
10. **платформа/ссылка/доступ**
    - «Добрый день. Завтра на какой платформе будут занятия? И время московское?»
    - «Добрый день! Получается, первое занятие уже было, к нему будет доступ?»

## Аномалии (важное для бота)

1. **Канал на 100% исходящий**: первым всегда пишет центр (шаблон WhatsApp Business по заявке с сайта). «Первые вопросы клиентов» здесь = ответы на аутрич. Чистого входящего спроса «с нуля» в этом файле нет — для частот «кто что спрашивает первым» канал смещён.
2. **Цена в первых сообщениях редка (4,4%, №13)** — не потому что неинтересна: центр сам высылает прайс первым сообщением (исходящие шаблоны «Стоимость…» — сотни вхождений). Спрос на цену гасится до вопроса. В Telegram-боте без аутрича доля вопросов о цене будет выше.
3. **Смешение брендов подтверждено**: в 596 чатах (12,9%) исходящие упоминают и УНПК/МФТИ, и Фотон. Для бота — недопустимый паттерн, живые менеджеры так делали массово.
4. **Файл исторический**: данные с 2022-08, заявленный период сен2025–май2026 = лишь 11,8% новых диалогов. Частоты усредняют 4 учебных года.
5. **Лагерь/ЛВШ — №2–3 реального интента** (16,5% первых сообщений), сильная сезонность (заявки май–август).
6. Жалоб в первых сообщениях практически нет (2 шт.), возврат — 0,6%: негатив приходит позже по диалогу, не во входе.
7. 26,7% чатов — клиент вообще не ответил на аутрич.

## Ограничения метода

- Multi-label регекс: суммы >100%; остаточный шум классификации единицы процентов (выборочно проверено, явные ложные срабатывания словаря исправлены: «девочек»≠«чек», «договориться»≠«договор»).
- Порог >15 символов отсекает короткие, но содержательные ответы («Онлайн», «<N> класс») — 444 чата дошли до классификации более поздним длинным сообщением, 248 выпали совсем.
- Классифицируется ТОЛЬКО первое содержательное сообщение — темы, всплывающие глубже по диалогу (возврат, жалобы, перенос), занижены.
- Тип медиа неразличим (один маркер на голосовые/фото/документы).
- 2 чата с именованными контактами не распарсились как входящие (формат строки отправителя отличается).
- Дата первого сообщения ≠ дата экспорта: помесячная таблица отражает старты диалогов.
- Бренд-разрез по ключевым словам исходящих — грубая оценка, не сверка по номерам аккаунтов.
