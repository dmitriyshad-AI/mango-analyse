# ТЗ-155 — Лёгкие замеры: git-синхрон + крошечные job-манифесты вместо 800МБ-бандлов

- Дата: 2026-06-30. Постановщик: Claude #1. Исполнитель: **D6** (инфра). Проверка — моя. Канон: main.
- Цель: убрать ~800МБ-бандлы и копирование файлов; «задание на прогон» = крошечный манифест (SHA + набор + флаги), а код+база приходят к M1 через `git checkout`. Замеры — в разы быстрее.

## Факты (проверены по сырью)
- Весь репозиторий, ВКЛЮЧАЯ `product_data/knowledge_base` (980 файлов), **отслеживается git**. Один снапшот пилота = 23 МБ.
- `build_mango_clean_bundle.py` делает `shutil.copy` всего дерева → тащит ВСЕ 68 снапшотов (585 МБ из 802). Балласт.
- Вотчер M1 заброшен (tombstone): M1 запускает человек ВРУЧНУЮ через Я.Диск. Схема должна быть ручной, без автомата.

## Схема (git-синхрон через bare-зеркало на Я.Диске)
1. **Bare-зеркало на Я.Диске:** создать `~/Yandex.Disk.localized/OpenClaw/mango_repo.git` (bare). На главном маке добавить remote `yandex` и `git push yandex main + рабочие ветки`. (git-объекты сжаты/дедуплицированы → зеркало много меньше 800МБ; `fetch` тащит только дельту. Интернет/ключи не нужны — синхронит сам Я.Диск.)
2. **M1 — разовый клон:** `git clone <bare> mango_run`; дальше только `git fetch yandex`.
3. **Наборы тестов — в git:** закоммитить eval-наборы (`product_data/telegram_dynamic_test_sets/*.jsonl`, включая P0-stability + forward-payment персоны + 25 Wappi what-if), чтобы они приезжали вместе с `checkout`, а не таскались отдельно.
4. **Job-манифест (крошечный, на Я.Диске):** `scripts/build_job_manifest.py` пишет `job_<sha>.json` = `{commit_sha, set_rel_path, set_sha256, snapshot_rel_path, env_flags (напр. TELEGRAM_SEMANTIC_FRAME_SHADOW=1), parallel, max_hours, run_cmd}`. Килобайты.
5. **Процедура M1 (ручная):** `git fetch yandex && git checkout <sha>` → проверить `HEAD==sha` и sha256 набора → запустить `run_telegram_dynamic_client_sim.py` с набором + env-флагами из манифеста. Результаты остаются локально / при желании малый `git push` ветки результатов.
6. **Тяжёлый путь** `build_mango_clean_bundle` (полная копия) — оставить как фолбэк за явным флагом, по умолчанию не использовать; основной путь — git+манифест.
