## A2 — детерминированный маппинг сигнал→приём
- `objection=price` на денежной/ценовой теме → продающий вариант денежного правила (ценность+способ оплаты+мягкий
  шаг), НЕ сухая цена/факт.
- `exit_signal=true` на любой безопасной теме → к ответу добавляется мягкий удерживающий шаг (без давления).
- Оба сигнала вместе → сначала отработать цену (objection), удержание — мягким шагом в конце.

## A3 — A-det композиция (конкретные варианты; числа ТОЛЬКО из факта)

### installment (Фотон), objection=price
- Сейчас: «В Фотоне можно оплатить обучение частями: 6, 10 или 12 месяцев, а также Долями.»
- Вариант: «Понимаю, важно, чтобы вложение было посильным. Оплату можно разбить — {месяцы из факта} или Долями,
  частями без нагрузки. Подобрать удобный вариант?» (месяцы подставляются из факта, не хардкод).

### installment/discount (УНПК), objection=price — банковской рассрочки НЕТ
- Вариант: «Понимаю. В УНПК можно платить помесячно, за семестр или за год — за семестр скидка 10%, за год 14%.
  Подсказать, что выгоднее в вашем случае?» (проценты из факта; НЕ предлагать рассрочку Фотона).

### price (есть факт цены), objection=price
- К цене из факта добавить указатель на способ оплаты по бренду + мягкий шаг: «… Если по бюджету важно — оплату
  можно разбить (рассрочка/Долями [Фотон] | помесячно/семестр/год [УНПК]). Подсказать удобный вариант?»
- price БЕЗ факта + objection → как в B1: `route=draft_for_manager`, цену НЕ выдумывать (вариант не применять).

### exit_signal (любая безопасная тема)
- Добавить мягкий удерживающий шаг. Если для active_brand есть ПОДТВЕРЖДЁННЫЙ факт пробного занятия → «Если
  поможет решить — можно начать с пробного, без обязательств. Подсказать, как записаться?» Если факта пробного
  НЕТ → нейтральный мягкий шаг без выдумки пробного: «Спокойно подумайте; если нужно — подскажу, что важно для
  решения по {тема}.»

## A4 — выходной гейт ВЕРИФИЦИРУЕТ (не ослаблять)
Гейт Шага 1 проверяет вариант: число только из факта, нет обещания результата/поступления, нет утечки второго
бренда, P0 не пропущен. Если вариант не грунтуется — гейт срезает до безопасного. Приём НЕ имеет приоритета над
гейтом.

## NEG-матрица (pytest, детерминированно — без LLM; без неё приём НЕ принят)
1. `objection=price` + installment(Фотон) с фактом → текст содержит месяцы ИЗ факта + мягкий шаг; нет выдуманного
   числа.
2. `objection=price` УНПК → только условия УНПК (помесячно/семестр/год, 10%/14%); **рассрочки Фотона НЕТ** (бренд
   не смешан).
3. Возражение про ВОЗВРАТ денег («верните деньги», «хочу вернуть») → **P0 manager_only, НЕ отработка возражения**.
4. `exit_signal` без факта пробного → мягкий шаг БЕЗ выдумки пробного; нет давления; route корректный.
5. `exit_signal` с фактом пробного по бренду → предложение пробного грунтовано фактом.
6. `exit_signal` НЕ перебивает P0/бренд/факт-гард.
7. price-no-fact + objection → `draft_for_manager`, цена не выдумана (B1 цел).
8. Контроль не-затронутого: `anxiety/unmet_need/readiness` в Волне 1 поведение НЕ меняют (нет варианта — обычный
   ответ).
9. «сколько стоит?» (objection=none, чистый факт) → обычный фактовый ответ, БЕЗ навязанного продающего довеска
   (приём только при сигнале, не на каждый ценовой вопрос).
