# Telegram Bot Autonomy Matrix v1

Дата: 2026-05-21

Назначение: зафиксировать, когда Telegram-бот Фотона/УНПК может отвечать сам, когда должен делать черновик менеджеру, а когда обязан передать вопрос менеджеру.

## Главный принцип

По умолчанию бот не отвечает автономно.

Автономный маршрут `bot_answer_self_for_pilot` допустим только если одновременно выполнены все условия:

- активный бренд известен: `foton` или `unpk`;
- тема входит в матрицу автономности;
- сообщение является самостоятельным вопросом;
- нет P0/high-risk темы;
- нет недостающих фактов;
- в контексте есть факт с явными флагами клиентской безопасности и актуальности;
- факт относится только к активному бренду.

Модель не имеет права сама решать, что факт актуален. Это определяется только флагами базы знаний и контекста.

## Зелёные темы

Зелёные темы - это темы, где бот может отвечать сам, если есть проверенный клиентский факт. Если LLM по такой теме вернула обычный `draft_for_manager`, контур может повысить маршрут до `bot_answer_self_for_pilot`, но только после всех защит.

В v1 в матрицу зелёных тем входят:

- `theme:001_pricing` - стоимость, только текущие подтвержденные цены;
- `theme:005_discounts` - скидки, только с условиями и без индивидуальных обещаний;
- `theme:006_installment` - рассрочка/частичная оплата, только утвержденные формулировки бренда;
- `theme:007_matkap_payment` - маткапитал, только справочная информация без обещания одобрения;
- `theme:008_tax_deduction` - налоговый вычет, только справочная информация без обещания решения ФНС;
- `theme:011_contract` - договор, без сбора персональных данных в рискованных случаях;
- `theme:012_certificates` - справки и документы, без сбора лишних данных;
- `theme:013_schedule` - расписание, только если есть актуальный факт;
- `theme:014_format` - формат обучения;
- `theme:015_address` - адреса активного бренда;
- `theme:016_program` - программа курса;
- `theme:018_materials_homework` - материалы и домашние задания;
- `theme:019a_positive_feedback` - благодарность/позитив;
- `theme:020_enrollment` - запись на обучение;
- `theme:021_continuation` - продолжение обучения;
- `theme:022_age_level_testing` - возраст, класс, уровень, тестирование;
- `theme:023_trial_class` - пробное занятие;
- `theme:024_account_access` - личный кабинет;
- `theme:025_missing_links_access` - ссылки и доступы;
- `theme:026_camp_general` - лагеря/выездные школы;
- `theme:027_camp_living_conditions` - бытовые условия лагеря;
- `theme:028_transport_logistics` - транспорт и логистика;
- `service:S5_general_consultation` - общая консультация.

## Только менеджер

Всегда `manager_only`:

- `theme:009_refund` - возврат денег;
- `theme:019b_negative_feedback` - жалоба, конфликт, негатив;
- `theme:029_legal_question` - юридический вопрос, суд, прокуратура, претензия;
- сообщения с персональными данными в рискованном контексте;
- любые смешанные сообщения, где есть хотя бы одна P0/high-risk часть.

## Черновик менеджеру

`draft_for_manager` используется, когда:

- тема не входит в матрицу автономности;
- факт есть, но не помечен как клиентский и актуальный;
- есть сомнение по бренду;
- вопрос требует уточнения у AMO/Tallanto/менеджера;
- есть несколько безопасных тем, но бот не уверен в главной;
- это не самостоятельный вопрос, а продолжение диалога.

Если точного факта нет, бот не должен ограничиваться пустым “уточним у менеджера”. Он должен дать полезный общий ответ без точных обещаний, задать 1-3 безопасных уточняющих вопроса и передать точные детали менеджеру.

## Следствие для пилота

Бот может становиться полезнее без ослабления P0-защиты: безопасные темы с проверенными фактами можно постепенно переводить в автономные ответы, а P0-темы всегда остаются у менеджера.
