# 10 идей: проактивность и исходящие. 2026-06-03

Автор: Claude (субагент-стратег). Угол: бот пишет первым и дожимает без навязчивости.
Инварианты: только тем, с кем уже был диалог; бренды не смешиваются; P0/жалобы — мимо;
только подтверждённые факты (closed-world).

---

## 1. Реактивация «ушёл подумать» — точечное касание через 48 ч

**Суть.** AMO CRM хранит этап воронки. Если диалог завис на «думаю» дольше 48 часов без ответа клиента
— бот отправляет одно короткое сообщение со ссылкой на конкретный факт, который был в последнем вопросе
(цена, формат, рассрочка). Не «как дела», а «держите то, о чём спрашивали — готов ответить».

**Почему полезно.** Менеджер не помнит каждый диалог; большинство «тёплых» теряется из-за паузы.
Одно точное касание возвращает лида без раздражения.

**Сложность.** Средняя. Нужен триггер по этапу/дате в AMO + шаблон факт-привязки.

**Как не стать спамом.** Один раз на диалог, только если активный этап (не закрыт/отказ).
Если ответа нет — больше не пишем. P0-контекст в истории → пропускаем полностью.

---

## 2. Напоминание о старте группы — за 5 дней и за 1 день

**Суть.** Когда клиент получил информацию о конкретной группе (класс, предмет, формат) —
зафиксировать это в памяти нити. Если до старта группы остаётся 5 дней → одно сообщение с фактом
«группа стартует X, осталось N мест» (только если факт подтверждён). За 1 день — финальное касание.

**Почему полезно.** Честная срочность из реального факта (не выдумка). Клиент, который «думал»,
получает естественный триггер принятия решения без давления.

**Сложность.** Средняя. Нужна связка дат групп из KB + этапа клиента в AMO.

**Как не стать спамом.** Максимум 2 касания на группу. Только тем, кто явно спрашивал об этой группе
(scope = точный подвопрос). Если клиент ответил «не интересует» — снимаем флаг.

---

## 3. Раннее бронирование со срочностью — только когда факт есть

**Суть.** Для тех, кто узнавал про ЛВШ или летние форматы 3+ месяца назад и не записался —
в момент открытия раннего бронирования (когда факт появился в KB) отправить персонализированное
сообщение: «вы интересовались лагерем — открылось раннее бронирование, мест мало».
Только по active_brand; никакого сравнения Фотон/УНПК.

**Почему полезно.** Закрывает «я собирался, но забыл» — самый частый тип потери тёплого лида.
Честная срочность: дефицит мест по ЛВШ — подтверждённый факт (MEMORY: «места почти распроданы»).

**Сложность.** Средняя. Нужен механизм «клиент интересовался темой X» → очередь на касание при
появлении нового факта по теме X в KB.

**Как не стать спамом.** Только один раз, только на появление факта (не повтор). Тема чувствительна
(деньги заранее) — только если факт client-safe и нет P0-контекста в истории диалога.

---

## 4. Сезонное касание в пик спроса — осень, август

**Суть.** В августе — начале сентября (пик записи) бот отправляет короткое приветствие тем, кто
общался в прошлом году, но не купил: «новый учебный год — группы открыты, готов ответить на вопросы».
Без давления, только факт о старте + один вопрос.

**Почему полезно.** Лето — провал (MEMORY: «деньги осенью, лето — провал»). Осенью тёплые прошлогодние
лиды — дешевле нового трафика. Одно сообщение в нужный момент возвращает диалог.

**Сложность.** Низкая. Сегмент — AMO, этап «не купил», диалог > 6 месяцев назад.
Триггер — календарная дата (1 и 20 августа).

**Как не стать спамом.** Один раз в год на клиента. Строго по active_brand. Не отправлять,
если последнее действие в диалоге — жалоба/отказ/P0.

---

## 5. Follow-up после паузы в середине диалога — «вы задали вопрос, я не услышал ответа»

**Суть.** Бот задал уточняющий вопрос (класс, предмет, формат) — клиент не ответил 24 ч.
Одно мягкое касание: «если что-то непонятно — уточните, готов помочь». Без напоминания о цене,
без давления. Просто держит диалог открытым.

**Почему полезно.** Незавершённый диалог чаще всего не «отказ», а «забыл/занят».
Одно касание с низким порогом ответа (без вопроса о деньгах) возвращает часть таких диалогов.

**Сложность.** Низкая. Детектор «бот спросил → ответа нет → 24 ч» — простой триггер.

**Как не стать спамом.** Только один раз. Только если бот сам задал вопрос (не клиент).
Если ответа снова нет — диалог закрывается без дальнейших касаний.

---

## 6. Реактивация через новость — появился новый факт, релевантный интересу клиента

**Суть.** В KB появляется новый факт (новая смена ЛВШ, обновлённые цены, новый формат).
Бот проверяет AMO: у кого в истории была тема этого факта и кто не купил.
Отправляет адресное сообщение: «появилась информация по тому, о чём вы спрашивали».

**Почему полезно.** Персонализировано по реальному интересу клиента, не массовый blast.
Клиент чувствует, что его «запомнили» — это снижает сопротивление.

**Сложность.** Высокая. Нужна связка KB-обновлений с историей тем в диалогах AMO.
На первом этапе можно вручную запускать для конкретного обновления KB.

**Как не стать спамом.** Только на реальное появление нового факта (не ретроспективное).
Тема должна точно совпадать с тем, что клиент спрашивал (scope = точный подвопрос).

---

## 7. Дожим возражения по цене — рассрочка или второй предмет

**Суть.** Если последний вопрос клиента был про цену и он не ответил после ответа бота —
через 72 часа одно сообщение: «кстати, если вопрос в оплате — есть рассрочка (только Фотон)
или онлайн-формат дешевле». Строго по active_brand, только факты из KB.

**Почему полезно.** Возражение по цене — самый частый стопор (playbook: рассрочка закрывает
большинство ценовых возражений). Бот сам не додумался предложить в ответе — касание закрывает этот пробел.

**Сложность.** Средняя. Нужен детектор «последняя тема = цена + нет ответа + active_brand = Фотон
(рассрочка)». УНПК: только скидка за год или онлайн-формат.

**Как не стать спамом.** Один раз. Только на ценовое возражение (не общее «думаю»).
Не предлагать рассрочку УНПК (её нет) — строгое соответствие factID.

## 8. Ninja по этапу воронки: клиент на «согласован договор» без оплаты > 5 дней

**Суть.** AMO показывает: договор согласован, оплата не поступила, прошло 5 дней.
Бот отправляет: «договор готов, по оплате — долями или рассрочка, если удобнее». Один вопрос,
без давления. Менеджер видит касание в AMO.

**Почему полезно.** Финальный шаг — самый дорогой потерянный лид. Часто клиент просто
«завис» на оплате, а не отказался. Одно напоминание конвертирует без давления.

**Сложность.** Средняя. Нужен этап AMO + факт «долями 4 части» из KB. Бот не подтверждает
оплату сам (только read-only), передаёт менеджеру при ответе.

**Как не стать спамом.** Только один раз. Если оплата не поступила ещё через 3 дня —
флаг менеджеру (не новое сообщение от бота). Спор об оплате/жалоба → P0, мимо бота.

## 9. Летний «тихий пилот» — короткая серия для уже лояльных клиентов прошлого года

**Суть.** Июль-август — провал по трафику. Вместо рекламы: бот пишет 20-30 самым лояльным
прошлогодним клиентам (купили, не жаловались) с коротким вопросом «планируете в следующем году?»
и ссылкой на лист ожидания ЗВШ Менделеево (факт: «дат нет, лист ожидания» — честно).

**Почему полезно.** Формирует лист ожидания заранее. Лояльные клиенты — самый дешёвый
источник. Одновременно тестирует проактивный канал без публичного риска.

**Сложность.** Низкая. Небольшой сегмент, ручной запуск, только подтверждённые факты.

**Как не стать спамом.** Один раз летом. Только сегмент «купил + нет P0 + > 6 мес с последнего
диалога». Ответ «нет» → флаг «не реактивировать». Обязательное одобрение Дмитрия перед отправкой.

## 10. Утренний дайджест для менеджера — «вот кого стоит написать сегодня»

**Суть.** Каждое утро в 09:00 бот собирает из AMO список из 3-5 клиентов, которым пора сделать
касание (истекает срок «думаю», стартует группа, не ответил на вопрос бота). Отправляет менеджеру
в Telegram сводку с именем/темой/предлагаемым текстом касания. Менеджер отправляет одним нажатием
или редактирует.

**Почему полезно.** Снижает когнитивную нагрузку менеджера: не надо помнить, кому писать.
Бот не пишет клиенту сам — менеджер контролирует. Безопасно, нет риска spam/P0-утечки.

**Сложность.** Высокая. Нужна сборка из AMO + ранжирование по приоритету + шаблон сводки.
Зато снимает страх «дать боту автономию» — менеджер всегда финальный отправитель.

**Как не стать спамом.** Бот пишет только менеджеру, не клиенту. Клиенту пишет менеджер
руками. Дайджест содержит только проверенные касания (нет P0-контекста, нет закрытых этапов).
