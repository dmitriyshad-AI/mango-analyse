## Правила переноса

### trial
- подвиды: online_fragment / offline_trial / process / data_needed / fragment_access / payment / ack.
- **free-trial-guard:** Фотон НЕ обещает бесплатное ОЧНОЕ пробное (`_foton_offline_free_trial_guard_template`).
- «передайте менеджеру» БЕЗ «как/способ/ссылка» → чистый хендофф, не ответ про пробное.
- «не онлайн» → не предлагать онлайн-фрагмент.

### camp_lvsh (САМАЯ грязная — особое внимание)
- **scope:** residential (ЛВШ Менделеево, проживание/трансфер) vs city_day (городской дневной) — по сигналам.
- **brand-split строго:** ЛВШ Фотона ≠ ЛВШ УНПК, НИКОГДА не сравнивать/смешивать.
- **места = LIVE-данные → менеджер:** «есть места / запишите» → живой менеджер, значения НЕТ (динамика CRM),
  НЕ обещать наличие. (blocking → seats_or_booking)
- **scarcity_is_fact:** «места почти распроданы» — подтверждённый ФАКТ, НЕ флагать как искусственную срочность.
- **ЗВШ 2026/27:** дат нет, лист ожидания.
- **camp_refund → P0:** лагерь + возврат/претензия → менеджер (blocking → camp_refund).
- «не лагерь» → не отвечать кампом.

### enrollment_process (вкл. гипотетический возврат)
- «как записаться/оформить» → шаги из факта.
- **payment_method «Долями/рассрочка»** → отдать правилу `installment` (единый источник), не дублировать.
- **предпродажный возврат:** гипотетический «если передумаю до оплаты, вернёте?» → спокойная справка
  «возвращается остаток неистраченных средств» (`_presale_refund_policy_template`, non-P0); РЕАЛЬНАЯ претензия
  (оплатил+нет занятий / верните деньги недоволен) → P0 менеджер.
- «закрепить/зафиксировать цену» → price_fix-намерение, не общий процесс.

## ⚠ Связь с корнем Прогона 1 (важно, но НЕ в объёме правки правила)
Гипотетический возврат на ПОВТОРНЫХ ходах защёлкивался в hard_p0 (вход `p0_pregate`, не правило). Миграция
enrollment_process даёт правилу способность отвечать справкой про остаток, НО латч p0_pregate — отдельная
калибровка ВХОДНОГО классификатора (требование #1 из `STEP2a_handoff_for_codex`). В 2b.5 НЕ трогаем p0_pregate;
только подтверждаем, что правило enrollment корректно отдаёт presale-справку, когда ему дают ответить.
Калибровку p0_pregate вынести отдельной задачей после миграции.
