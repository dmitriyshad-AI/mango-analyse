# Описание подготовленных материалов по WhatsApp-выгрузке. Клод 2, 2026-05-30

Назначение по ТЗ Клода 1: WhatsApp — канал до перехода на Telegram, источник реальных
**вопросов и тона** для **customer pipeline**, а НЕ фактов для базы знаний. В базу из выгрузки
ничего не вливаем. Ниже — описание по единому шаблону.

## 1. Источник

- **Период (содержательные сообщения):** 2022-09-02 — 2026-05-25.
  По годам: 2022 — 533, 2023 — 4660, 2024 — 26 995 (пик), 2025 — 21 135, 2026 — 233.
  То есть основная масса 2024–2025; 2026 мало (уже шёл переход на Telegram).
- **Объём:** 4538 диалогов, 61 540 сообщений. Роли: клиент 30 235, менеджер 23 321,
  служебные 7984. Исходник `all_whatsapp_chats.txt` — 17 МБ, ~366 тыс. строк.
- **Что исключено на входе (по решению Дмитрия):** 82 не-телефонных диалога — длинные
  внутренние WhatsApp-ID, иностранные номера, рекламные баннеры WhatsApp Business и
  отдельный чат «WhatsApp Calls». Оставлены только диалоги с ID-телефоном `[78]\d{10}`.
- **Бренды:** физически в одном потоке (одна БД), но у каждого диалога есть поле `brand_hint`,
  поэтому УНПК и Фотон **разделяемы**. Разметка диалогов: без бренда 1817, УНПК 1490,
  смешанные 917, Фотон 314.

## 4. Качество и ограничения

**Надёжно (проверено `whatsapp_verify.py`, вердикт PASS):**

- целостность: 0 «осиротевших» сообщений, 0 без даты, все телефоны в формате +7, метка бренда
  в БД совпадает с пересчётом;
- матчинг: `primary_phone` = нормализованный `chat_id` совпадает на 100%;
- бренд-метки: риски ложной метки по слову «фотон» (это ещё и термин физики) и «долями»
  (обычное «частями») проверены адресно — **не подтвердились**;
- P0: исправлена переловля «суд» внутри «обсудим» (юр-угрозы 91 → 14 реальных).

**Мусор / ограничения (аналогично Telegram-батчу transcripts — мусор был, отфильтрован):**

- 7984 служебных сообщения (рекламные баннеры WhatsApp Business, «Not supported WhatsApp
  internal message», плейсхолдеры экспорта «The media is missing», «Replying to…») помечены
  `is_service_message=1` и исключаются из аналитики;
- 100 диалогов (2,2%) без единого содержательного сообщения (пустые / только-медиа);
- матчинг с CRM покрывает 2764 диалога (61%); остальные 39% — без записи в CRM (новые лиды
  без звонка либо номер не в базе);
- менеджеры в выгрузке обезличены как `You` — атрибуция к конкретному сотруднику невозможна;
- 60% диалогов без чёткого бренда (1817 null + 917 mixed) — для них бренд не определён однозначно;
- репрезентативность: только диалоги с телефоном-ID (98%+ выгрузки); 82 не-телефонных чата выкинуты.

**Персональные данные (важно):**

- БД содержит ПДн в открытом виде: телефоны (chat_id), а в текстах — ФИО детей, email, адреса.
  По решению Дмитрия БД хранится **локально** и не публикуется.
- Часть **открытых md-отчётов** содержит фрагменты реальных сообщений с именами (например, в
  P0-примерах) и в паре мест — `chat_id` (= телефоны клиентов). **Перед передачей этих отчётов
  за пределы команды телефоны и имена стоит замаскировать.** Для customer pipeline сами ПДн
  должны оставаться в локальной БД, в открытые материалы их не выносить.

## 5. Бренд-разделение

- Бренд определяется по диалогу полем `brand_hint` (по однозначным маркерам: УНПК — унпк/мфти/
  менделеево/kmipt/сретенк/институтск/долгопрудн/пацаев; Фотон — фотон/цдпо/црдо/скорняжн/
  cdpofoton/долями). Оба маркера → `mixed`; иначе пусто.
- Во всех выжимках (темы, вопросы, тон, P0) разбивка **по бренду**; `mixed` и `null`
  **не приписываются** ни одному бренду.
- В сравнении каналов бренд берётся только со стороны WhatsApp; звонкам бренд **не приписывался**,
  чтобы не смешивать.
- Смешения брендов в выжимках нет. Отдельный отчёт `whatsapp_brand_leaks` — это, наоборот,
  список диалогов, где **менеджеры** в переписке упоминали оба бренда (материал для тренинга
  команды), а не смешение в моих выжимках.

## 6. Назначение для customer pipeline

- **Частота тем и вопросов** → приоритизация: что клиенты спрашивают чаще всего. Топ вопросов:
  программа/уровень (1962), цена/оплата (1534), формат онлайн/очно (796), лагерь/ЛВШ (758),
  расписание (674), документы/справка/вычет (652), олимпиады (625). По бренду раздельно.
- **Тон менеджеров** → обучение ответам и калибровка черновиков: привычная клиенту манера
  (приветствие, вежливые «пожалуйста/спасибо», умеренные emoji ~5–9%, короткие сообщения).
- **Реальные P0-формулировки и их частота** → калибровка safety-слоя (Волна 1a): P0 редок
  (0,46%), значит детектор должен быть точным, пограничные случаи — на менеджера.
- **Контекст по телефону** (провайдер + ТЗ Этапа 5) → встраивание WhatsApp как канала в боевой
  customer timeline: события WhatsApp склеиваются с звонками/CRM по телефону. Главная ценность —
  за 2022–2024 (до телефонии) WhatsApp был единственным письменным следом клиента, и 1773
  клиента (39%) вообще не имеют звонков.

**Готово:** БД, вся аналитика, верификация (PASS), провайдер, ТЗ врезки.
**Требует доработки:** реализация врезки в timeline — это код, зона Codex, после «ок» Дмитрия.

**Важно по установке Клода 1:** из WhatsApp в базу знаний ничего не вливаем — это вопросы и тон,
не утверждённые факты. Поэтому подготовленный ранее файл `kb_version_check_and_gap_candidates`
нужно трактовать **не как KB-факты**, а как «темы/вопросы, которые команде стоит проверить и,
если нужно, оформить отдельно». Я ничего в базу не вливал.
