## 1. Памятка менеджеру (обновлена под примечания)

- Бот клиенту НЕ пишет. Он кладёт в карточку сделки примечание: «ЧЕРНОВИК БОТА, не отправлено» + бренд + маршрут.
- Твой шаг: прочитать черновик → отправить клиенту через чат AMO (можно с правками — правки очень ценны, мы по ним учимся).
- Маршрут в примечании: «бот считает безопасным» — обычно можно отправлять; «проверить» — взгляни внимательнее (цены/даты/условия); «бот передал менеджеру» — бот сознательно не ответил (острая тема/нет данных) — веди сам.
- Если появился НОВЫЙ черновик по тому же чату — старый устарел, отправляй свежий.
- Видишь смешение Фотон/УНПК в одном тексте — НЕ отправляй, сообщи нам (это баг, не норма).
- Стоп-кран: если черновики систематически вредны — скажи Дмитрию, выключение мгновенное (один файл).

## 2. Стоп-критерии пилота — РЕШЕНИЕ Дмитрия (без изменений по сути)

Немедленный стоп при любом из: выдумка в ОТПРАВЛЕННОМ клиенту; смешение брендов в отправленном; P0 ушёл клиенту от бота; систематическая жалоба менеджера. Мягкий: unedited-rate < X% за неделю → пауза на разбор. **Рекомендация X=30% по ходам на старте.** Кто жмёт стоп: Дмитрий или дежурный менеджер через Кодекса/файл.

## 3. ПДн — РЕШЕНИЕ Дмитрия (без изменений)

Сообщения клиентов уходят в модель для генерации черновика → пункт в политику обработки данных («используются ИИ-сервисы»); журнал черновиков/правок хранится локально на маке (`~/.mango_local/draft_loop/`), вне репозитория и облака; техническая гарантия не-эха ПДн (presale П1 + склонения) работает. Свериться с прошлой консультацией юриста по данным детей.

## 5. Метрики пилота — РЕШЕНИЕ Дмитрия

Северная звезда: **unedited-rate по ходам** (черновик отправлен без правки; автокласс из журнала ТЗ-6: unedited ≥0.95 / edited / replaced + ratio всегда). Базовое ожидание из симулятора ~40%; **рекомендация цели к концу пилота ≥45-50%**. Жёсткие нули: выдумка в отправленном, бренд-микс, P0 мимо менеджера. Диагностика: причины правок (коды появятся из журнала), темы с частым reгеном верификатора, время от черновика до отправки. Режим: 5-7 дней 100% разбора мной → далее еженедельный дайджест + 3 худших черновика цитатами.

## Три решения Дмитрия на утро

1. Стоп-критерии + порог X (рекомендую 30%) + кто жмёт стоп.
2. ПДн-пункт в политику (формулировка) — можно параллельно пилоту на тестовой сделке, но ДО реальных клиентов.
3. Цель unedited-rate к концу пилота (рекомендую ≥45%) + длительность фазы 100%-разбора (рекомендую 5-7 дней).

## Аддендум по адверсариальному аудиту (ночь 10.06) — ОБЯЗАТЕЛЬНЫЕ дополнения

1. **Если гипотеза 0(в) ложна** (исходящие из AMO не видны в Wappi): unedited-rate считается РУЧНЫМ протоколом до мини-ТЗ — Claude раз в день сверяет черновики журнала с отправленными в AMO (выборка 100%, первые дни и так в 100%-разборе). Мягкий стоп-критерий работает по этому ручному счёту.
2. **Окно сопоставления:** 4 часа занижает unedited для вечерних черновиков (менеджер ответит утром → ложный no_reply). До старта пилота поменять на «до конца следующего рабочего дня» ИЛИ исключить нерабочие часы из знаменателя (решит Кодекс по простоте, зафиксировать в отчёте).
3. **Памятка, четыре добавки:** (а) черновик появляется ~1.5-3 минуты после сообщения клиента — если нет дольше 5 минут, отвечай сам, не жди; (б) на голосовые/фото/стикеры черновика НЕ будет — это норма, отвечай сам; (в) хороший черновик отправляй копированием, не перенабирай (иначе метрика посчитает правкой); (г) если правишь ФАКТ (цену/дату/условие) — сообщи в тот же день в [канал: чат с Дмитрием], бот иначе будет повторять ошибку базы.
4. **P0 ночью (письменно):** в текущем пилоте ночные P0 ждут утра — менеджер начинает день с примечаний «бот передал менеджеру», P0 разбирает первыми. Дежурный контур — к этапу автономии (см. NIGHT_ops_one_pager).
5. **Четвёртый жёсткий ноль:** эхо ПДн ребёнка в отправленном клиенту (наравне с выдумкой/брендом/P0).
6. **Метрики:** вернуть разрез бренд/тема/день-ночь из v1. Live-проба дополняется: серия из 3 сообщений подряд (один черновик, не три), голосовое (нет черновика + тип в журнале), новое сообщение при висящем черновике (superseded).
