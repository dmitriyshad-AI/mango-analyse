# 10 идей: качество ответов и продающее мастерство бота Mango

Дата: 2026-06-03. Автор: Claude (субагент, продуктовый угол).
Приоритет: качество текста, продающий тон, живость, убедительность, удержание нити.

---

## 1. Ответ с «частичной ценностью» при уходе к менеджеру

**Суть.** Сейчас при любом хендоффе бот говорит только «передам менеджеру» и ничего больше. Правило: если у бота есть хотя бы один безопасный факт по теме — дать его клиенту ДО хендоффа. Формат: «По [безопасной части] — [факт]. Про остальное подключу менеджера — он ответит до [срок].» Бот перестаёт быть стеной.

**Почему важно для конверсии.** 87% уходов происходят при наличии фактов на руках. Клиент уходит не потому что бот не знает — а потому что не отдаёт. Частичный ответ удерживает внимание, показывает компетентность, снижает тревогу.

**Сложность внедрения:** низкая (правило в диспетчере: answer_partial как отдельный маршрут; уже обсуждался в архитектуре).

---

## 2. «Тёплый мост» перед закрытием сделки

**Суть.** Когда понимание бота фиксирует «клиент готов» (прямой вопрос про запись, оплату, даты), ответ строится по схеме: признать, что решение принято → дать чёткий следующий шаг → убрать трение («я сейчас передам, вам напишет [имя/роль] в течение [срок]»). Никакого «уточните у менеджера», только конкретный маршрут.

**Почему важно для конверсии.** Клиент на стадии «готов» — самый дорогой момент. Сухое «обратитесь к менеджеру» убивает импульс. Чёткий шаг снимает трение и конвертирует намерение в контакт.

**Сложность внедрения:** средняя (новый маршрут в планнере: detect_ready_to_buy → specialist composing pattern).

---

## 3. Финансовый барьер → ценность + оплата по бренду

**Суть.** При любом сигнале ценового возражения («дорого», «нет такой суммы», «есть что подешевле») бот не повторяет цену голо — он: (а) признаёт, что цена — реальный вопрос; (б) даёт ключевую ценность продукта одной фразой (формат/результат); (в) называет конкретный способ снизить барьер для ДАННОГО бренда (Долями, рассрочка — только у Фотона; скидка УНПК 14% за год; второй предмет). Никакого давления.

**Почему важно для конверсии.** Это самый денежный момент диалога. Бот, который говорит только «курс стоит 41 800» на возражение «дорого» — сливает клиента. Признание + ценность + конкретный выход = удержание.

**Сложность внедрения:** низкая (правило objection_price уже в rules_engine; доработать composing pattern).

## 4. «Честная срочность» — факт, не давление

**Суть.** Когда у бота есть факт о реальном ограничении (места в лагере почти распроданы, группа набирается, запись до [дата]) — называть его один раз, коротко, как нейтральный факт, а не нагнетать. Формат: «На следующую смену осталось мало мест — если хотите гарантировать место, лучше записаться сейчас.» После — предложить шаг.

**Почему важно для конверсии.** Реальная срочность (ЛВШ, группы) — законный дожим без давления. Большинство клиентов не знают, что места заканчиваются. Один точный факт решает больше, чем три абзаца описания продукта.

**Сложность внедрения:** низкая (добавить поле `urgency_signal` в базу знаний для фактов с ограниченной доступностью; composing pattern вставляет одну фразу).

## 5. Антиповтор формулировки внутри диалога

**Суть.** Бот запоминает последние 2-3 открывающих фразы ответа в памяти диалога и при генерации следующего хода явно запрещает их повторять. Реализуется детерминированно: перед вызовом composer — передать список запрещённых зачинов из истории диалога.

**Почему важно для конверсии.** Дословные повторы («Спасибо за сообщение» × 3 подряд) разрушают ощущение живого общения быстрее любого другого сигнала. Клиент чувствует стену. Антиповтор — дешёвый способ поднять тон без изменения факта.

**Сложность внедрения:** низкая (детерминированный запрет зачинов в памяти; уже обсуждался в IDEAS_backlog как T7).

## 6. Реакция на «невысказанную потребность»

**Суть.** Планнер учится распознавать сигналы без прямого вопроса: «у ребёнка тройка по физике», «он очень стеснительный», «хочет в Физтех, но мы боимся». Бот не ждёт прямого вопроса — предлагает релевантное сам: пробное занятие, формат, результаты похожих учеников (без гарантий). Одним предложением, не навязчиво.

**Почему важно для конверсии.** Клиент с невысказанной потребностью часто уходит «подумать» — потому что не получил ответа на настоящий страх. Бот, который слышит подтекст, строит доверие быстрее любого факта.

**Сложность внедрения:** средняя (расширить understanding-вызов: добавить extraction поля `implicit_concern`; маппинг в таблице правил → composing pattern).

## 7. Вариативные фразы при уходе к менеджеру

**Суть.** Вместо одной фразы «передам менеджеру» — пул из 5-6 формулировок, выбор детерминированно (по хешу хода или типу темы). Часть формулировок включает: признание ситуации («понимаю, что вопрос срочный»), конкретный срок ответа, мягкий шаг клиенту пока ждёт.

**Почему важно для конверсии.** Хендофф — законный исход для P0 и сложных тем. Но если он звучит одинаково каждый раз, бот превращается в автоответчик. Вариативность + тепло + срок = хендофф как забота, а не стена.

**Сложность внедрения:** низкая (пул фраз в конфиге; детерминированный выбор по хешу).

---

## 8. «Якорь понимания» при переходе темы

**Суть.** Когда клиент резко меняет тему («а вот ещё вопрос — про лагерь»), бот перед ответом на новую тему коротко подтверждает, что услышал предыдущую и не забыл: «По оплате ответил выше — теперь про лагерь.» Не пересказ — одна фраза привязки. Память диалога используется явно.

**Почему важно для конверсии.** Клиенты с несколькими вопросами часто не уверены, что их слышали. Явное подтверждение памяти снижает тревогу и удерживает в диалоге — не нужно переспрашивать.

**Сложность внедрения:** низкая (детерминированная логика: если тема изменилась И предыдущая тема была отвечена — вставить якорь-фразу из памяти).

---

## 9. Снятие риска через пробное занятие

**Суть.** При любом сигнале тревоги/сомнения/недоверия (где угодно в диалоге — «вы точно нормальные?», «а вдруг не подойдёт?», «боюсь, что зря потратим») бот называет пробное занятие как безрисковый первый шаг. Без давления: «Есть пробное занятие — можно попробовать и решить.» Только если пробное действительно есть по бренду.

**Почему важно для конверсии.** Тревога перед покупкой — главный тормоз в образовании. Пробное — это «снятие риска» из playbook Тропиной и Козловой, один из самых работающих приёмов. Бот должен предлагать его проактивно, а не ждать прямого вопроса.

**Сложность внедрения:** низкая (добавить сигнал `client_anxiety` в understanding; маппинг → правило trial_offer по бренду).

---

## 10. Формат ответа по длине вопроса

**Суть.** Короткий вопрос («сколько стоит?») → ответ 1-2 фразы: цена + один способ снизить барьер. Развёрнутый вопрос с контекстом → ответ 3-4 фразы с признанием ситуации. Запрет генерировать три абзаца на «цена?». Реализуется как мета-правило в composing: передаётся длина/тип вопроса + ограничение токенов.

**Почему важно для конверсии.** Длинный ответ на короткий вопрос = сигнал, что бот не понимает. Клиент перестаёт читать. Соразмерность ответа — базовый признак живого собеседника.

**Сложность внедрения:** низкая (мета-правило в composing prompt: query_length → max_sentences; уже обсуждался в IDEAS_backlog как T9).

---

*Все 10 идей не нарушают P0/бренд/факт-ограничения. Каждая предполагает негативный контроль при внедрении.*
