# ТЗ-137 (после аудитора) — снижение лишних уходов на direct-пути: слоты + fallback (добивание ADR-002)

- **Дата:** 2026-06-17. Заказчик: Дмитрий. Постановщик: Claude #1 (+ аудитор по коду). Исполнитель: **D3** (свежий контекст по спайку этого же кода). Цель — давить ~24% настоящего дожима (upsell_miss честной метрики), НЕ усиливая уходы и НЕ вводя детерминированных «пониматоров».
- **Опора:** спайк D3 (`ADR002_direct_determinism_spike_report`). Вывод спайка: слот-регулярки промахиваются РЕДКО → большой переписи не надо; реальное — баг чтения слотов + вредный широкий fallback.
- **Дисциплина:** три независимых флага (A/B/C), все **default OFF** через `_default_off_flag_enabled`, OFF-паритет (выключено = байт-в-байт прежний пилот), микро-набор для итерации, мой регрейд по сырью. Смысл — модели; детерминизму — узкая верификация.

## Флаг A. Фикс чтения слотов (баг, не новый детерминизм)
`direct_path.py:1361` читает `plan.get("slots")`, а план кладёт слоты под **`known_slots`** (`conversation_intent_plan.py:74`; ключа `"slots"` в сериализации плана нет). Direct не видит слот, который план уже извлёк → лишний переспрос/уход.
- Читать `plan.get("known_slots")` (косметический fallback на `plan.get("slots")` для старых логов/replay).
- **Это корректное чтение уже извлечённого слота, не новое понимание.**
- **Риск (аудитор):** фикс домешивает extract-слоты текущей реплики → может вырасти conflict-демоут (больше слотов → больше конфликтов → adjacent → модель уходит). Поэтому:
  - В приёмку — мерить НЕ только уходы, но и `scope_demoted_ids` count (direct_path.py:1249) OFF vs ON. Рост демоутов = парадоксальный рост уходов, разбираемся.
  - `do_not_reask_slots` (план, :167) — только по **confirmed-провенансу** (`_CONFIRMED_SLOT_SOURCES`, :456), иначе бот молча допустит непод��верждённый класс (тише, но опаснее ухода).
  - Замер гонять **в паре с `assumed_scope_guard`** (там `_direct_path_confirmed_slot_scope`, :574), иначе протухший memory-grade зашумит.

## Флаг B. Сузить вредный keyword-fallback (БЕЗ «возврата пусто»)
**ВАЖНО (правка аудитора):** НЕ возвращать пустой пак на `empty_selection`/`timeout` — это прямой РОСТ уходов (пусто → модель уходит, а страж ложного ухода `_direct_path_route_rubric_should_regenerate` :1988 на пустых фактах НЕ срабатывает: `return bool(facts)`). И НЕ гейтить на close-detect (5 слов :1492) — это был бы новый детерминированный классификатор закрытия. Вместо этого:
- **B1.** Сузить keyword-fallback по релевантности: убрать ветки «верхние N без релевантности» (`list(records)[:max_facts]`, :864-865) и `fallback_core`-на-весь-снапшот; оставлять только факты с положительным relevance-score (:869-883). Снимает «demote-на-широких-фактах» без обнуления самоответа.
- **B2.** На `empty_selection` при ОТКРЫТОМ вопросе — путь **переспрос (как tz123)**, а не пусто и не уход. Переспрос держит диалог у бота = снижение уходов.
- **B3.** Расширить страж `_direct_path_route_rubric_should_regenerate` (:1975): при `fallback_reason in {empty_selection, timeout}` и открытом вопросе НЕ пускать модель в `draft_for_manager` без конкретного `missing_facts` даже при пустых фактах (сейчас дыра — `return bool(facts)`).
