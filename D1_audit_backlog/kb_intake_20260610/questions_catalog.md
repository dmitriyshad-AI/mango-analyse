# Каталог реальных вопросов клиентов (Этап А). Клод #2, 2026-06-10

Вход для сверки покрытия базой (Этап Б). Read-only по корпусам; ПДн замаскированы скриптом ДО вывода (`<имя>`, `<тел>`, `<почта>`, `<N>`). Бренды раздельно. P0-классы — политика (менеджер), не дыры.

## Объём обработанного

- **WhatsApp:** 9460 клиентских сообщений-вопросов (по ВСЕМУ диалогу, не только первые; канал исходящий, период 2022-08…2026-05, фактически затих с янв-2026).
- **Telegram УНПК:** 1314 клиентских вопросов (оператор отфильтрован; весь бренд УНПК; сен-2025…май-2026).
- **Звонки:** 65086 звонков просканировано (client-канал, разбивка на вопросительные предложения; дедуп кластеров по звонку).

**Единица частоты:** WA/TG — одно клиентское сообщение; звонки — наличие темы-вопроса в звонке (дедуп). Мультилейбл (сообщение может попасть в несколько кластеров). Счётчики — паттерн-кандидаты, выверены на выборке (верхняя граница), приоритет надёжнее абсолютов.

**Бренд:** WA — по чату; TG — УНПК; звонки — по маркерам в транскрипте, без маркера → «неопр» (бренд силой не приписан). Большой «неопр» по звонкам — норма (короткие реплики без брендовых слов).

**Колонки таблиц:** WA / TG / Звонки — частота по каналам; Ф/У/неопр — бренд (вкл. звонки).

## Заметки метода и ограничения

- Маскирование ПДн — скриптом до вывода (телефоны, email, ФИО из 2-3 заглавных слов, «N класс/лет»). Остаточный риск имён в редких формах — низкий.
- Регекс-классификатор мультилейбл; выверен на выборке (убраны переловли «договор/справка/кабинет/пропуст» из прошлых итераций).
- Звонки: классификация по вопросительным предложениям клиента; бренд по маркерам, большой «неопр».
- A_recommend (явный «что посоветуете») в чате редок: клиенты чаще называют класс+предмет и спрашивают про конкретную программу (это уходит в ФАКТ-кластеры), а не просят подбор абстрактно. В звонках подбор выражен сильнее.
- P0-классы (возврат-претензия, спор оплаты, жалоба, юр-угроза) даны для полноты, но по политике их ведёт менеджер — в дыры базы НЕ идут.
