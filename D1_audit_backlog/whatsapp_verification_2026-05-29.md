## 2. Точность бренд-меток (главная проверка)

Подозревал две ловушки и проверил их адресно:

**Риск «фотон» = термин физики, а не бренд.** Чатов, где метка `foton` держится ТОЛЬКО
на слове «фотон» — 36. Выборка из 8: во всех «фотон» — это название центра, не физика:
«Это учебный центр ФОТОН», «Договор с ООО Фотон», «Квитанция ФОТОН.pdf». Ложных
срабатываний по физике в выборке нет.

**Риск «долями» = обычное «частями».** Чатов, где `foton` держится только на «долями» — **0**.
Риск не реализовался.

**Основные foton-маркеры надёжны:** «скорняжн» (адрес, 323), «cdpofoton» (домен, 304),
«фотон+цдпо» (279). UNPK-сторона — «мфти» (399), «kmipt» (350), «менделеево» — однозначные
имена собственные.

**Mixed — настоящие, не артефакт.** Из 917 mixed только у 9 foton-сторона держится на слабом
маркере, и эти 9 — реальные смешения. Самый показательный (чат 79267280379, клиент сам пишет):
«в школах УНПК МФТИ. Договор заключается с ООО Фотон. Объясните, какая связь?» — это живой
вопрос про связь брендов, ровно тот риск, который описан в CLAUDE.md.

## 3. Корректность матчинга

| Проверка | Результат |
|---|---|
| Сматчено чатов | 2764 |
| `primary_phone` ≠ нормализованный `chat_id` | 0 (полное совпадение) |
| Уверенность high / medium | 616 / 2148 |

Важно про `medium` (78% матчей): это НЕ слабый матч. Идентичность — сам телефон (chat_id =
телефон = ключ в CRM). `high` — это когда вдобавок ФИО из CRM встретилось в тексте чата.
То есть medium = надёжный матч по телефону без бонусной сверки имени, а не «сомнительный».

## 4. P0-формулировки (сверено вручную ранее)

После правки границ слова: refund 108, complaint 21, legal 14, payment_dispute 0 — все
проверены глазами, ложных («обсудим» внутри «суд») не осталось.

## 5. Контракт безопасности провайдера

`whatsapp_context_provider.py`: открывает БД `mode=ro`, без сети, без подпроцессов, без записи
в CRM/runtime (`context_provider_safety_contract`). Соответствует образцу боевого `context_provider.py`.
