# Пакет передачи базы знаний v2

Дата: 2026-05-17

Этот пакет можно передавать Claude, сотрудникам и другим ИИ-агентам для проверки базы знаний Telegram-пилота.

## Что внутри

- `01_agent_pack/` — компактный пакет для ИИ-агента и сотрудника.
- `02_release_core/` — основная машинная база знаний v2: факты, правила брендов, политика бота, отчёт качества.
- `03_audit_summary/` — что было реализовано, какие тесты прошли, какие риски остались.
- `04_stage6_summaries/` — только сводки Stage 6, без подробных таблиц с текстами диалогов.
- `05_specs_and_diffs/` — ТЗ и сравнение базы Claude/Codex.

## Главное правило

Фотон и УНПК МФТИ нельзя смешивать в клиентском сообщении. Для клиента используется только активный бренд:

- `active_brand=foton` — отвечаем только по Фотону.
- `active_brand=unpk` — отвечаем только по УНПК МФТИ.
- `active_brand=unknown` — точные брендовые условия, цены, документы и рассрочку не называем.

Внутреннюю историю клиента можно учитывать полностью, но в клиентский ответ выводятся только факты активного бренда.

## Что просить у проверяющего

1. Проверить, нет ли смыслового смешения Фотона и УНПК.
2. Проверить, не стали ли маткапитал, налоговый вычет и оплата слишком свободными.
3. Проверить, достаточно ли строгие правила для возврата, жалоб, суда и претензий.
4. Проверить, можно ли использовать эту v2-базу как кандидат для ограниченного Telegram-пилота.
5. Отдельно отметить факты из `approval_queue_for_rop_v2.csv`, которые нужно утвердить РОПом.

## Что не включено

Подробные Stage 6 CSV/XLSX с текстами диалогов не включены в пакет для передачи, чтобы не распространять лишние клиентские данные. Они остаются локально в рабочей папке проекта.

В `04_stage6_summaries/` смотреть в первую очередь файлы с префиксами:

- `stage6_codex_*` — реальный прогон через `codex exec`;
- `stage6_fake_*` — быстрый проверочный прогон без реальной модели.
