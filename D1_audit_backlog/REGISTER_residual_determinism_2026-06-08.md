# РЕЕСТР остаточного детерминизма — где бот ещё «тупее модели» (аудит 2026-06-08)

**Запрос Дмитрия:** где ещё лазанья / детерминизм, подменяющий понимание. Аудит по коду (не по памяти). Главный вывод: **прямой путь починил ВЫХОД (модель пишет прозу), но ВХОД остался детерминированным** — тему и факты выбирает keyword-код ДО модели, а на выходе судит скоуп-слепой числовой гейт. Модель получает заранее суженную картину и не может её исправить.

## А. Легитимный детерминизм (оставить по ADR-001)
P0-вход/латч до модели (p0_pre_gate, p0_recall_spec); бренд-токены на выходе (verify_output); сам факт проверки продуктовых чисел; мета/ПДн/identity-сканы выхода; output_sanitizer. Это верификация ВЫХОДА и безопасность — остаётся.

## Б. Лазанья / подмена понимания (по убыванию вреда)

**Б1. Числовой гейт СКОУП-СЛЕПОЙ (оба пути).** dialogue_contract_pipeline.py:4207/4165; фид subscription_llm.py:2746. fact_surfaces = склейка ВСЕХ фактов хода в один мешок чисел (exact+adjacent плоско). → (а) пропускает wrong-scope (онлайн-цена на очный вопрос «подтверждена», если есть в любом adjacent-факте); (б) рубит верный перефраз/арифметику модели (числа нет дословно в мешке → unsupported_product_number → текст сносится в generic-handoff). Прямо запрещает боту думать числами.

**Б2. Тема и факты прямого пути выбираются keyword ДО модели (оба).** conversation_intent_plan.py:301; telegram_pilot_context_builder.py:142-200. topic_id/primary_intent — длинный regex/role-каскад → задаёт required_fact_keys/fact_scope/blocked_neighbor_scopes → решает, КАКИЕ факты вообще увидит модель. Промах keyword по теме = модель не получит факт и не исправит.

**Б3. Wide-pack exact/adjacent — regex по реплике+слотам (прямой).** subscription_llm.py:2360/2283/2260. Раскладка «релевантен/смежен» = regex-сравнение формата/класса/семейства. Верный факт, ошибочно сочтённый «смежным», получает в промпте подсказку «используй только если…» — сужает мышление прямо в промпте.

**Б4. dialogue_memory: слоты/тема keyword (оба).** dialogue_memory.py:869/916. Класс/предмет/формат/цель — регексами на эллипсисе. Даже модельный апдейт зажат keyword-валидацией (:624-665): модели не дают проставить слот, если regex не нашёл слово. Скрипт решает «что понять про клиента».

**Б5. Verifier пропускает качественные выдумки (оба).** subscription_llm.py:6749; pipeline:202/6123. _FACTUAL_CLAIM_RE ловит только ₽/%/даты. Текст-хэндофф без чисел → pure_handoff → смысловой верификатор СКИПается. «Сильные преподаватели / гарантируем интерес» в обёртке хэндоффа уходят клиенту без смысловой проверки.

**Б6. Faithfulness-критик ещё с ВЕТО в лазанье (ШАГ 5 НЕ ВЫПОЛНЕН).** subscription_llm.py:3519-3546. При дефолтном shadow=OFF дословный критик рушит ответ в SAFE_FALLBACK. На прямом пути его НЕТ; в лазанье-фолбэке — рубит. Снять вето (shadow→дефолт) = дешёвый чистый выигрыш.

**Б7. Маршрут «уйти» на прямом пути наследует keyword-тему (прямой).** subscription_llm.py:2827 should_force_manager_only завязан на keyword-topic_id (Б2): промах темы → легитимный вопрос в менеджера до модели. unknown_brand глушит ход. Это политика допуска к автоответу — трогать только после метрик пилота.

**Б8. Прочее в ЛАЗАНЬЕ (на прямом пути не зовётся, но живёт):** close-классификатор «спасибо» (regex, перезаписывает текст); phase2 tone / generic-handoff тексты; estimate/travel keyword-классификация (влияет на числовой гейт → косвенно оба пути!); planner/composite/partial_yield keyword-разбор; fact_scope_spec keyword-таблицы соседних скоупов (кормит retrieval → оба пути).

## ТОП-3 для скачка ума (+ безопасность при черновиковом пилоте)

1. **Скоуп-осознанный числовой гейт (Б1).** Передавать в гейт разметку exact/adjacent + атрибуты факта; сверять число с фактом, чей скоуп совпал с вопросом. Закрывает wrong-scope И перестаёт рубить перефраз. Пилот: ужесточение (ловить wrong-scope) — сразу; ослабление (не рубить перефраз) — под флагом.
2. **Закрыть pure_handoff-дыру верификатора (Б5).** Скип только для дословного шаблона без содержательных утверждений; качественные обобщения — через смысловую проверку. Пилот: безопасно (добавляет проверок, не автономии).
3. **Тему/факты — модели, скрипту только client-safe+бренд-фильтр (Б2/Б3/Б4).** Безопасная часть на пилоте: расширить wide-pack, снять regex-подсказку «не используй» (Б3) — выход и так фильтруется гейтом и идёт в черновик. НЕбезопасно сразу: убирать keyword-допуск автономии (Б7).

**Дешёвый бонус: довести Шаг 5 (Б6)** — shadow-поведение критика дефолтом в лазанье. Гейт остаётся нижним жёстким слоем.

## Порядок (предложение, после приёмки прямого пути)
Если прямой путь принят пилотом: Б1 (ужесточение) + Б5 + Б3-расширение пака — первая волна «ума» (все безопасны для черновиков). Б2/Б4 (тема/слоты на модель) — вторая, крупная. Б7 (автономия) — только по метрикам пилота. Лазанья (Б8) выводится из эксплуатации по ADR-002, когда прямой путь станет магистралью.
