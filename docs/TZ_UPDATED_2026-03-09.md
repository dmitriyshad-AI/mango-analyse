# Обновленное ТЗ и статус (2026-03-09)

## 1. Изначальная цель

Собрать локальную систему для экспорта Mango Office:

1. Массовый ingest звонков (до 100k).
2. Распознавание диалога с ролями и таймингами.
3. Структурный конспект разговора для CRM:
   - краткая история контакта;
   - отдельные поля (ФИО, контакты, класс, интересы, предметы, возражения, next step).
4. Подготовка данных для приоритизации повторного обзвона.
5. Работа на MacBook в стабильном режиме (UI + фоновая обработка).

## 2. Что реализовано

1. Ingest с парсингом имени Mango-файла: дата/время, номер, менеджер, call_id.
2. Dual-ASR контур (MLX + GigaAM) с merge в один финальный текст.
3. Диалоговый экспорт по ролям и таймингам (`*_text.txt`) + варианты ASR (`*_variants.json`).
4. Resolve-этап (оценка качества, rescue/fallback, ручная очередь).
5. Analyze-этап с нормализованной схемой `v2`:
   - `history_summary`;
   - `structured_fields`;
   - `quality_flags`;
   - совместимость со старыми ключами.
6. Экспорт для ручной проверки/CRM-полей через CLI.
7. GUI и stable_runtime запускатели.

## 3. Что еще не закрыто до полного production

1. Полноценный боевой amoCRM rollout (валидированные field_id, staged rollout, e2e проверка).
2. Формальный контроль версий/релизов (git baseline + release tags).
3. Финальные правила качества для случаев со спорной очередностью реплик.

## 4. Целевой результат текущего этапа

1. Надежно получить массовые расшифровки и структурные конспекты локально.
2. Иметь операционный контур: повторные запуски, восстановление, очереди ручной проверки.
3. Подготовить стабильный фундамент для следующего слоя: массовый `analyze` и CRM загрузка.
