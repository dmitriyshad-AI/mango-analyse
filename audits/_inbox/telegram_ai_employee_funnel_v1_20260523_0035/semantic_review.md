# Semantic Review

## Verdict

`PASS_WITH_NOTES`

## Что прошло

- Новый слой не расширяет P0-автономность: если воронка видит `p0_manager_only` или `manager_only_p0`, автономный ответ блокируется.
- Известные поля клиента и диалога передаются в prompt как `known_slots`, а повторный запрос класса/предмета/формата ловится guard-ом.
- Менеджерская сводка отделена от клиентского текста и сохраняется только в metadata.
- В сводке есть практический следующий шаг для менеджера и предупреждение о запретах.
- Prompt явно закрепляет: не выдумывать важнее, чем отвечать конкретно.
- Brand isolation не ослаблялся.

## Блокеры

Блокеров для внутреннего пилота нет.

Блокеры для расширения на внешний поток:

- Полный v8 dynamic и targeted16 после этой правки не прогонялись.
- Нет подтверждённой политики ответа на “вы кто / вы бот / как вас зовут?”.
- Не проверено на живых диалогах сотрудников после перезапуска.

## Неблокирующие риски

- `funnel_state` использует простые эвристики, а не полноценное понимание всех продуктовых нюансов.
- Сводка менеджера может содержать персональные данные из уже имеющегося локального контекста; это допустимо для внутреннего пилота, но требует отдельной политики экспорта.
- Метрики воронки считаются из metadata и зависят от корректной записи runtime.
- Новая логика улучшает память и следующий шаг, но сама по себе не делает ответы “идеальными”: дальше нужен большой диалоговый тюнинг.

## Недостающие проверки

- targeted16 dynamic после перезапуска бота.
- v6/v5 быстрые regression-паки.
- полный v8 dynamic с сохранением полных транскриптов.
- ручной просмотр 20-30 новых живых сотруднических диалогов.

## Регрессии, которые добавлены

- P0-составной вопрос о цене и возврате останавливает квалификацию.
- Известный клиент не должен заново просить телефон/имя/класс/предмет.
- Вопрос по лагерю спрашивает класс, а не возраст.
- Prompt должен сохранять `funnel_state`, `known_slots`, `missing_slots`, `next_best_question`.
- `funnel_state=p0_manager_only` блокирует автономный ответ даже при LLM-route `bot_answer_self_for_pilot`.
- Store не перетирает funnel/summary при повторном idempotent upsert.
- Daily metrics считают funnel progress и повторный запрос известных данных.

## Следующий шаг

Перезапустить ботов, затем прогнать быстрый targeted-пак: P0/static + v8_targeted16. Если нет P0 и массового over-handoff, переходить к полному v8 dynamic.
