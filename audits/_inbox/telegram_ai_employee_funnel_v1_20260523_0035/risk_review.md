# Risk Review

## P0

- P0-логика не ослаблялась.
- Добавлен дополнительный блок автономности по `funnel_state`.
- Составной вопрос с P0 не должен давать частичный автономный ответ.

## Brand Isolation

- Новые модули не добавляют брендовых фактов.
- Prompt продолжает требовать один активный бренд.
- Off-topic и handoff не должны смешивать Фотон и УНПК.

## Персональные данные

- Клиентский prompt дополнен правилом: не называть имя родителя/ребёнка первым из CRM/Tallanto/local history.
- Внутренняя manager summary может содержать известные поля, поэтому её нельзя показывать клиенту.

## Факты и выдумки

- Новый слой воронки не генерирует цены, даты, скидки или расписание.
- Он выбирает только недостающие поля и следующий шаг.
- Фактические ответы остаются в зоне `subscription_llm.py` и KB.

## Операционные риски

- SQLite-схема не мигрировалась; риск несовместимости низкий.
- Старые записи без funnel metadata должны читаться как раньше.
- Runtime нужно перезапустить, иначе изменения не попадут в работающих ботов.
