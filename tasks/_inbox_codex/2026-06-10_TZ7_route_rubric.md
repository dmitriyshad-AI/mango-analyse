# ТЗ-7 (ФИНАЛ после аудита): рубрика выбора маршрута в промпте прямого пути

Цель по разбивке уходов (REPORT_handoff_source_breakdown): модель пишет «передам менеджеру» в 66/101 своих черновиков и обещает менеджера в 46/122 самостоятельных ответов. Даём модели критерий маршрута + обязательное обоснование ухода + содержательные черновики. ВХОДНАЯ правка (промпт), не пост-обработка — этим отличается от провалившейся Волны 5. Маршрут постфактум НЕ меняется кодом нигде (анти-Волна-5 ассерт в тестах).

Флаг: `TELEGRAM_ROUTE_RUBRIC`, default OFF, context-алиас `route_rubric_enabled` (паттерн `_llm_retrieve_enabled`). В `DIRECT_PATH_PILOT_PROFILE_DEFAULT_ON_FLAGS` НЕ добавлять. Реализация сейчас; замер на М1 — после решения по Волне 6. Номера строк ниже — ориентир, искать по именам символов (файл активно меняется).

Пре-замер (по wave4_base, посчитано): missing_facts пуст в 19/101 модельных черновиков → реген сработает на ~19% черновиков (~19 доп. вызовов на 89 диалогов). Принято как допустимая цена.

## Часть 1. Рубрика в промпте (`_build_direct_path_prompt`, после миссии, до фактов)

```text
Выбор маршрута:
- "bot_answer_self_for_pilot" — когда факты из блока «Факты по вашему вопросу» покрывают вопрос клиента и не требуется действие менеджера. Отвечай по фактам уверенно и не обещай, что «менеджер свяжется», — ты уже отвечаешь. Смежные факты покрытием НЕ считаются: на их основе самостоятельный ответ не выбирай.
- "draft_for_manager" — когда фактов не хватает, нужно ДЕЙСТВИЕ или проверка менеджера (оформить запись, отправить документы, проверить оплату, персональные данные) или вопрос требует личной оценки. Обязательно заполни missing_facts: какого факта или какой проверки не хватает. В черновике пиши содержательный ответ по фактам для менеджера — а не «передам менеджеру» как весь текст.
Развилка по процессам: РАССКАЗАТЬ, как устроен процесс (как проходит запись, что после оплаты, есть лист ожидания), — это самостоятельный ответ по факту процесса. ВЫПОЛНИТЬ действие по просьбе клиента («запишите меня», «пришлите договор», «проверьте оплату») — это draft_for_manager.
Запрещено: выбирать "draft_for_manager" на всякий случай при полных фактах.
```

**Правка миссии под тем же флагом (КРИТИЧНО, иначе противоречие):** строка миссии «можно написать „менеджер свяжется" без срока» при ON скоупится: «…— только в черновике для менеджера, не в самостоятельном ответе». При OFF миссия байт-в-байт прежняя.

## Часть 2. Одна перегенерация при необоснованном уходе

Условие (упрощено по аудиту, одинаково при В6 ON/OFF): `route == draft_for_manager` И `missing_facts` пуст И факты в промпте непусты И `model_called == True`. Точка вставки: ТОЛЬКО в else-ветке `_build_direct_path_draft` после успешного `_direct_path_draft_runner`, ДО `apply_semantic_output_verifier` (преблоки/unknown_brand с пустым missing_facts сюда не доходят — у них model_called=False; покрыть тестом). Реген-промпт строится по паттерну `build_semantic_output_regen_prompt`: включает ПЕРВЫЙ JSON-ответ модели + добавка в конец промпта (после JSON-схемы, не разрывая «Верни только JSON»):

```text
В предыдущем ответе выбран "draft_for_manager", но missing_facts пуст, хотя факты по вопросу есть. Либо ответь самостоятельно по фактам, либо заполни missing_facts конкретным недостающим фактом или нужной проверкой менеджера.
```

Ровно ОДНА попытка; таймаут = таймаут draft-вызова; исключение/таймаут → остаётся первый результат. Результат регена проходит верификатор и гейт как обычный ответ. Латентность худшего случая (draft + rubric-реген + верификатор + его реген) принята.

## Часть 4. Тесты

Механика (юнит, fake-runner по паттерну test_subscription_llm_draft_provider):
- триггер-матрица: route=self / missing_facts непуст / фактов нет / флаг OFF / model_called=False → регена НЕТ; все условия выполнены → ровно один реген;
- реген-ошибка → первый результат + rubric_reason;
- preblock unknown_brand не вызывает реген;
- АНТИ-ВОЛНА-5 ассерт: код никогда не меняет route сам (только модель в ответе);
- интеграционный: реген вернул self с числом вне фактов → гейт блокирует (это сценарий Волны 5 — он должен умирать на гейте);
- метаданные пишутся и переживают finalize.

Golden-тест промпта (создать, сейчас дословных тестов промпта нет): OFF == текущий снапшот промпта байт-в-байт; ON == OFF + блок рубрики + скоуп-правка миссии.

Поведенческие сценарии (тревожный ребёнок, «за два предмета выйдет X», выдуманный порядок) — НЕ юнит: кандидаты в смоук-набор М1, перечислить в отчёте для сценариев замера.
