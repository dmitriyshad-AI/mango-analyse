# Backward Compatibility

## Что осталось совместимым

- Публичная структура `SubscriptionDraftResult` не менялась.
- Поля ответа LLM не переименовывались.
- Старые маршруты `draft_for_manager`, `manager_only`, `blocked` сохранены.
- Старые типы сообщений сохранены.
- Для валидных тем поведение не меняется, кроме усиленного prompt.

## Что изменилось намеренно

- Невалидный `topic_id` теперь принудительно переводит результат в `manager_only`.
- Невалидные `alternative_themes` удаляются из результата.
- В metadata сохраняется исходная невалидная тема, чтобы можно было разбирать ошибки модели.

## Почему это безопасно

Изменение сужает поведение системы: неизвестные темы больше не проходят как штатные. В худшем случае менеджер получит больше ручных проверок, но бот не будет опираться на несуществующее правило РОПа.
