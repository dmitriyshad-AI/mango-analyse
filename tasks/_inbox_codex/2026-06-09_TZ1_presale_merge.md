# ТЗ-1: Мерж presale-фиксов П1-П4 в main (приоритет 1, до любых прогонов качества)

Контекст: пакет в ветке `codex/presale-safety-fixes` (ревью PASS), в main НЕ влит (CODE_INVENTORY_2026-06-09, раздел B:82). Это безопасность черновика: эхо ПДн из памяти, fail-soft верификатора («не проверено» вместо молчаливого «корректно»), русские мета-фразы, голые факт-идентификаторы (`*_facts_YYYY_MM_DD`).

Сделать:
1. Перебазировать ветку на актуальный `main`, разрешить конфликты хирургически (правки только presale-файлов).
2. Проверить покрытие П1: маскирует ли он СКЛОНЁННОЕ одиночное имя («передайте Ирине», «для Артёма») — это последний блокер приёмки прямого пути. Если не кроет — расширить в рамках этого же ТЗ + NEG-тест на склонённые формы.
3. Полный pytest на маке (не срез песочницы) + NEG-набор presale: при выключенных `TELEGRAM_PRESALE_*` поведение байт-в-байт прежнее.
4. Мерж в main, отчёт в `tasks/_done` (коммит-хеш, список NEG, что с склонениями).

Критерий приёмки: presale-флаги в main; полный pytest зелёный; NEG зелёные; склонённые имена — покрыты или зафиксирован отдельный блокер; дифф не трогает ничего вне пакета.

НЕ делать: не включать флаги по умолчанию (включение — через профиль пилота, ТЗ-2); не запускать прогоны качества.
