## Принцип (полная формула)
Модель **ПОНИМАЕТ** (детекция подтекста) И **ФОРМУЛИРУЕТ** (тёплый текст) → детерминизм **ЗАПИРАЕТ** факты →
гейт **ВЕРИТ** (проверяет выход). A-det пол = страховочный fallback. Это безопасно ИМЕННО сейчас, потому что
есть выходной гейт (раньше его не было — отсюда те недели выдумок/давления).

## Часть 1 — детекция в модель (runtime-free, снимает потолок подтекста)
Сейчас `selling` (objection/exit_signal) ставит regex (`_selling_signals`, d57bff06) — глух к подтексту
(«серьёзная сумма для нашей семьи», «посоветуюсь с мужем», «а если не подойдёт»). Перенести детекцию в
**модельное понимание планнера**: расширить выход `understand()` (Шаг 3a) полями selling — **0 новых вызовов**
(тот же вызов). Keyword `_selling_signals` ОСТАВИТЬ как **пол детекции** (если модель не emit/не уверена —
keyword ловит явное). Приоритет: модель основная, keyword страховка.
- objection: price|none (Волна 1 — только price); exit_signal: bool.
- Контроль гиперактивности: нейтральный вопрос → objection=none (детекция не должна видеть возражение везде).

## Часть 2 — gen-композиция (флаг)
`TELEGRAM_A_SELLING_MODE = "gen" | "det"` (default `gen`; `det` = откат на пол d57bff06). В gen: когда правило
сматчилось + продающий сигнал (objection=price / exit_signal) — вместо det-шаблона композер модели пишет тёплый
ответ из `{retrieved_facts с числами, active_brand, приём из маппинга, client_message, краткая история}`:
- признать → ценность/способ оплаты → мягкий следующий шаг; тон playbook (Тропина/Козлова);
- числа/проценты — ТОЛЬКО из переданных фактов; НЕ обещать результат/поступление; НЕ давить; бренд только
  active_brand; коротко (1-3 предложения).
Переиспользовать draft-инфраструктуру (draft_prompt_builder/композер), не строить вызов с нуля. +1 LLM-вызов на
продающий ход — счётчик `llm_calls` роль `bot_selling_compose`.

## Часть 3 — ВЫХОДНОЙ ГЕЙТ железно верифицирует gen-текст (ключ безопасности)
После gen-композиции детерминированная проверка; при нарушении → **fallback на A-det шаблон того же правила**
(d57bff06), НЕ в сухой факт и НЕ в хендофф:
1. **Числа verbatim из факта:** каждое число/процент/сумма в тексте обязано встречаться в переданных фактах
   active_brand. Негрунтованное число → fallback (не публиковать).
2. **Нет обещаний/гарантий** (result/admission_guarantee-гарды + «гарантируем/100%/обязательно поступит»).
3. **Бренд один** (cross_brand-гард: нет второго бренда).
4. **P0 цел:** возврат-претензия/спор/жалоба → manager_only, приём не перебивает (как в d57bff06: selling только
   `if not risk_signals`).
5. **Анти-давление (новое):** детектор фраз искусственной срочности/давления («только сегодня», «успейте»,
   «последний шанс», «решайте сейчас») → fallback/чистка. Тон-давление, что детектор не ловит, читаю по сырью.

## NEG-матрица (pytest, детерминир.) — обе ветви флага
Все NEG из d57bff06 (числа из факта; УНПК без рассрочки Фотона; возврат→P0 manager_only; exit без факта пробного
→ без «пробн»; price-no-fact→manager; «сколько стоит»→без довеска) — зелёные в режимах `det` И `gen`. ПЛЮС:
- детекция-подтекст: «серьёзная сумма для семьи» → objection=price (модель); «посоветуюсь с мужем» → exit; при
  этом нейтральный «расскажите про курс» → objection=none;
- gen-специфика: форс gen-вывода с негрунтованным числом → fallback (число не публикуется); с фразой давления →
  чистка/fallback; с упоминанием второго бренда → cross_brand fallback.
