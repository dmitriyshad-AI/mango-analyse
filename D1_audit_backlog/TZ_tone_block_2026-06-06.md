## Зачем (по сырью)

Тон ~59 при УЖЕ включённых Phase2-флагах → дело не во флагах. Корни:
1. **Детерминированные шаблоны-дампы**: «По подтверждённым данным:», «Онлайн/Очный формат подтверждён:», «Нашёл такую группу» — роботские префиксы cite/recover-путей. Худшее: на «потянет ли сын?» бот дважды ответил «платформа МТС-Link».
2. **Нет продажи**: после ответа нет шага; на «спасибо» — три моды провала: пунт к менеджеру / дословный повтор цены (hp_price_offline T2) / сухость.
3. **Составной вопрос игнорируется** (hp_topic_change ×3).
4. **Ценность T1 — дженерик**, хороший ответ только на T2.
5. **Scope-промахи — главный реальный класс ошибок** (не выдумки): правильный факт не той области (онлайн-цены на очный вопрос; «МТС-Link» на вопрос о способностях).
6. Мусор: «Здравствующий момент», «Никакого спешки», «---».

**Цель Дмитрия (дословно):** супер-дружелюбно, продающе, максимум пользы и предугадывание с ПЕРВОГО сообщения; квалифицировать клиента, вести к покупке или получить явное «нет» и красиво завершить; форматирование. Продажа БЕЗ давления. **Решающая приёмка — Дмитрий по сырым диалогам, метрики — прокси.**

## Часть A — Убить шаблонную сухость (детерминированная, волна 1)

**A1. Префиксы-дампы → тёплая рамка.** Убрать роботские префиксы из ВСЕХ точек карты (Тест входа-1 — пререквизит коммита). Замена: 2-3 **ротационных** варианта тёплой рамки, дедуп через `previous_bot_texts`. Формулировки рамок — согласовать со мной до коммита. Рамки пишутся для клиента, НЕ под regex tone_score.

**A2. Close-детект («спасибо»-закрытие).** Закрывающее сообщение = благодарность/прощание БЕЗ нового вопроса. «Вопрос» = «?» ИЛИ вопросительная лексика («подскажите», «а когда», «сколько»…) ИЛИ message_type=question контракта. Поведение:
- тёплое закрытие + ОДИН лёгкий шаг; приоритет шага: контакт (если `contact_requested` ещё не стоит) → пробное (только Фотон) → «возвращайтесь»;
- НЕ handoff-шаблон И НЕ повтор фактов/чисел предыдущих ходов;
- активный P0-латч → детект МОЛЧИТ (уход остаётся);
- pending НЕ-P0 manager-handoff («жду менеджера») → тёплое подтверждение ожидания БЕЗ нового CTA;
- metadata: `close_detect: fired|suppressed_handoff|suppressed_p0|suppressed_pending` + счётчик в build_summary;
- формулировки закрытий — согласовать со мной до коммита (голос бренда, как A1).

Ожидание hp_price_offline-T2 («Спасибо, цена понятна»): тёплое закрытие + 1 шаг; НЕ handoff; НЕ повтор цены.

**A3. Мусор-фразы.** Перечислимый список наблюдённых плохих паттернов (пополняется по регрейдам, НЕ «грамматический детектор»): строка из дефисов `^-{3,}$`; корень «здравствующ»; «Никакого спешки». Мета «Заменяю только этот абзац» закрыта 3c97aee6 — подтвердить NEG-прогоном.

## Часть B — Продающий промпт черновика (главный рычаг, prompt-led)

**B1. Front-load + scope-якорь.** На первый вопрос — максимум пользы сразу: полный ответ из фактов + 1-2 предугаданных смежных факта. **Якорь scope:** предугаданный факт обязан совпадать по бренду, формату (онлайн/очно), классу и предмету с известными слотами; при несовпадении — не подмешивать.

**B2. Квалификация + шаг.** Содержательный ответ заканчивается ОДНИМ естественным шагом.
- **Квалификация — через существующий канал**: `known_slots`/`missing_slots`/`next_best_question`/`funnel_state` (draft_prompt_builder:102-105, правила :239 «используй next_best_question» и :241 «missing_slots пуст — не анкетируй»). Слоты: бренд, класс, предмет, формат, цель. ≤1 квалифицирующего вопроса на ход, встроенного в шаг.
- **Анти-давление:** ≤1 призыва на сообщение; отклонённый шаг не повторять; **правило игнора** — при `recent_ignored ≥ 2` (существующий `_proactive_recent_ignored`, rules_engine:2258; порог уже применяется в :2047) prompt-builder вставляет директиву «в этом ответе шаг не предлагай». Уточнить в дизайне, кто инкрементит счётчик игноров, когда selling-слой заглушен B8. Metadata-счётчик на срабатывание правила.
- **Контакт:** ЗАПРОС контакта — prompt-led, максимум 1 раз на диалог через общий per-dialog флаг `contact_requested` в dialogue memory (читают и промпт-слой, и close-детект A2). ЗАХВАТ контакта — детерминизм, не трогаем (см. B8).
- Явный отказ («не куплю») → красивое завершение без дожима.

**B3. Ценность-T1.** На «за что платим / чем лучше репетитора» — структурный ответ из фактов (формат, контроль, преподаватели, группа) на ПЕРВОМ ходе. Закрывает «ценность-T1» роадмапа.

**B7. Анти-выдумка при свободе** (идёт с ПЕРВОЙ продающей волной): «всё продуктовое только из фактов; чего нет — честно скажи и предложи шаг»; страховки (число-гард, верификатор, детекторы) НЕ ослабляются.

**B8. CTA-дедуп (обязателен вместе с B2).** Sell-флаг ON ⇒ глушится **ВЕСЬ suffix-блок selling-слоя** `_apply_selling_layer` (price_objection / exit_signal / readiness / anxiety / unmet_need / proactive_step + gen-режим selling_compose; включая точку вызова :283 — по карте Тест-входа-2). Старые `_phase2_objection_step`/`_phase2_anxiety_step` — туда же.
**ИСКЛЮЧЕНИЕ (критично): `_a2_contact_capture_handoff` (subscription_llm:3011, вызов :3005) — это ЗАХВАТ присланного контакта (передача менеджеру, manager_followup_required, no_auto_send), НЕ CTA. Остаётся всегда-ON.** Глушится только CTA-суффикс ЗАПРОСА контакта (`_resolve_a2_proactive_step`/`_a2_proactive_step_text`).
Цена: если промпт не выдал шаг — возражение остаётся без CTA. Приемлемо на старте; счётчик `sell_prompt_step_missing` в metadata; решение о fallback — на чекпойнте волны 2 (архитектор, не Кодекс).

## NEG-контроли

1. P0 (4 типа) → менеджер, строгие тексты, без эмодзи/продажи.
2а. Фотон-«дорого» → ценность из фактов + рассрочка 6/10/12/Долями, без выдуманных скидок.
2б. УНПК-«дорого» → помесячно/семестр/год + скидки 10%/14%, БЕЗ рассрочки/Долями (бренд-граница).
3. «Спасибо»-закрытие → тёплый клоуз + 1 шаг; НЕ пунт; ответ НЕ содержит цен.
4. Явный отказ → завершение без второго дожима.
5. Составной (курс+лагерь) → обе части на T1.
6. Бренд-провокация → каноничная фраза, без продажи чужого.
7. Число-гард: тёплые правки не вносят новых чисел (hard-скан после генеративных шагов).
8. P0-латч активен → close-детект молчит.
9. Счётчик «≤1 призыва на сообщение» по транскриптам (CTA-дедуп).
10. Очный вопрос → в ответе НЕТ онлайн-цен (класс ov_value_expensive, scope-якорь).
11. Мета «Заменяю только этот абзац» не воспроизводится.
12. Клиент прислал телефон при sell-флаге ON → захват сработал (manager_followup_required стоит).
13. «Спасибо! А когда старт?» → close_detect НЕ срабатывает, вопрос отвечен.
14. «Спасибо, жду менеджера» (pending handoff) → подтверждение ожидания БЕЗ нового CTA.
15. Возврат не-P0 (гипотетический) → спокойный ответ без рамки/эмодзи/CTA.
