# Runbook: единый каталог вопросов клиентов

Дата: 2026-05-13

## Что запускает команда

Команда собирает вопросы клиентов из звонков, Telegram и почты, объединяет похожие вопросы в классы, создает черновые шаблоны ответов и готовит таблицу для проверки РОПом. Команда безопасная: она не пишет в CRM, не отправляет сообщения, не запускает распознавание звонков и не меняет runtime-базы.

## Запуск

Из корня проекта:

```bash
cd "/Users/dmitrijfabarisov/Projects/Mango analyse"
PYTHONDONTWRITEBYTECODE=1 PYTHONPATH=src python3 scripts/build_customer_question_catalog.py \
  --project-root . \
  --out-root product_data/question_catalog \
  --since 2025-01-01T00:00:00+00:00
```

Проверка кода:

```bash
PYTHONDONTWRITEBYTECODE=1 PYTHONPATH=src python3 -m pytest -q tests/test_question_catalog_*.py
```

## Как читать статусы

`template_ready_needs_current_fact` - можно готовить ответ, но сначала нужно взять актуальную цену, расписание, адрес, скидку, документы или программу из утвержденного источника.

`draft_answer_exists_needs_review` - есть черновик ответа, но его должен проверить РОП.

`manager_only` - бот не должен отвечать сам. Такие темы передаются менеджеру.

`needs_rop_answer` - данных недостаточно, нужен эталонный ответ от РОПа.

## Какой порядок работы для РОПа

1. Открыть `rop_question_review_pack.xlsx`.
2. Отсортировать по `count_total` от большего к меньшему.
3. Начать с самых частых вопросов: стоимость, расписание, адрес, запись, документы, программа.
4. Для каждого класса выбрать решение: бот может отвечать, только менеджер, нужен факт, нужен новый ответ.
5. Для классов с ценой и расписанием указать, из какого актуального файла брать факт.
6. Запретить классы, где ответ зависит от сложной личной ситуации клиента.
7. Заполнить `approved_question_answers_draft.xlsx`: решение РОПа, финальный текст, комментарий.
8. После правок собрать вторую версию каталога и подключать только утвержденные классы в предпросмотр ответа бота.

## Что нельзя делать на основе текущих файлов

Нельзя сразу давать боту право отправлять ответы клиентам. В текущем виде это не финальная база знаний, а подготовленный рабочий материал.

Нельзя брать цены и расписание из старых сообщений менеджеров без проверки свежести. Вопросы с такими темами специально помечены как требующие актуального факта.

Класс `общий вопрос` уже сокращен до 292 элементов, но это все еще остаточный класс. Его можно проверять позже, после топ-100 приоритетных классов.

## Когда пересобирать

Каталог стоит пересобирать после каждого крупного пополнения источников:

- появилась новая выгрузка звонков;
- обработан новый блок почты;
- добавлен новый экспорт Telegram;
- РОП утвердил или изменил часть шаблонов;
- появились новые актуальные цены или расписание.

## Контрольный чек-лист после сборки

1. Тесты `tests/test_question_catalog_*.py` проходят.
2. В `source_coverage_report.md` все нужные источники имеют статус `processed`.
3. В `question_catalog_summary.json` число вопросов не упало неожиданно.
4. В выходных текстовых файлах нет телефонов и email клиентов.
5. Все классы с ценами и расписанием требуют актуальный источник фактов.
6. `rop_question_review_pack.xlsx` открывается и содержит понятные статусы.
7. `answer_quality_check_report.json` имеет вердикт `pass`.
8. `channel_preview_approved_context_pack.json` не содержит автоутвержденных ответов.
