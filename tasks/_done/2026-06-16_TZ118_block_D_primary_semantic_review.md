# TZ-118 Block D Primary: смысловая проверка

Артефакт: D primary для офлайн-назначения ролей в моно-звонках.

Аудитория: внутренний аналитик / офлайн-контур CRM-аналитики.

## Вердикт

`PASS_WITH_NOTES`.

## Что прошло

- Принятое бизнес-решение отражено верно: отклонить segment guard, оставить сырой `codex_selective` для слабых мест в моно-звонках.
- Клиентский и брендовый риск не добавлен: изменение не трогает ответы бота, клиентские черновики, CRM-тексты, записи в AMO/Tallanto и факты базы знаний.
- Смысл режима понятен: `primary` здесь не означает запись во внешние системы, это только офлайн-аналитика.
- Резервный путь разумный: уверенное правило остаётся детерминированным, слабое правило отдаётся в Codex.
- Провал segment guard превращён в регрессионные тесты, которые не дают случайно включить его в `primary`.

## Блокеры

Для офлайн-аналитики блокеров нет.

## Неблокирующие риски

- Остаточные ошибки модели остаются примерно на уровне 6% по принятому gold-замеру. Для офлайн-аналитики это принято решением Дмитрия, но для клиентского live-пути этого недостаточно.
- Название runner-а `run_tz116_mono_role_shadow_real.py` исторически содержит `shadow`, хотя теперь поддерживает `primary`. Это может путать, но поведение не делает опасным.
- Код `segment_guard` остаётся доступен для shadow-экспериментов. В `primary` он принудительно выключен; будущие задачи не должны считать этот guard одобренным.

## Чего не проверяли заново

- В этом коммите не запускался новый реальный Codex-прогон; изменение опирается на уже принятые цифры регрейда от Дмитрия/Claude.
- Новый расширенный gold-набор сверх уже принятого D-замера не создавался.

## Регрессионные тесты

- `test_real_mono_primary_uses_codex_selective_after_regrede_without_segment_guard`
- `test_mono_role_gold50_primary_forces_segment_guard_off`
- Существующие тесты на default-off и Codex-only остаются активны.

## Следующее действие

Переходить к следующему блоку TZ-118. Не возвращать детерминированный segment repair для D без нового дизайна, gold-замера и отдельного регрейда.
