# Wappi note format and "место" sense fix

Дата: 2026-06-25
Ветка: `codex/wappi-context-window`

## Что изменено

- AMO note теперь начинается с содержательного черновика для менеджера.
- Технический блок `ЧЕРНОВИК БОТА, не отправлено` перенесён вниз заметки.
- При длинном черновике обрезается только текст черновика; технический блок, маршрут и safety flags сохраняются.
- Детектор `live_availability` больше не считает бытовое "привезу на место / живём неподалёку" вопросом о наличии мест.
- Реальные вопросы про место/бронь/наличие сохраняют старую защиту: "Можно закрепить место на ЛВШ?" остаётся `live_availability`.

## Semantic review

Вердикт: `PASS_WITH_NOTES`.

Что прошло:
- Менеджер сначала видит полезный текст, а служебные детали ниже.
- Скриншотный кейс "привезу на место, живём неподалёку" больше не уходит в шаблон про свободные места в группе.
- Защита от обещания места не снята: слова про наличие, бронь, свободные/оставшиеся места и закрепление места остаются live-проверкой.

Остаточный риск:
- Это узкий фикс классификации слова "место". Нужен отдельный широкий прогон Wappi/Step1 на реальных диалогах, чтобы поймать соседние смысловые ошибки.
