## ГЛОБАЛЬНАЯ ЦЕЛЬ (зафиксирована 2026-06-06, не меняется)

Автономный ИИ-сотрудник, по качеству соизмеримый с живым менеджером отдела продаж. Ведёт продажу сам: клиент из Яндекс.Директа → сайт (Фотон или УНПК МФТИ) → диалог (предпочтительно Telegram через AMO CRM; вариант — WhatsApp через Wappi) → продвижение от интереса до завершения ОПЛАТЫ, по дороге консультирует, ведёт сделку, отправляет документы. Подцели могут меняться (при смене — первой строкой «что это обесценивает»), глобальная цель — нет.

**Текущая эпоха (с 12.06.2026): черновиковый пилот.** Приёмка качества пройдена, техника закрыта (профиль pilot_gold_v1, база r4.1, монолит разнесён на модули). Целевой контур: Wappi (шлюз к Telegram/Max аккаунтам организаций, НЕ WhatsApp) → бот → примечание «ЧЕРНОВИК БОТА, не отправлено» в сделку AMO → менеджер копирует и шлёт сам. Клиентам бот сам НЕ пишет. Публичные TG-боты — только внутренний тест, ссылки клиентам не раздавать. Режим архитектора: ежедневный разбор черновиков по сырью (первые 5-7 дней — 100%); северная звезда — доля отправленных без правки; жёсткие нули в отправленном: выдумка / смешение брендов / P0 / ПДн. Дальше: автономия по матрице.

## Главное правило: бренды не смешиваются

УНПК МФТИ и Фотон — отдельные организации. Нельзя: сравнивать их условия в одном ответе; называть один партнёром другого; объяснять условия одного в боте другого; упоминать историю сотрудничества. Безопасная фраза: «Это отдельные организации, по вашему вопросу сориентирую в рамках текущего учебного центра». Два отдельных бота; `active_brand` задаётся каналом.

## Архитектура (ADR-001 принят, реализован)

Гибрид: **модель понимает, решает «ответить/передать» и черновит; детерминизм верифицирует ВЫХОД**. Жёсткое (P0-пол по текущему сообщению+латчу, бренд-токены, продуктовые числа, мета/жаргон) — детерминированное, 0 LLM, не разблокируется моделью. Смысловое (производные о продукте, обобщения, диагноз ребёнка) — единый смысловой верификатор выхода (advisory: annotate / downgrade_keep_text / 1 регенерация; fail-soft с таймаутом). Причины уходов — честная таксономия: `is_manager_deferral` + `reason_class` (вход/политика | output_safety | provider_runtime), инвариант «не-self роут ⇒ причина есть». Чистый детерминизм для понимания ЗАПРЕЩЁН (даёт сухость + лишние уходы). Дословный критик не имеет вето над решением отвечать (Шаг 4). Будущий автоном: верификатор недоступен после 1 повтора → автоотправка блокируется; производные — градация по `relation_to_base` (adjacent + не жёсткий класс → можно с маркером неуверенности).

## Дисциплина измерений

Замер = 1 прогон, `--parallel 4`, оба мозга на вехах (итерации — один основной). Сводка судьи НЕ источник правды: регрейд по СЫРЬЮ (транскрипты + греп базы) обязателен; метку fabrication проверять грепом client_safe фактов. Каждая правка — за флагом default OFF, с контрольным отрицательным тестом (NEG); сравнение прогонов — только на одной машине. Реюз плеча (старый ON как новый OFF) разрешён ТОЛЬКО при совпадении всех шести: дерево-коммит, снимок базы, набор сценариев, версия судьи, машина, окружение CLI/модели — и возрасте ≤2 дней; иначе полная пара (решение Дмитрия 2026-06-10; OFF-плечо пары = заодно смоук-регрессия нового канона). Сравнение прогонов с разными `judge_prompt_version` запрещено без пере-суда обеих сторон одним судьёй. Replay `--replay-from` валиден только для вопроса «как бот отвечает на тот же вход» и не валиден для новой динамики клиента; структурные изменения диалога мерить обычной парой. Тон — детерминированная `tone_metric` (судейский тон-балл удалён).

## Semantic review

Зелёные тесты = только formal_pass. Для клиентских текстов/базы/черновиков нужен semantic_pass: факты правдоподобны; цены/проценты/даты не перепутаны; скидки с условиями; бренд не смешан; юр.данные/мета/source_id не в клиентском тексте; high-risk не ослаблены. Гейт: `scripts/run_kb_semantic_review.py --release-dir <актуальный релиз>`.

## Где что лежит

История — `PROJECT_HISTORY.md` (не источник текущей правды). Отчёты Claude — `audits/_inbox/...`. Ключевые документы эпохи — `D1_audit_backlog/`: ADR_001 (архитектура), SPEC_acceptance_smoke (приёмка), REGISTER_keyword_vs_understanding (следующая лазанья — вход), REGRADE_* (регрейды), kb_v6_6_staging.
