# Semantic Review

## Verdict

PASS_WITH_NOTES.

## Что прошло

Каталог ведёт себя как безопасный слой доказательства, а не как генератор клиентского ответа. Он доказывает существование только через свежие client-safe факты и честно возвращает `unknown`, если доказательства нет.

Смысловые проверки:

- УНПК, олимпиадная физика 9 класс онлайн -> `exists` по факту расписания.
- УНПК, летняя школа для 7 класса -> `exists` по факту ЛВШ 5-10.
- УНПК, химия -> `not_offered` по явному отрицательному факту.
- Фотон, химия -> `unknown`, отрицательный факт УНПК не смешан с Фотоном.
- УНПК, только формат онлайн -> `needs_slot`, потому что не хватает предмета или продукта.
- УНПК, биология 3 класс онлайн -> `unknown`, отсутствующий факт не превращён в "нет".
- УНПК, летняя школа широким запросом -> `exists`, отменённая отдельная смена не закрывает весь продукт.
- Фотон, интенсив широким запросом -> `exists` по факту ОГЭ-интенсива, а не по отрицательному EGE-факту.
- УНПК, "кабинет 9" как класс -> `needs_slot`, мусорное число не становится 9 классом.

## Неблокирующие риски

- Каталог пока извлекает оси эвристически из KB-фактов. Это допустимо для shadow/proof-layer, но перед runtime-проводкой нужен дополнительный отчёт по ложным `exists` на большем наборе.
- `issues=55` означает, что часть client-safe свежих фактов не имеет достаточных продуктовых осей. Это не опасно для клиента, потому что такие факты не становятся доказательством, но снижает полноту.
- `exists` сейчас означает "есть подтверждённый продукт/формат", а не "есть свободные места" и не "можно записать". Любая будущая клиентская формулировка должна сохранять эту границу.

## Что НЕ проверено

- Нет runtime-включения, поэтому не проверялся клиентский текст.
- Не проверялось снижение `draft_for_manager` в direct path.
- Не проверялись реальные live-dialog inputs.

## Требуемые регрессии перед runtime

- Сохранить тесты на `unknown != not_offered`.
- Сохранить тест на brand isolation.
- Сохранить тесты на узкие negative facts, `raw_value=False`, payment/enrollment exclusions и шумные числа.
- Добавить paired eval перед любой active-проводкой: route/text diff, P0 floor, brand, no live availability claim.
