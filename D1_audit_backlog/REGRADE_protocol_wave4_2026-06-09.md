## 2. ГЛАВНЫЙ ФОКУС — релевантность low-risk ответов (урок partial-floor)
Волна 3 partial-floor вставляла нерелевантный факт (цену на жалобу). Волна 4 должна давать ПОЛЕЗНЫЙ ответ от модели, а не мусор. Проверка по сырью:

1. **Diff по ходам:** найти ходы, где route разошёлся BASE→GRADED (в BASE уход к менеджеру, в GRADED — ответ). Это и есть эффект Волны 4.
2. **Классифицировать каждый такой ход** (читать транскрипт обоих прогонов):
   - **ВЫИГРЫШ** — в GRADED полезный, по теме ответ из grounded-факта вместо лишнего ухода BASE.
   - **РЕГРЕСС** — в GRADED нерелевантный / мутный / не по вопросу ответ (повтор болезни partial-floor) ИЛИ маркер-штамп без пользы.
   - **ОК-уход** — оба ушли правильно (P0/жёсткое) — не считается.
3. **Маркер неуверенности:** дозирован? Стоит только при реальной low-risk находке, не штампом на каждом ходе (П5 ТЗ). Если штампуется везде → тон/доверие просядут.
4. Доля ВЫИГРЫШ vs РЕГРЕСС среди разошедшихся ходов = вердикт по полезности.
