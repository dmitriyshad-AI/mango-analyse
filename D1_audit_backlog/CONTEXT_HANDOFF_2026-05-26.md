## Что за проект (Mango Analyse)
ИИ-бот-консультант для отдела продаж двух учебных центров — **Фотон** и **УНПК МФТИ**. Это РАЗНЫЕ
организации, их условия НИКОГДА не смешивать в одном ответе (active_brand задаётся каналом). Бот в
режиме ЧЕРНОВИКА: не пишет клиенту сам, готовит ответ + риски, менеджер одобряет. CRM (AMO/Tallanto)
только на чтение. P0 (возврат/жалоба/гарантия результата/спорная оплата) — всегда менеджеру.

## Архитектура v2 (актуальная)
Умная модель ПОНИМАЕТ диалог → строит «контракт ответа» (нужные КЛЮЧИ фактов, не значения) →
детерминированный слой достаёт значения из снимка базы по бренду → черновик строго по фактам →
жёсткий верификатор (бренд/числа/P0/мета) + семантическая проверка (fail-closed) → ремонт/хендофф →
тёплый слой X2 (только форма, выход ре-верифицируется). Корень старых проблем был обратный (keyword-
слой делал понимание, LLM только прозу) — это перевёрнуто.
