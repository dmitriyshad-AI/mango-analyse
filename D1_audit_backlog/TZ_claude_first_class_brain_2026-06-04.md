### Часть A — claude-бэкенд для ВСЕХ ролей (возможность полностью-Claude)
Зеркалить паттерн `--bot-mode claude` для остальных ролей: добавить `claude` в `--client-mode`, `--judge-mode`,
`--memory-mode`, `--semantic-mode`, `--selling-mode` (каждый → `ClaudeJsonModel`/`ClaudeCliRunner` с нужной моделью/
effort). Плюс удобный флаг `--all-brains {codex,claude}` (выставляет все роли разом). Дефолт — codex/gpt-5.5.
```python
# в build_*_model для каждой роли — ветка claude, как уже сделано для bot_mode:
if args.<role>_mode == "claude":
    return CountingGenerateModel(ClaudeJsonModel(model=args.claude_model, reasoning=args.<role>_reasoning,
                                                 timeout_sec=args.timeout_sec))
```
**Нюанс судьи (важно):** при полностью-Claude бот И судья = Claude → риск коррелированной слепоты (модель судит себя).
Поэтому: разрешить СМЕШАННУЮ конфигурацию (напр. бот=Claude, судья=GPT) и оставить её как РЕКОМЕНДУЕМУЮ для честного
замера; полностью-Claude — для самодостаточности на машинах без codex. Дмитрий регрейдит по сырью, так что судья
вторичен, но в отчёте всегда фиксировать, какой судья.
