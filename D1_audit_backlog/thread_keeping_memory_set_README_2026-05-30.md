# Набор замера «удержания нити» — как использовать. 2026-05-30.

Файл: `D1_audit_backlog/thread_keeping_memory_set_20260530.jsonl` (16 строк = simulator_spec +
judge_spec + 14 персон). Это шаг 6 плана памяти (D1_audit_backlog/MEMORY_dialogue_state_and_plan).

## Назначение

Замерить, держит ли бот нить диалога на УТОЧНЯЮЩИХ ходах с эллипсисом — до и после правок памяти
(шаги 1-4 плана). Это baseline: прогнать СЕЙЧАС (как есть), потом перепрогнать после правок и сравнить.

## Что внутри (14 персон, 7 Фотон + 7 УНПК, бренды раздельно)

Паттерны потери нити (поздние реплики не повторяют тему — бот должен помнить из предыдущих ходов):

- `ellipsis_format` — «информатика 10 класс есть?» → «а онлайн?» → «сколько?»
- `ellipsis_grade` — «физика очно для какого класса?» → «а для 9?» → «когда занятия?»
- `pronoun_reference` — «...математика 11 класс» → «это онлайн или очно?» → «почём это?»
- `return_to_topic` — тема → вопрос про длительность → возврат к цене темы
- `ellipsis_camp` (Фотон) — «летний лагерь есть?» → «а в августе в Москве?» → «сколько?»
- `ellipsis_installment` (Фотон) — цена → «частями можно?» → «на сколько месяцев?»
- `ellipsis_year_discount` / `pronoun_discount` (УНПК) — цена → «если за год?» → «сколько со скидкой?»
- `premature_format` (УНПК) — «онлайн или очно?» (бот НЕ должен сам выбирать формат)
- `control_explicit` (оба) — те же темы, но реплики ПОЛНЫЕ, без эллипсиса; контроль: если и тут плохо —
  проблема не в памяти.

Каждая персона: `expected_route=bot_answer_self`, `success_criteria` = держит тему/не переспрашивает
известное/не уходит зря к менеджеру; `fail_criteria` = потерял тему / подставил соседний факт
(олимпиада/смена/другой класс) / выбрал формат сам. `brand_forbidden` — маркеры другого бренда.

## Как прогнать (та же механика, что батч 5.1+5.2)

```bash
PYTHONDONTWRITEBYTECODE=1 PYTHONPATH=src python3 scripts/run_telegram_dynamic_client_sim.py \
  --scenarios product_data/telegram_dynamic_test_sets/thread_keeping_memory_set_20260530.jsonl \
  --snapshot product_data/knowledge_base/kb_release_20260530_v6_4_team_answers/kb_release_v3_snapshot.json \
  --out-dir runs/<дата>_thread_keeping_baseline
```

Набор маленький (14) — прогон быстрый. Файл лежит в D1_audit_backlog (моя зона); при прогоне
скопировать в `product_data/telegram_dynamic_test_sets/` и в bundle M1 (это сделает Кодекс/я при сборке
замера — НЕ сейчас, M1 занят батчем 5.1+5.2).

## Что мерить (регрейд по сырью, яделаю)

По каждой персоне на каждом ходе: ответил ли бот в теме предыдущего хода, не переспросил ли уже
названное (класс/предмет/формат из held_facts), не подставил ли факт соседней темы, не ушёл ли к
менеджеру при наличии факта. Метрика «удержания нити» = доля ходов без потери. Контрольные
(`control_explicit`) должны быть заметно лучше эллиптичных — если разрыв большой, корень именно в
памяти/эллипсисе; если плохо и на контролях — проблема в фактах/маршруте, не в нити.

## Важно

- Это инструмент ИЗМЕРЕНИЯ, не финальный holdout (`do_not_tune_against=false`): на нём можно итеративно
  мерить эффект шагов 1-4. Перед публичным пилотом сделать отдельный свежий holdout.
- Реплики — канва; клиент-модель играет живо, но порядок тем сохраняет.
- held_facts завязаны на реальные продукты (классы 5-11, информатика/физика/математика, онлайн/очно,
  лагерь Москва август, рассрочка Фотон, скидка за год УНПК) — факт для проверки существует в KB v6.4.
