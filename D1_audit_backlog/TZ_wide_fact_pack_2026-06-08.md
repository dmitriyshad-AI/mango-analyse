# ТЗ — Широкая скоупированная выдача фактов для прямого пути (ФИНАЛ, синтез Кодекс+аудит)

**Дата:** 2026-06-08, вечер. Ночью — приёмочный смоук, это ТЗ коммитится ДО него.
**Промерено аудитом:** client-safe v6.6 = 607 фактов (foton 271/unpk 336); «все по бренду» = 17-21k токенов — отклонено; темы влезают целиком (pricing 7.5k симв., лагерь ≤4.1k, документы 1.5k), кроме schedule (14-18k — единственная режется); текущий потолок прямого пути 18 фактов (_direct_path_context_fact_items :2038) — корень потерь.

## Контракт wide scoped pack

1. **Hard filters (детерминизм решает только «что НЕ войдёт»):** `active_brand` + client-safe предикат ДОСЛОВНО: `allowed_for_client_answer && !forbidden_for_client && !internal_only && (valid_until отсутствует || >= today)` (летние факты с valid_until=2026-08-31 — фильтр по дате обязателен). `internal_text`/`manager_check_text` НЕ сериализуются никогда. Факты с route_policy=draft_for_manager и bot_template_required ВКЛЮЧАЮТСЯ (черновиковый пилот; решение записано здесь).
2. **Темы — recall-first:** объединение (тема текущего хода + латч темы из памяти); при неуверенности — 2 категории, не argmax.
3. **Два блока в промпте — РАЗМЕТКА, не фильтр:** «факты по вашему вопросу» и «смежные — используй только если вопрос реально про это». Источник разметки — известные СЛОТЫ из памяти (класс/формат/продукт), НЕ regex по текущей реплике. Инвариант: факт, прошедший hard filters и совпавший со слотами, НЕ может быть вытеснен релевантность-скором.
4. **Пустые слоты (частейший T1):** прайс-факты включаются СИММЕТРИЧНЫМИ ПАРАМИ (оба формата/обе вилки) в exact-блок — вилка с подписями это фича (ov_value_expensive T1 понравился Дмитрию); асимметрия хуже полноты.
5. **Лимиты:** ≤10 000 символов факт-блока, ≤60 фактов; порядок: hard filters → тема влезает целиком → отдать всё (top-N не нужен); не влезает (schedule) → резка по слотам, остаток по релевантности; exact-блок не режется никогда.
6. **Деградация «тема не распознана»:** фиксированный core-пак бренда (~25-30 фактов: годовые цены, формат, старты, адрес, скидки/рассрочка — рассрочка ОБЯЗАТЕЛЬНА в core, это главная УНПК≠Фотон ловушка), selected_category="fallback_core".
7. **Метаданные факта в паке:** fact_key + fact_type + product (есть в базе). Деривацию format/grades НЕ делать сегодня (структурных полей нет, скоуп и так дословно в текстах фактов).
8. **Наблюдаемость:** wide_facts_count, wide_fact_keys, selected_category в direct-metadata каждого хода.

## Закрытие скоуп-слепоты числа-гейта (открыта аудитом)

fact_surfaces = объединение всех фактов промпта → при 60 фактах любое правдоподобное число «подтверждено» (коллизия 49 000 реальна: УНПК-семестр И лагерный тариф). Два слоя:
- **Сегодня (0 стоимости):** пометка в миссию: «каждое число называй вместе с форматом/классом/продуктом факта, из которого взял».
- **Следом (не блокер ночи):** узкий детерминированный прайс-скоуп-чек на СУЩЕСТВУЮЩЕЙ машинерии fact_scope_spec (detect_fact_scopes уже применяется :7643): число в ответе → факт-источник → сверка скоуп-маркеров предложения → wrong_scope → downgrade. Это сверка размеченных строк, не «понимание». Пост-hoc LLM-чек ОТКЛОНЁН (дублирует верификатор).
