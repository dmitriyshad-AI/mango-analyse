# TZ-121 Блок E: смысловая проверка

Артефакт: shadow-режим `cyrillic_v2` для определения бренда в офлайн customer timeline / CRM-аналитике.

Аудитория: внутренний аналитик / CRM-аналитика.

## Вердикт

`PASS_WITH_NOTES` для shadow-этапа перед регрейдом.

## Что прошло

- Фотон определяется по корню `фотон`, включая склонения и склейки.
- УНПК определяется по `унпк` и `мфти`.
- Cross-brand случаи уходят в `unknown`, то есть fail-closed, без угадывания.
- Default остаётся legacy, поэтому живое поведение без явного режима не меняется.
- Shadow trace показывает правило, кандидата, эталон и причину.

## Блокеры

Для shadow-этапа блокеров нет.

## Неблокирующие риски

- Корневой матч `фотон` может ловить небрендовые слова вроде физических терминов. В текущем микро-наборе это не проверялось как блокер, потому что ТЗ явно требует подстроку-корень. Перед primary на полном сырье нужен регрейд trace.
- В текущем `main` legacy уже ловил многие формы Фотона, поэтому ожидаемый эффект `foton->unknown` на микро-наборе равен нулю. Основной выигрыш здесь: cross-brand fail-closed и `МФТИ -> unpk`.
- `МФТИ` теперь трактуется как УНПК в `cyrillic_v2`; это соответствует ТЗ, но может требовать ручного контроля в редких общих упоминаниях МФТИ без учебного центра.

## Чего не проверяли

- Полные реальные наборы не запускались по ТЗ.
- Модельные вызовы не делались: блок E детерминированный, `llm_calls_total=0`.
- E primary не включался.

## Регрессионные проверки

- `test_infer_brand_cyrillic_v2_foton_root_and_cross_brand_fail_closed`
- `test_tz121_brand_e_micro_shadow_reports_root_matching_and_fail_closed`
- `test_tz121_brand_e_micro_runner_rejects_non_shadow_mode`

## Следующее действие

Отдать E shadow-артефакты на регрейд. После одобрения можно отдельным шагом включать E primary и только потом переходить к C.
