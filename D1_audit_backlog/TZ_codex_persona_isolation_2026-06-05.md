# ТЗ — изоляция персоны для бот-вызовов Codex + тон-тест. Для Codex. 2026-06-05.

Автор: Клод #1 (на базе предложения главного Кодекса). Опора — находка [[codex_personality_confounds_tone]]: `codex exec` наследует личную Personality аккаунта (у Дмитрия — Pragmatic) → GPT-черновики бота сухие из-за оболочки, а не модели. Конфаунд A/B. Цель: личный Codex-приложение остаётся Pragmatic для разработки, а **бот-вызовы идут изолированно и нейтрально** — тон задаёт НАШ промт.

## Часть 1 — изолированный bot-run в CodexJsonModel
В `CodexJsonModel.generate` для бот-вызовов запускать `codex exec` БЕЗ личного конфига/правил/персоны, но С СОХРАНЕНИЕМ подписки (auth):
```
codex exec --ignore-user-config --ignore-rules --ephemeral --skip-git-repo-check \
  -C <чистый временный каталог вне репо> -m <модель> --ask-for-approval never \
  --sandbox read-only --output-last-message <out.json> -
```
- `--ignore-user-config` → личный `~/.codex/config.toml` (Pragmatic) НЕ грузится.
- Auth/подписка сохраняются через `CODEX_HOME` (логин остаётся) — НЕ откатываться на OPENAI_API_KEY.
- `-C <чистый каталог>` → проектный `AGENTS.md`/инструкции не влияют на тон.
- `--ignore-rules` → локальные execpolicy не влияют.
- За флагом (напр. `--codex-isolated`, для бота по умолчанию ON; client/judge — позже).
- **Точные имена флагов подтвердить по `codex exec --help`** (Кодекс уже видел --ignore-user-config/--ignore-rules/--ephemeral/-C/-c). Неизвестный `-c personality=...` НЕ использовать.

**Проверка изоляции (обязательно):**
1. Тини-проба `codex exec --ignore-user-config ... "верни {\"ok\":true}"` → валидный JSON, идёт через подписку (НЕ API-ключ, расход аккаунта есть).
2. В контекст НЕ попадают личные/проектные инструкции (лог команды + что подцепилось).
3. `--bot-mode fake` smoke + pytest green.

## Часть 2 — тёплый бот ТОЛЬКО через наш слой (НЕ в этом тесте)
Тон делать промтом/слоем, не «личностью Codex»: явный tone/playbook в bot prompt, 5-10 эталонов «плохо→хорошо», form-only X2-перепись ПОСЛЕ фактов, выходной гейт после переписи, tone-метрика. Граница: «живой, спокойный, участливый, ясный шаг» — НЕ «болтливый/давящий/обещающий». **В ЭТОМ тесте промт НЕ меняем** — измеряем чистый эффект изоляции.

## Протокол теста (5-7 диалогов, отвечает на вопрос Дмитрия)
Набор: `tone_test_set_2026-06-05.jsonl` (6 персон: тревожный родитель, цена, уточнение, совет, возражение, закрытие). Один и тот же bot prompt, меняем ТОЛЬКО оболочку:
- **A (базлайн):** GPT-текущий (CodexJsonModel как сейчас, личный Pragmatic).
- **B (изолированный):** GPT с `--codex-isolated`.
Клод #1 регрейдит по сырью: стал ли B теплее/менее канцелярным при ТОМ ЖЕ промте; безопасность цела (выдумок 0, бренд/P0/диагноз). Если B заметно теплее → конфаунд подтверждён, фикс работает, мозги остаются на подписке Codex.
