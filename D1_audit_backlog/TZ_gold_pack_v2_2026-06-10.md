# ТЗ для отдельного Cowork-диалога: голд-пак v2 — с 12 до ~40 эталонных примеров из звонков лучших менеджеров

Ты — контент-контур проекта Mango. Работа с данными, БЕЗ кода: папка `Mango analyse` только на чтение, выход — в новую папку `D1_audit_backlog/gold_v2_intake/`. Git не трогать.

## Что такое голд-пак и зачем

Голд-пак — эталонные пары «вопрос клиента → ответ лучшего менеджера», которые подаются боту в промпт как образцы СТРУКТУРЫ и ТОНА ответа. Сейчас в паке 12 примеров (`product_data/bot_improvement_candidates_20260523/01_gold_and_few_shot/real_manager_gold_2026-06-08.yaml` — это ОБРАЗЕЦ ФОРМАТА, изучи его первым). Голд доказанно двигает качество, но узок по темам. Цель: ~40 примеров, покрывающих реальный спрос, с «продажностью» лучших живых менеджеров.

## Железные правила (нарушение = брак)

1. **Числа, даты, адреса, цены, проценты в ответах МАСКИРУЮТСЯ** плейсхолдерами `[суммы из фактов]`, `[день из фактов]` — как в образце. История проекта: голд с конкретикой научил бота выдумывать расписание. Ни одного живого числа в manager_response_masked.
2. **ПДн:** никаких ФИО/телефонов/email клиентов нигде, включая поле client.
3. **Бренд** у каждого примера: foton / unpk. Условия брендов не смешивать (рассрочка — только Фотон!).
4. **Не учить уходам:** «передам менеджеру» в эталоне допустимо ТОЛЬКО для действий (оформить запись, прислать документы), не как ответ на информационный вопрос.
5. Каждый пример — с указанием источника (файл + позиция/фрагмент), чтобы его можно было проверить.

## Источники (в порядке ценности)

1. **Расшифровки звонков лучших менеджеров** (Тропина, Козлова — ~700 расшифровок): ищи в `product_data/transcripts/` и соседних папках; не найдёшь за 15 минут — спроси у Дмитрия путь. Их фирменные приёмы, которые надо поймать в примеры: снятие риска («если педагог не подойдёт — переведём»), гашение тревоги, честная срочность (без выдуманного дефицита!), чёткий следующий шаг в конце ответа.
2. **WhatsApp:** `all_whatsapp_chats.txt` (17 МБ; учти — канал исходящий, лучшие ответы менеджеров ищи в середине диалогов).
3. **Карта спроса:** `D1_audit_backlog/DEMAND_map_whatsapp_2026-06-10.md` — частоты тем для выбора покрытия.

## Какие темы покрыть (целевое распределение ~40 примеров)

По 2-4 примера на тему, в порядке частоты спроса: подбор курса по классу/предмету; формат очно/онлайн; запись (как записаться, что дальше); оплата и способы; рассрочка/частями (СТРОГО по брендам: Фотон — частями/Долями, УНПК — помесячно/семестр/год); расписание/дни; лагерь/смены/лист ожидания; документы (оферта, справка для вычета, маткапитал); возражение «дорого» (3-4 примера — главная продажная тема!); возражение «подумаем» / мягкое закрытие; тревога родителя («потянет ли», «не отстанет»); два ребёнка; пробное/фрагмент занятия; платформа/доступ онлайн; перенос/пропуски. Существующие 12 тем не дублировать — дополнять.

## Формат выхода

1. `gold_v2_candidates.yaml` — строго в формате образца (id, brand, topic, mission_gold, client, manager_response_masked, prompt_example) + поле source (файл/фрагмент).
2. `coverage_table.md` — таблица «тема → частота спроса → сколько примеров в v1 → сколько добавлено в v2».
3. `SUMMARY.md` — что взято, что НЕ удалось найти в корпусах (честно), спорные места на решение Дмитрия.

## Приёмка (не твоя зона, но знай конвейер)

Твой выход → смысловое ревью архитектора (бренды/обещания/маскировка) → утверждение Дмитрия → Кодекс подключает как новую версию пака (механизм версий пака в боте уже есть) → замер парой против текущего пака. В бой без этих ступеней не попадает ничего.
