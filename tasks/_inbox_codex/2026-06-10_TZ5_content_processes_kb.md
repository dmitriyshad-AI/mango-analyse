## Часть B. Рендер констант из KB (за флагом, волнами)

B1. Флаг `TELEGRAM_TEMPLATE_FROM_KB`, default OFF. Поиск факта — runtime по `(brand, fact_key)` (пара уникальна в снимке; hash в fact_id зависит от ТЕКСТА факта и протухает при любой правке — по id не искать; id из инвентаря только ориентир). Три текста на константу: `literal_text` (путь OFF — байт-в-байт текущий), KB-текст (ON), `neutral_fallback` без цифр/дат (ON + факт недоступен/просрочен/чужой бренд). Скелет (доработать под реальные структуры):

```python
def _direct_path_template_from_fact(
    *, active_brand: str, fact_key: str,
    literal_text: str, neutral_fallback: str,
    context: Optional[Mapping[str, Any]] = None,
) -> str:
    if not _template_from_kb_enabled(context):
        return literal_text
    snapshot = _direct_path_load_snapshot(_direct_path_snapshot_path_from_context(context))
    fact = _direct_path_fact_by_brand_key(snapshot, active_brand, fact_key)  # реализовать
    if isinstance(fact, Mapping) and _direct_path_client_safe_snapshot_fact(fact, active_brand=active_brand):
        text = _direct_path_snapshot_fact_text(fact)  # diалоговый текст, НЕ сырой реестровый
        if text:
            _trace_template_from_kb(context, fact_key=fact_key, outcome="hit")
            return text
    _trace_template_from_kb(context, fact_key=fact_key, outcome="fallback")
    return neutral_fallback
```

`active_brand` — из контекста ДИАЛОГА; mismatch бренда ⇒ neutral_fallback + трейс (NEG: факт чужого бренда не рендерится никогда).

B2. ⚠ Сырой client_safe_text фактов — реестровый («Фотон: контакты — …») и непригоден как реплика. Для волны 1 в C1 завести `*_bot_template`-факты с готовым диалоговым текстом (каркас литерала + значения), ИЛИ рендерить «каркас литерала + подстановка значения из факта». Волна 1 = только однофактные константы (6-8 шт): адреса×3, контакты×2, даты смен ЛВШ/ЛШ×3. Мультифактные (TAX, MATKAP, INSTALLMENT, скидки) — волна 2 ПОСЛЕ обкатки, отдельным ТЗ.
