# TZ-122 over-handoff tail backlog

Дата: 2026-06-16

## C2: лагерные POS после снятия `wrong_intent_fact`

Статус: `open`

Класс проблемы: `over_handoff` / `fact_selection_wrong_scope`

Примеры:

- `tz122_pos_camp_with_contract`
- `tz122_pos_camp_without_contract`

Что подтверждено:

- после включения `TELEGRAM_WRONG_INTENT_FACT_CALIBRATION=1` ложный `wrong_intent_fact` на лагерных POS снимается;
- часть лагерных ответов всё равно остаётся `draft_for_manager` из-за слоя политики/ретривера и сигналов недостающих фактов;
- это не адресная ветка и не текущая доводка `academic_year/start_date`.

Что не делать в TZ-122:

- не маскировать лагерные POS правилом в `wrong_intent_fact`;
- не ослаблять `camp_scope_mismatch`;
- не включать флаг в профиль до отдельного C2-замера.

Следующий безопасный срез:

1. Отдельно картировать, почему ретривер/политика теряют или недоверяют лагерному intent.
2. Сравнить C2: model fact retrieval / policy route на лагерных POS.
3. Проверить NEG: городской лагерь без проживания не должен получать ЛВШ с проживанием.
4. Только после этого решать, можно ли разрешить автономный ответ по лагерной цене.
