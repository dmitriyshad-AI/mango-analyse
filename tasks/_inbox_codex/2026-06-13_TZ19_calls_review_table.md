# ТЗ-19: большая таблица разборов звонков для ручного анализа Дмитрием

Дата: 2026-06-13. Постановщик: главный архитектор по запросу Дмитрия. Режим: read-only по канонической базе звонков, без перезапуска ASR/Analyse.

## Зачем

Дмитрий хочет глазами пройти разборы звонков (Analyse v7) и сам решить: что из выжимок можно писать в карточки учеников, что нельзя, как дорабатывать. Нужна одна большая таблица-выгрузка.

## Источник

Каноническая база: `stable_runtime/canonical_master_20260523_audio_working_store_v1/canonical_calls_master.db` (после вливания v7, 22 679 строк `analysis_prompt_version=v7`). Только чтение. Бэкап не нужен (не пишем).

## Что в таблице (один звонок = одна строка)

- ID звонка (canonical_call_id).
- Дата/время звонка.
- Длительность и длина транскрипта (символов) — чтобы видеть «короткий/длинный».
- Клиент по Resolve: статус привязки (нашёл одного / не нашёл / неоднозначно) + если нашёл — обезличенный признак (можно tallanto_contact_id, БЕЗ ФИО/телефона в открытой колонке — ПДн отдельным закрытым полем или хешем по решению Дмитрия; по умолчанию маскировать).
- Привязанная сделка AMO (lead_id), если Resolve дал.
- Выжимка Analyse v7: о чём говорили, интерес, следующий шаг, возражения — как отдельные колонки, не одним комом.
- Флаг качества: входит ли звонок в blacklist-77 (ложный «автоответчик») — отдельной колонкой, чтобы Дмитрий видел заведомо подозрительные.
- Бренд/филиал, если есть.

## Формат

- Excel (.xlsx), несколько листов если строк много: «Все», «Только blacklist-77», «Длинные >10к символов». Заголовки закреплены, автофильтр.
- Объём большой (22 679) — если один лист тяжёлый, разбить по диапазонам ID или по месяцам, лист-оглавление.
- Файл с ПДн — НЕ в репозиторий. Класть в `~/Claude Projects/Foton/` (локальная папка Дмитрия) или на Яндекс.Диск по его указанию. В git — только скрипт сборки и обезличенная сводка (счётчики).

## Приёмка

Отчёт в `_done`: путь к файлу, число строк, разбивка по статусу Resolve (нашёл/не нашёл/неоднозначно), сколько blacklist-77, сколько длинных. Скрипт сборки — в `scripts/`, идемпотентный, повторный запуск не плодит дубли.

## Важно

Это разовая выгрузка для анализа, НЕ запись в карточки. Решение «что из выжимок писать в карточки учеников» Дмитрий примет ПОСЛЕ анализа таблицы — это будет отдельное ТЗ.
