**Важно: это НЕ факты.** Только тон и приёмы. Цены/даты/проценты/возвраты/документы — всегда из базы, не из исторических ответов. Всё ниже — поверх безопасности (P0→менеджер, бренд раздельно, scope-гард, число-гард).

## Часть 1 — рубрика тона (пункт 5): тёплый помощник vs канцелярит
Метрика тона мерит присутствие/отсутствие маркеров. Цель — высокий «тёплый», низкий «канцелярит».

**Тёплые маркеры (+):**
- прямой ответ в первой фразе («Да, есть…», «По 9 классу…»), потом детали;
- признание запроса клиента («понимаю, хочется увидеть цену сразу»);
- один понятный следующий шаг от БОТА, не от менеджера;
- человеческая краткость, живые слова вместо клише.

**Канцелярит (−), ловить и срезать:**
- «в рамках текущего учебного центра», «по вашему обращению», «осуществляется», «предоставляется», «уточнить детали»;
- двойные отрицания, безличные конструкции;
- дежурный хвост «менеджер уточнит ближайший шаг» там, где бот сам может дать шаг;
- ответ диапазоном вместо конкретного класса/формата.

**Эталон (из playbook):** «коротко по сути → отделить стиль от фактов → точные условия на менеджера», но СТИЛЬ при этом живой. Линтер тона = детерминированный счётчик канцелярит-маркеров; цель — снижать его, рост «тёплых» мерить судьёй/семантикой.

## Часть 2 — паттерны возражений (пункт 4)
Каждый: триггеры → ход лучшего менеджера → безопасный шаблон бота (без выдумки, scope-гард).

1. **«Дорого»** → снятие риска + ценность, не скидка-рефлекс. Бот: признать + напомнить про рассрочку/Долями (только если факт есть и бренд верный) + предложить подобрать формат под бюджет. НЕ выдумывать скидку.
2. **«Подумаю»** → честная рамка + лёгкий следующий шаг, без давления. Бот: «конечно; если поможет — могу прислать расписание/пробное, чтобы решать на конкретике». Один шаг, не гонка.
3. **«Сравниваю с другими»** → честное отличие своей программы общими словами, без принижения конкурента и без сравнения брендов. Бот: сильные стороны формата по фактам + предложить пробное.
4. **«Не уверен в формате» (очно/онлайн)** → прояснить потребность вопросом (пара к Фазе 1 B1), затем ответ по нужному scope. Бот: один уточняющий вопрос → ответ из факта нужного формата.
5. **«Далеко добираться»** → оценка дороги (Level A, с маркером) + альтернатива онлайн, если есть. Бот: ориентировочная логистика + онлайн-вариант по факту.

## Часть 3 — паттерны тревоги (пункт 6): признать эмоцию, потом ответить
Приём «гашение тревоги» из playbook: сперва короткое признание, потом содержание.

1. **«Боюсь, не потянет / сложно»** → «нормально переживать; программа идёт от базового уровня, преподаватель ведёт в мини-группе» (по фактам про формат) + пробное.
2. **«Поздно начинать / упустили год»** → спокойно: «начать можно и сейчас», честный план входа (старт-даты по фактам), без вины.
3. **«Не уверен, подходит ли уровень»** → не диагностировать самому (страж general_advice): «уровень лучше подобрать на пробном/с менеджером», тёпло, не отфутболивая.

**Граница:** тревога ≠ P0. Реальная претензия/возврат/жалоба → менеджер. Тревога-сомнение → бот успокаивает по фактам + ведёт к шагу.

## Как это станет кодом (вход для конвейер-ТЗ 4/5/6)
- **Тон (5):** детерминированный линтер канцелярит-маркеров в санитайзере + флаг X2-переписи под рубрику; метрика тона в summary раннера.
- **Возражения (4):** детектор триггеров (детерминированный) → выбор безопасного шаблона-приёма; всё через число-/scope-/бренд-гард.
- **Тревога (6):** детектор эмоции → префикс-признание перед содержательным ответом; страж general_advice на «подходит ли уровень».
Все три — в детерминированном слое → одинаково на обоих мозгах. NEG на каждую: P0 не ослаблен, выдумок 0, бренд раздельно, scope держит.
