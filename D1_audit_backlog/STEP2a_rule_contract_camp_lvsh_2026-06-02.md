# Контракт правила (Шаг 2a): «лагерь / ЛВШ». 2026-06-02.

Автор: Клод 1. По эталону. Источник — `_camp_safe_template` (130 LOC, самая сложная) + ~16 камп-констант
(188-316). Прочитано по сырью + памятные факты. Это стресс-тест формата на самой грязной лазанье.

## Как сейчас (лазанья)
130 строк мешают: бренд + распознавание камп-намерения + различение выездной(ЛВШ Менделеево)/городской
дневной по сигналам (проживание/питание/трансфер vs дневн/без ночёвки) + подвид (цена/места/что-входит/
программа/11-класс/ЗВШ/август) + отрицание («не лагерь») + знание класса + выбор из 16 текстов. Плюс
«места» — это live-наличие, не статический факт.

## Целевой контракт (слой 2)

```yaml
rule_id: camp_lvsh
title: Лагерь / ЛВШ / летние школы
intent: camp_inquiry
intent_subvariants: [overview, price, seats_availability, living_transfer, program, grade_specific, zvsh]
# ИЗМЕРЕНИЕ scope (ключевое для кампа)
scope_dimensions:
  format: [residential, city_day]      # выездной (ЛВШ Менделеево, проживание) vs городской дневной
  brand: [foton, unpk]                 # ЛВШ Менделеево есть у ОБОИХ, но смены/условия РАЗДЕЛЬНО
required_fact_keys:
  - camp.foton.lvsh.*                   # смены/даты/цена Фотон (значения в KB)
  - camp.unpk.lvsh.*                    # ЛВШ УНПК (значения в KB)
  - camp.city_day.*                     # городской дневной
  - camp.zvsh.status                    # ЗВШ 2026/27: лист ожидания, дат нет
data_rules:
  residential_vs_city: by_client_signal # проживание/трансфер → residential; дневн/без ночёвки → city
  seats_are_live: true                  # «места/наличие/запись» → live-данные → МЕНЕДЖЕР (не обещать)
  scarcity_is_fact: true                # «места почти распроданы» = подтверждённый ФАКТ, НЕ искусств. дефицит
  zvsh: {dates: none, status: waitlist} # ЗВШ Менделеево 2026/27 — дат нет, лист ожидания
  prices: from_kb_under_verification    # цены ЛВШ в KB, помечены «под сверку» (kc_knowledge_base GDrive)
  grade_specific: true                  # 11 класс ЛВШ — отдельные условия
brand_split: true
blocking_conditions:
  - camp_refund                # лагерь + возврат/претензия → P0 менеджер (C3_camp_refund)
  - seats_or_booking           # «есть места / запишите» → live → менеджер (нельзя обещать наличие)
  - zvsh_dates                 # даты ЗВШ → лист ожидания, дат нет
route_effect: bot_answer_self  # обзор/что-входит/программа из факта; места/возврат/ЗВШ-даты → manager
text_effect: composer_generates_from_data
preserve_exceptions:
  - «не лагерь / не ЛВШ» → НЕ отвечать кампом (отрицание)
  - сигналы residential (проживание/питание/трансфер/менделеево/выездн) vs city (дневн/без ночёвки) — не путать
  - «места почти распроданы» — подавать как ФАКТ, НЕ как искусственную срочность (память: это правда)
  - 11 класс ЛВШ → отдельные условия/цена
  - бренд: ЛВШ Фотона ≠ ЛВШ УНПК (раздельно, не сравнивать)
```

## Распределение по слоям
- **Planner:** `intent=camp_inquiry` + subvariant + format(residential/city) + brand + grade + uncertainty.
  Заменяет сигнальную детекцию (проживание/дневн/менделеево/«не лагерь») из 130 строк.
- **KB-факты:** смены/даты/цены ЛВШ (значения, «под сверку»).
- **Слой 2:** правила (residential_vs_city, seats_are_live→manager, scarcity_is_fact, zvsh=waitlist,
  brand_split) + ссылки на факты.
- **Composer:** обзор/что-входит/программа из факта; места/ЗВШ-даты/возврат → честный дефер/менеджер.
- **Гейт:** camp_refund → P0 manager; seats_or_booking → manager (live); бренд-смешение ЛВШ → block.

## Особенность кампа: live-данные ≠ факт
«Сколько мест / запишите на смену» — это НЕ статический факт, а live-наличие (CRM/Tallanto), которое бот не
читает на запись. Правило `seats_are_live:true` → такие подвиды всегда менеджер. Это отличает камп от цен:
у цен значение в KB-факте, у мест — значения нет вообще (динамика наличия). Третий случай: правило =
«передать менеджеру по live-причине».

## Обязательные НЕГАТИВНЫЕ тесты
1. «есть места на смену 20 июня?» → менеджер (live-наличие), НЕ обещать места.
2. Лагерь + «верните деньги» → P0 менеджер (camp_refund).
3. ЗВШ 2026/27 даты → лист ожидания, дат нет (не выдумать).
4. «места почти распроданы» в ответе → как факт, НЕ флагать как искусственную срочность.
5. Бренд: ЛВШ Фотона и ЛВШ УНПК — раздельно, НЕ сравнивать/смешивать.
6. «не лагерь, обычный курс» → НЕ отвечать кампом.
7. residential vs city: «без проживания, дневной» → городской, не выездной.
8. ПОЗИТИВ: «что входит в ЛВШ?» + факт → проживание/питание/трансфер из факта, route=answer_self.
