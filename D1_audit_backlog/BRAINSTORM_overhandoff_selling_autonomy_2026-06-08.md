# Брейншторм: снизить over-handoff, поднять продажи и автономность без роста рисков (2026-06-08)

**Метод:** 6 субагентов × 10 идей (разные углы) → дедупликация → ранжирование → аудит архитектора-ревизора.
**Главная находка (по сырью):** в 167/167 ложных уходов факт БЫЛ найден (`retrieved_match`), но срублен: ~50% `*_redacted` (число вырезано), ~40% `unsupported_promise`, ~59% verifier/semantic_fallback. Ретривал — чистый keyword/alias (нет эмбеддингов) → прочие no_match = промах по форме, не отсутствие факта. **Корень over-handoff = «знание есть, но мы его выбрасываем», не «нет знаний».** → Приоритет: перестать срубать grounded-факт + чинить retrieval-промах; наполнение базы — вторично.

## Группа 1 — Ретривал и база (агент A)
1. **Ядро-пак бренда** — при пустом матче по теме отвечать из выверенного ядра ~15-25 client-safe фактов бренда. Дёшево, до пилота.
2. **Двухпроходный recall + маркер** — нет точного матча → топ-1 смежный факт по scope&topic с маркером неуверенности. Дёшево, за флагом.
3. **Синонимы/алиасы из данных** — вынести KEY_ALIASES в KB-поле, наполнить реальными формулировками клиентов. Бьёт в корень retrieval_miss. Средне.
4. **Деривация диапазонов** — из атомарных цен собрать «от X до Y» (по бренд×продукт×формат) с пометкой «деривировано». Закрывает обзорные вопросы. Средне.
5. **Частичная выдача составного** — ответить на покрытые подвопросы + по непокрытому маркер. Средне.
6. **Грейдинг по relation_to_base** — exact/adjacent/off; adjacent+не-жёсткий класс → с маркером. Дорого, после пилота.
7. **Аудит покрытия по no_match-логам** — адресный список дыр базы. Дёшево, сейчас.
8. **Разделить retrieval_miss vs coverage_gap** — диагностика: факт есть но не сматчился ≠ факта нет. Множитель для 3/7. Дёшево.
9. **Память→слоты для ретривала** — на эллипсисе дополнять слоты из памяти до поиска. Средне.
10. **Fallback с пользой** — вместо сухого «передам менеджеру» дать процесс+шаг (без чисел). Дёшево.

## Группа 2 — Понимание и маршрут (агент B)
1. **Тройная развилка по confidence** — ответить/уточнить/уйти; низкая уверенность на безопасной теме → уточнение, не уход. Средне, калибровка на пилоте.
2. **Уточняющий вопрос вместо ухода** — 1 точный встречный вопрос при неясном scope (лимит 1, P0 не уточняем). Дёшево, до пилота.
3. **«Не знаю факт» ≠ «к менеджеру по политике»** — разделить причины ухода в решении. Средне.
4. **Partial-yield** — ответить что знаешь + честно отметить чего нет (непокрытое не домысливать). Средне.
5. **Декомпозиция составного с раздельной маршрутизацией** — P0-часть уводит только себя, не топит весь ответ. Средне.
6. **Эллипсис-резолвер перед retrieve** — короткие реплики разрешать по памяти ДО поиска. Чинит потерю нити. Средне.
7. **Смежный факт как вторичная справка** — adjacent в том же бренде с маркером, не cross-brand. Дёшево-средне.
8. **Полезный хендофф с контекстом** — отвечает что может + сводка менеджеру; лид не остывает. Дёшево клиентская часть.
9. **Анти-защёлка хендоффа** — решение пересчитывается каждый ход; латч только для реального активного P0, не отыгранной гипотетики. Средне.
10. **Авто-аудит «ушёл при наличии факта»** — офлайн few-shot отрицательные примеры маршрутизатору. Дёшево, петля.

## Группа 3 — Продающесть и тон (агент C)
1. **Квалификация в первом полезном ответе** — польза + 1 естественный уточняющий вопрос, не анкета. Средне.
2. **Контракт «ответ + следующий шаг»** — каждый self-ход кончается конкретным шагом (пробное/запись/звонок). Дёшево.
3. **Возражения few-shot из лучших менеджеров** — приём (признать→переформулировать→шаг), не вставка фразы; классификатор возражение-продажа vs претензия-P0. Средне.
4. **«Дорого» → цена/мес + рассрочка только где правда бренда** — Фотон рассрочка, УНПК скидка за год; бренд жёстко. Дёшево.
5. **Гашение тревоги родителя без диагноза ребёнка** — снятие риска (мягкое вхождение, перевод педагога), страж general_advice. Средне.
6. **Честная срочность только из фактов дефицита** — whitelist тем (ЛВШ места — факт), с маркером источника. Средне.
7. **Захват лида как первоклассный исход** — callback/запись с фиксацией контакта вместо безличного ухода. Средне, зависит AMO.
8. **Front-load по сегменту** — первое сообщение под источник трафика (ночь/Директ). Дёшево.
9. **Тон-полировщик выхода** — детерминир. слой формы (абзацы, акценты, женский род, анти-канцелярит), факты не трогает. Дёшево, мерится tone_metric.
10. **Память как актив продажи** — ссылаться на сказанное, не переспрашивать, двигать к оплате. Средне.

## Группа 4 — Безопасная автономность (агент D)
1. **Матрица «тема×этап→режим»** (дефолт DRAFT) — автоном клетками поштучно; P0 нет клетки SEND. Средне.
2. **Двойные ворота автоотправки** — SEND только при чистом гейте И confidence И retrieved_match. Дёшево.
3. **Теневой автоном (shadow-SEND)** — бот метит «отправил бы», но не шлёт; сверяем с одобрениями. Дёшево, в пилоте.
4. **unedited-rate по клетке как ворота расширения** — DRAFT→SEND по накопленному доверию, авто-откат. Средне.
5. **Узкое ядро «ночь-1»** (5-7 клеток) — только справочное низкорисковое; цены/дожим/ребёнок — DRAFT. Дёшево.
6. **Разные пороги день/ночь + утренняя ресинхронизация** — ночью строже + очередь постфактум-ревью. Средне.
7. **Авто-эскалация по сигналам неуверенности** — DRAFT по эллипсису/составному/низкому матчу, не «на всякий случай». Лечит over-handoff. Средне.
8. **Стоп-кран и конфиг без передеплоя** — hot-reload матрицы + мгновенный рубильник. Дёшево, до пилота.
9. **Канареечный автоном** — X% трафика на клетке, не 100%; плавный обратимый рост. Средне.
10. **Deferred-SEND с окном отзыва** — таймер + алерт менеджеру до отправки спорного. Средне-дорого.

## Группа 5 — Безопасность как энейблер (агент E) [по анализу 167 ложных уходов]
1. **Calc-over-grounded** — детерминир. калькулятор над retrieved_match-числами (сумма + 1 скидка) с белым списком операций. Закрывает «посчитай 2 предмета». Средне.
2. **Soft-redact с маркером на низкорисковых числах** — размер группы/типовой день → «около…, уточнит менеджер»; цена/процент/даты — жёстко как есть. Дёшево.
3. **Per-clause gating** — гейт по каждой части составного отдельно; P0-часть не топит безопасные. Средне.
4. **Градация порога верификатора по relation_to_base** — exact пропуск / adjacent с маркером / далёкий блок. Средне.
5. **Promise-vs-fact split** — промис-гард только на личные обязательства («гарантирую»), не на пересказ grounded-факта в будущем. Снимает 40% срубов. Дёшево-средне.
6. **Closed-world negative** — «такого нет, есть вот это» как ответ, когда факт-отсутствие подтверждён. Средне.
7. **Скоуп-адресная подача** — класс клиента входит в диапазон факта (5-11 ⊇ 7) → ответить адресно «для 7 класса». Дёшево.
8. **Topic-state latch** — переспрос той же темы наследует режим+факт, не переклассифицируется; P0 перебивает. Средне.
9. **Partial-answer floor** — инвариант: есть retrieved_match-факт ⇒ ОБЯЗАН произнести его до ухода; «передам менеджеру» без факта запрещён. Дёшево, офлайн.
10. **Allowlist-by-topic** — реестр авто-безопасных тем (единый источник для бота/судьи/тестов), хендофф запрещён при факте; деньги/P0 вне списка. Средне, политика.

## Группа 6 — Данные, обучение, операционка (агент F)
1. **Gap-miner** — еженедельный ранжированный список вопросов без факта. Дёшево.
2. **Петля «правка пилота → факт + NEG»** — диф черновик→отправленное в кандидаты-факты/тесты. Средне, зависит пилота.
3. **Gold v2 на unedited-черновиках** — few-shot из одобренного без правок. Средне, зависит пилота.
4. **unedited-rate северная звезда** (по ходам, разрез бренд/тема/ночь) + семплинг руками. Дёшево.
5. **Приоритизация дозаполнения «уход × частота × близость к деньгам»**. Дёшево.
6. **Shadow-A/B на живом трафике** — два черновика, второй в лог; без риска клиенту. Средне, пилот.
7. **Персонализация из CRM-карточки** — класс/предмет/история как контекст с маркером «проверь». Средне, AMO.
8. **Сегментные плейбуки из 703 звонков** — приёмы/тон (не цифры) под сегмент. Дорого, сырьё есть.
9. **Еженедельный авто-отчёт «куда утекает автоном»** — топ уходов + динамика + 3 худших черновика цитатами. Дёшево.
10. **Proven-topic gate** — тема→автоном после ≥K подряд unedited; счётчик обнуляется при смене цены. Дёшево, пилот.

## Аудит архитектора-ревизора + совместное решение

**Замечание аудитора 1 — половина топа уже спроектирована, не изобретать заново.** partial-yield (#1) ≈ Блок partial_yield роадмапа; градация relation_to_base (#4) = ADR-001; эллипсис-резолвер (#6) = диагноз dialogue_memory; матрица автономии (#18) = TZ_night_funnel; unedited-rate (#17) = уже в summary. → Перед реализацией свериться с бэклогом, взять готовый дизайн.

**Замечание 2 — главный инсайт диктует первую волну.** Раз факт был в 167/167 и срублен, приоритет №1 = НЕ наполнение/ретривал, а пакет «перестать выбрасывать найденный факт». Топ-1..5 — это одна связная волна.

**Спор по calc-over-grounded (#2).** Возражение: арифметика над ценами = новое число клиенту. Резолюция: разрешён ТОЛЬКО белый список операций (сумма + одна известная скидка) над retrieved_match-числами одного бренда, результат помечен производным; любой не-grounded вход → запрет. Безопаснее текущего (сейчас бот либо уходит, либо мог бы посчитать в прозе без контроля). NEG: чужой бренд / несуществующая скидка → не считать.

**Спор по soft-redact (#4).** Возражение: маркер «около» на цене опасен. Резолюция: маркер ТОЛЬКО на закрытом списке низкорисковых (размер группы, типовой день, длительность); цена/процент/даты/юр — никогда не маркер, жёстко как сейчас.

**Спор по уточняющему вопросу (#8).** Возражение: допрос/зацикливание. Резолюция: лимит 1 на тему, P0 не уточняем, после 2-го неясного — хендофф.

**Согласие:** автоном (#18) и обучение/CRM (#15,#17,#19,#20) — после пилота / зависят от AMO. Не тащить в первую волну. Продающие дёшевые (#10,#11) — параллельно, но с регрейдом тона.

### РЕШЕНИЕ — что внедряем

**ВОЛНА 3 «Перестать выбрасывать найденный факт» (приоритет, сейчас, офлайн-проверяемо, за флагами default OFF):**
- partial-answer floor (есть факт ⇒ произнеси до ухода)
- calc-over-grounded (арифметика над подтверждёнными числами, белый список)
- promise-vs-fact split (промис-гард ≠ пересказ факта)
- soft-redact + градация relation_to_base (маркер на низкорисковых, цена/дата/процент жёстко)
- скоуп-адресная подача (класс в диапазоне факта)
Все детерминированные, над уже-grounded, жёсткие классы не трогают. Ожидание: over-handoff 60% существенно вниз БЕЗ роста рисков. Дисциплина как Волны 1/2: флаг+NEG на каждую, один симулятор-замер, регрейд по сырью, в main после приёмки.

**Параллельный продающий пакет (дёшево, за флагами, регрейд тона):** тон-полировщик формы + контракт «ответ+шаг» (CTA, анти-повтор) + уточняющий вопрос (лимит 1, не P0).

**Вторая под-волна (после Волны 3):** эллипсис-резолвер перед retrieve + синонимы/алиасы из данных (чинят retrieval-промах). Средняя стоимость.

**После пилота:** автоном-каркас (матрица/двойные ворота/теневой/стоп-кран/канарейка), персонализация из CRM, петля «правка→факт+NEG» + gold v2, gap-miner на живом трафике, proven-topic gate.

**Не делаем (пока):** deferred-SEND с таймером (#D10, дорого, поздний автоном); standalone-плейбуки целиком (дорого, ручная разметка) — берём приёмы по мере надобности.
